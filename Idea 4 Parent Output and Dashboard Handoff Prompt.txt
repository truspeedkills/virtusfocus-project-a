COLLABORATIVE DESIGN SESSION HANDOFF

SESSION MODE: COLLABORATIVE DESIGN
============================================================

This session designs the Parent-Facing Output expansion and
Parent Insights Dashboard — the final idea in the VirtusFocus
pipeline improvement sequence. This idea introduces the first
NON-INSTITUTIONAL output in the pipeline: content designed for
parents in the direct-to-consumer (D2C) ecosystem.

Ideas 1-3 are complete. The pipeline is at its most capable
state (v9.5 / v1.6 / v1.3 / v1.2 / Stage 6 v1.1 / Stage 7
Determination Rules v1.0). All architectural decisions from
Ideas 1-3 are locked and inform this session.

============================================================
WHAT THIS IDEA IS
============================================================

Idea 4 has TWO components:

COMPONENT 1 — EXPANDED PARENT COACHING MESSAGE
  The pipeline already generates a lightweight "Parent Summary"
  in Stage 3 (Coaching Output Engine). The current parent
  message is minimal. This component expands it into a real
  coaching message for parents — helping them understand what
  their athlete is working on, where progress is happening,
  and how to support growth at home WITHOUT turning them into
  the coach.

COMPONENT 2 — PARENT INSIGHTS DASHBOARD
  A new output stream that provides parents with structured
  visibility into their athlete's development. Similar in
  concept to Stage 4 (Coach Insights) but with FUNDAMENTALLY
  DIFFERENT compliance rules. Parents are NOT subject to the
  same institutional/NCAA restrictions as coaching staff.
  Parents CAN see execution behavior data that coaches cannot.

============================================================
WHY THIS IS DIFFERENT FROM EVERYTHING ELSE IN THE PIPELINE
============================================================

Every pipeline output built so far operates under ONE of two
compliance frameworks:

  1. ATHLETE-FACING (Stages 3, 6): Coach Arron's voice, no
     clinical language, no data surfacing, no pattern naming,
     no surveillance framing. Protective of the athlete's
     emotional experience.

  2. INSTITUTIONAL COACH-FACING (Stages 4, 7): Neutral
     institutional voice, no raw execution data, no score
     names, no flag IDs. Protective of institutional safety
     across all staff levels (position coach → AD).

Idea 4 introduces a THIRD compliance framework:

  3. PARENT-FACING (D2C only): Supportive coaching voice
     (not Coach Arron, not institutional — a parent-guidance
     voice), MORE data visibility than coaches get, but with
     a HARD boundary around athlete journaling and personal
     reflections. Protective of the athlete's private
     reflective space while giving parents genuine insight.

This is the first time the pipeline produces output for a
non-coach, non-athlete audience. The compliance rules must be
designed from scratch — they are neither a subset nor a
superset of the existing frameworks.

============================================================
THE D2C PRODUCT MODEL
============================================================

CONTEXT: VirtusFocus operates in TWO markets:

  INSTITUTIONAL: Teams/programs pay for VirtusFocus. Coaches
  get Stage 4 weekly reports + Stage 7 daily signals. Athletes
  get Stage 3 coaching messages + Stage 6 daily snippets.
  All athletes are full-featured. No tiers.

  D2C (DIRECT-TO-CONSUMER): Individual athletes/families
  subscribe directly. The athlete gets the full VirtusFocus
  app experience. Base tier = standalone daily + weekly.
  Premium tier = enriched by weekly pipeline.

THE PARENT APP IS D2C ONLY:
  The Parent App is an optional $5/month add-on for D2C
  families. It gives parents their own app experience on their
  own phone, connected to their child's VirtusFocus account.

  Institutional athletes do NOT have the Parent App — their
  coaches get Stage 4/7 output instead. The parent and coach
  audiences are mutually exclusive by product model.

  This means Idea 4 output is NEVER consumed alongside
  Stage 4/7 output for the same athlete. A parent will never
  see a "Coach Insights" report. A coach will never see a
  "Parent Insights" dashboard. They exist in separate product
  ecosystems.

============================================================
WHAT PARENTS SEE VS. DON'T SEE
============================================================

From the D2C Parent App Overview document:

PARENTS SEE:
  - Parent-facing coaching messages (what the athlete is
    working on, where progress is happening, where support
    may be needed, how to respond constructively)
  - Enhanced dashboard with metrics and insights
  - App participation and consistency patterns
  - Completion patterns and engagement trends
  - Progress over time
  - Key mental performance themes
  - Growth indicators (confidence, focus, resilience,
    ownership, consistency)
  - Practical support guidance tied to development

PARENTS DO NOT SEE:
  - Private journal entries
  - Exact personal reflections
  - Private written responses from the coaching process
  - Raw confidential athlete input

THE CORE BOUNDARY:
  "Honest reflection requires psychological safety. If athletes
  believe everything they write will be visible to a parent,
  the quality and truthfulness of their reflection will drop."

  Journal content is private in BOTH institutional (coaches
  can't see it) and D2C (parents can't see it). This is the
  one rule that crosses all compliance frameworks.

HOW PARENT VISIBILITY DIFFERS FROM COACH VISIBILITY:

  Parents get MORE data visibility than coaches in several
  areas because they are not subject to NCAA/institutional
  compliance restrictions:

  PARENTS CAN SEE (coaches cannot):
    - Participation patterns (did the athlete engage?)
    - Completion data (which components were completed?)
    - Usage behavior (consistency of app usage)
    - Engagement metrics (trends in how actively the athlete
      uses the system)
    - Streak counts (how many consecutive days engaged?)
    - Progress patterns with specific data points

  COACHES CANNOT SEE (institutional compliance):
    - Completion rates
    - Daily completion counts
    - Last login timestamps
    - Engagement scores
    - Compliance or adherence ratings
    - Streak counts

  This asymmetry is intentional and by design. Parents are
  supporters in the home environment. Coaches are evaluators
  in the institutional environment. Different relationships
  require different data boundaries.

KEY DESIGN QUESTIONS FOR THIS SESSION
============================================================

TASK 16 — PARENT COMPLIANCE FRAMEWORK:

Q1. What execution data CAN parents see?
    The D2C Parent App Overview says parents see participation,
    completion, engagement, usage, consistency, progress. But
    how granular?
    Options:
    (a) Categorical only — "Your athlete completed 5 of 7
        days this week" (counts, not rates/percentages)
    (b) Trend-based — "Engagement is improving this week"
        (directional, no specific numbers)
    (c) Full dashboard — daily completion breakdown, streak
        counts, component-by-component engagement, historical
        charts (maximum visibility)
    (d) Hybrid — some metrics as specific data, others as
        trends (calibrated per metric)

Q2. What about the Interpretation JSON fields that coaches
    can't see?
    Stage 4 has an explicit list of "inform-only" fields that
    are never reproduced (risk_flags, narrative_arc, coach_
    insights subfields, raw execution metrics). Do parents get
    access to some of these? Specifically:
    - risk_flags: Should parents see risk indicators?
    - narrative_arc: Should parents see developmental story?
    - Composite scores: Should parents see score bands?
    - Coach flags: Should parents see behavioral warnings?

Q3. How does the parent compliance framework interact with
    the existing Clinical Language Prohibition?
    Clinical language is prohibited across ALL pipeline output
    (athlete, coach, and presumably parent). But the parent
    voice may need different replacement terms than the coach
    voice. "Stress load indicator" works for coaches but may
    confuse parents.

Q4. What is the parent-facing voice?
    Not Coach Arron (that's the athlete's coach). Not
    institutional reporting (that's for staff). Options:
    (a) "VirtusFocus" as the voice — brand speaking to parent
    (b) "Coach Arron" adapted — same coach persona but
        speaking to the parent about their child
    (c) Unnamed coaching voice — authoritative but warm
    (d) Something else?

Q5. Does the parent output use the athlete's name or "your
    athlete"?
    Coach output uses the athlete's first name. Parent output
    could use the athlete's first name (more personal) or
    "your athlete" (more formal/protective).

TASK 17 — PARENT COACHING MESSAGE:

Q6. Where does the Parent Coaching Message come from?
    Options:
    (a) Stage 3 expansion — add parent message as a third
        output of the Coaching Output Engine alongside
        Coaching Message and Deep Dive. Same generation
        session, same JSON input, different audience rules.
    (b) Separate new stage (Stage 8) — independent parent
        coaching engine that reads the Interpretation JSON
        and produces parent-specific output.
    (c) Derived from athlete output — generate the athlete
        CM first, then produce a parent version that
        references the same themes but with parent framing.

Q7. What sections should the Parent Coaching Message have?
    The athlete gets: Opening, Drift/Strength, Growth, Identity
    Anchor, and Micro-Commitment. What does the parent get?
    Possible sections:
    - What your athlete is working on this week
    - Where progress is showing
    - Where support might help
    - How to reinforce growth at home
    - What to avoid (common parent mistakes for this pattern)
    - A specific support suggestion

Q8. How long should the Parent Coaching Message be?
    Athlete CM = 250-450 words. Parent message needs to be
    useful but not overwhelming. Parents are not coaches and
    may not read lengthy documents.

Q9. Does the parent message reference the athlete's CM
    content? Or is it independently generated from the JSON?
    If the parent reads "Your athlete is working on focus this
    week" and the athlete's CM says "This week is about
    preparation consistency" — that's a coherence problem.

Q10. Is there a Parent Deep Dive equivalent?
     The athlete gets a Deep Dive (detailed analysis). Do
     parents get a deeper document, or just the message?

TASK 18 — PARENT INSIGHTS DASHBOARD:

Q11. Is the Parent Insights Dashboard a weekly document
     (like Stage 4) or a live app feature (always updated)?
     Options:
     (a) Weekly generated document — produced after weekly
         pipeline runs, like the Coach Insights report
     (b) Live dashboard — app backend computes metrics in
         real-time from accumulated data, always current
     (c) Hybrid — weekly narrative summary + live metrics

Q12. What metrics appear on the Parent Dashboard?
     From the Parent App Overview, candidates include:
     - Participation patterns (days engaged this week/month)
     - Completion trends (components completed over time)
     - Streak data (current streak, longest streak)
     - WTD execution pattern (Growth/Mixed/Reset week trend)
     - Growth Phase progression (Emerging → Leadership)
     - Confidence and consistency directional signals
     - Bullseye focus pattern (controllable alignment)
     - Journaling engagement (frequency, NOT content)
     - Support suggestions tied to patterns

Q13. Does the Parent Dashboard consume the Interpretation
     JSON directly, or does it need its own processing stage?
     Stage 4 reads the Interpretation JSON. Can the Parent
     Dashboard read the same JSON with different compliance
     filters? Or does it need a "Parent Interpretation" layer?

Q14. Does the Parent Dashboard show daily data?
     Stage 7 produces daily signals for coaches. Should
     parents see a daily signal too? The daily traffic light
     (Green/Yellow/Orange) could be valuable for parents —
     but the coach-facing labels (Aligned/Building/Reset Day)
     might need parent-appropriate translations.

Q15. How does the dashboard handle the athlete's first
     few weeks? Many metrics need 2-4 weeks of data. What
     does a parent see in Week 1?

TASK 19 — PARENT COACHING MESSAGE SPEC:
(No design questions — implementation of Tasks 16-17 decisions)

TASK 20 — PARENT INSIGHTS DASHBOARD SPEC:

TASK 21 — EDITORIAL AUDIT UPDATE:

Q16. Does the parent output go through the existing Stage 5
     Editorial Audit, or does it need a separate audit?
     The current Stage 5 audits athlete-facing Coaching
     Messages and Deep Dives against the athlete compliance
     framework. Parent output has a DIFFERENT compliance
     framework. Options:
     (a) Expand Stage 5 with parent-specific criteria
     (b) Create a separate parent audit specification
     (c) Embed self-validation in the parent generation
         (same architecture as Stage 6 daily snippets)

Q17. What parent-specific audit criteria are needed?
     Candidates:
     - Journal content leakage (THE critical boundary)
     - Clinical language in parent context
     - Parent-as-coach framing (output should NOT position
       the parent as the coach)
     - Data consistency (parent message aligns with parent
       dashboard)
     - Athlete name/privacy in dashboard metrics
     - Tone appropriateness (supportive, not evaluative)

============================================================
KEY CONSTRAINTS
============================================================

1. PARENT APP IS D2C ONLY
   This output NEVER exists in the institutional product. No
   coach will ever see it. No NCAA compliance concerns. But
   the clinical language prohibition still applies (VirtusFocus
   is non-clinical across ALL audiences).

2. JOURNAL CONTENT IS THE HARD BOUNDARY
   The one rule that is absolute across all audiences: no
   journal entries, personal reflections, or raw written
   athlete input appears in ANY parent-facing output. Not
   quoted. Not paraphrased. Not summarized. Not referenced.

3. SUPPORT, NOT COACHING
   The parent output helps parents SUPPORT their athlete's
   development. It does not position the parent as the coach.
   "Support should increase ownership, not replace it." The
   parent should never feel they need to coach their child
   based on this output — they should feel equipped to be a
   better supporter of the process.

4. THE ATHLETE'S TRUST IS THE PRODUCT
   If athletes believe their parent sees everything they enter,
   engagement quality drops. The parent output must be designed
   so the athlete would be comfortable with what their parent
   receives. This is not just a compliance rule — it's a
   product survival rule.

5. PROPORTIONALITY TO PRICE POINT
   The Parent App is $5/month. The output should be valuable
   but not require disproportionate compute resources. Consider
   whether some parent dashboard metrics can be computed
   deterministically (like Stage 7) vs. requiring LLM
   generation.

6. ECOSYSTEM QUESTIONS ALREADY DESIGNED
   The intake form (Task 1) includes Q22-Q25 specifically for
   Idea 4. These fields are available in the Athlete Snapshot's
   Ecosystem Profile section. The parent output can use:
   - Q22: Parent awareness of VF (informed/general/unaware)
   - Q23: Home conversation pattern (supportive/analysis-advice/
     pressure/don't discuss)
   - Q24: Development driver (parent/athlete/coach/self)
   - Q25: Support network (family/peer/coaching/self-directed)
   These calibrate the parent message tone and guidance level.

============================================================
PIPELINE ARCHITECTURE QUESTIONS
============================================================

Beyond the per-task design questions, the session needs to
resolve where parent output fits in the pipeline:

A1. Does the Parent Coaching Message come from Stage 3
    (third output alongside CM + DD) or a new Stage 8?
    Trade-offs:
    - Stage 3: Same JSON read, same generation session,
      guaranteed thematic coherence with athlete CM. But
      makes Stage 3 more complex and mixes audiences.
    - Stage 8: Clean separation, independent prompt tuning,
      can use different model. But risks thematic divergence
      and doubles the JSON read.

A2. Does the Parent Insights Dashboard come from Stage 4
    (parallel output) or a new Stage 9?
    Same trade-offs as A1, plus: the parent compliance
    framework is fundamentally different from the coach
    framework. Running both through the same stage with
    different compliance filters is possible but complex.

A3. Does the parent output need its OWN editorial audit,
    or does Stage 5 expand?
    If parent output is LLM-generated, it needs audit. If
    some dashboard metrics are deterministic (like Stage 7),
    those don't need LLM audit but may need QA monitoring.

A4. Should there be a daily parent signal alongside the
    weekly parent output?
    Stage 7 produces daily signals for coaches. Parents
    could receive a similar daily signal — potentially
    even the same traffic light with parent-friendly labels.
    Or daily parent visibility could be limited to the live
    dashboard.

A5. How does Base/Premium tiering affect parent output?
    Parents of Base athletes only have Mini-JSON + Snapshot
    data. Parents of Premium athletes also have the weekly
    Interpretation JSON. Does this create two tiers of
    parent experience? Is that acceptable at the same
    $5/month price point?
