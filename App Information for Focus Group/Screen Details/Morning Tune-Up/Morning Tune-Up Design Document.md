# Morning Tune-Up: Design Document

**Screen**: Morning Tune-Up
**Phase**: 1, Screen 2
**Version**: 1.0
**Date**: 2026-03-11
**Status**: Design Complete — Ready for Development

---

## 1. Purpose & Context

The Morning Tune-Up is the athlete's first daily activity — a guided, linear flow that helps them step into the day with intention and commitment. It is **not motivation**. It is **identity priming + ownership**.

By the time the athlete exits this screen, they should feel:

> "I know how I want to show up today — and I've committed to it."

This screen is a **pull-and-display system**, not a form. All content is pre-generated by the coaching system and pulled from the database. The athlete reads and commits — they do not create any content.

### What This Screen Is

- A coach checking in before the day starts
- A single linear flow with no branching, no optional exits, and one completion outcome
- A commitment device — checkboxes are acknowledgment acts, not task creation
- Calm, intentional pacing that transitions the athlete from passive → active ownership

### What This Screen Is NOT

- A read-only content display (the old mockup was wrong — this requires active engagement)
- A form for the athlete to fill out
- A gamified challenge screen
- A motivational hype delivery system

## 2. Entry & Exit Points

### Entry Points

| Source | Trigger | Notes |
|--------|---------|-------|
| Home — Primary CTA | Athlete taps **START** on the Morning Tune-Up card | Primary entry path |
| Bottom Nav — Dynamic Action | Athlete taps ☀️ sun icon during morning state | Same destination |

Both entry points lead to the same screen state. If the athlete has a partially completed session (checked items from a prior visit), they resume where they left off (see Section 8.2).

### Exit Points

| Trigger | Destination | Behavior |
|---------|-------------|----------|
| Flow completion | Home (auto-return after ~2 seconds) | Completion state shown briefly, then auto-navigate |
| Back button / swipe back | Home | Checked state is preserved. No warning dialog. |
| App backgrounded | Stays on Tune-Up screen | On foreground return, state is preserved |
| Bottom nav tap | Respective destination | Checked state is preserved for return |

### Post-Completion Re-Entry

After completion, if the athlete taps the Dynamic Action (sun icon) or the completed card on Home:

- **Destination**: Read-only review of the completed Tune-Up
- **Content**: All 6 sections displayed with completed styling (checkmarks shown, no active checkboxes)
- **Navigation**: Back button returns to Home
- **Purpose**: Lets the athlete revisit their Focus Word, Challenge, and Quick Wins throughout the day as reference

## 3. Screen Architecture — Single Scroll Layout

The Morning Tune-Up is a **single scrollable screen** with all sections visible. The Finish CTA remains disabled until all required acknowledgments (Mindset Challenge + 3 Quick Wins) are completed.

```
┌──────────────────────────────┐
│  TOP BAR                      │
│  ← Back        Morning Tune-Up│
├──────────────────────────────┤
│                               │
│  SECTION 1: MINDSET STATEMENT │
│  (read-only, largest text)    │
│                               │
├──────────────────────────────┤
│                               │
│  SECTION 2: FOCUS WORD + QUOTE│
│  FOCUS WORD label             │
│  [WORD] in ALL CAPS, large    │
│  Quote beneath, lighter weight│
│                               │
├──────────────────────────────┤
│                               │
│  SECTION 3: MINDSET CHALLENGE │
│  ☐ [Challenge commitment text]│
│  REQUIRED — must tap to       │
│  proceed                      │
│                               │
├──────────────────────────────┤
│                               │
│  SECTION 4: QUICK WINS        │
│  ☐ Win 1 (static)            │
│  ☐ Win 2 (static)            │
│  ☐ Win 3 (static)            │
│  ALL REQUIRED                 │
│                               │
├──────────────────────────────┤
│                               │
│  SECTION 5: CLOSING STATEMENT │
│  (read-only, identity         │
│   reinforcement)              │
│                               │
├──────────────────────────────┤
│                               │
│  [  FINISH TUNE-UP  ]        │
│  (disabled until 4 checks     │
│   are complete)               │
│                               │
├──────────────────────────────┤
│  BOTTOM NAVIGATION            │
│  Inbox · ☀️ · Home · 🎯 · 📓 │
└──────────────────────────────┘
```

### Visual Hierarchy Rules

1. **Mindset Statement** is the largest text on screen — it anchors the experience
2. **Focus Word** is the second-largest element — ALL CAPS, bold, high contrast (amber/gold)
3. **Mindset Challenge** and **Quick Wins** are the interactive sections — clear checkbox affordance
4. **Closing Statement** is medium-sized — larger than body text, smaller than the headline elements
5. **Finish CTA** is a full-width button at the bottom — prominent but not competing with content
6. **Bottom nav** is persistent

### Scrolling Behavior

- All content is visible on load (no progressive reveal, no dimming)
- On most devices, the screen will require scrolling to reach the Closing Statement and CTA
- The athlete naturally reads top-to-bottom, encountering the checkboxes as they scroll
- No sticky elements other than the top bar and bottom nav

## 4. Section Specifications

### 4.1 Top Bar

| Element | Behavior |
|---------|----------|
| **Back arrow** | Left-aligned. Navigates to Home. Preserves checked state. |
| **Screen title** | "Morning Tune-Up" — centered or right of back arrow |

- No additional icons or actions in the top bar
- No close (X) button — the back arrow is the only exit control

### 4.2 Section 1 — Mindset Statement

**Purpose**: Set emotional tone and steady the nervous system.

| Attribute | Specification |
|-----------|---------------|
| **Content** | 1–2 short sentences. Pre-generated, unique per day. |
| **Typography** | Largest text on screen. Display/headline sizing. |
| **Tone** | Neutral, grounded, coach-like. NOT motivational hype. |
| **Interaction** | None — read only |
| **Visual treatment** | No container/card — text flows directly on the background |

**Example copy** (for mockup reference only — real content is pre-generated):

> "Stay grounded in tough moments. Your steadiness becomes your edge."

**Hard rules**:
- No quotes in this section (quotes belong in the Focus Word section)
- No explanations or elaboration
- No hype language ("Let's crush it!", "Today is YOUR day!")
- No first-person ("I am...") — this is coach-voice

### 4.3 Section 2 — Focus Word + Quote

**Purpose**: Give the athlete one mental filter for the entire day. This is a lens, not content.

| Attribute | Specification |
|-----------|---------------|
| **Label** | "FOCUS WORD" — small uppercase label above the word |
| **Focus Word** | ALL CAPS, bold, large/prominent. Pre-generated, unique per day. |
| **Quote** | Italic or lighter weight. 1–2 sentences max. Pre-generated, unique per day. |
| **Visual treatment** | Contained in a card or distinct visual block. Amber/gold color palette for the word. |
| **Interaction** | None required. Optional: tap to pulse/re-highlight the Focus Word (subtle micro-interaction, decorative only). |

**Important context**: The Focus Word is already visible on the Home screen's Primary Action Card before the athlete enters this flow. The Tune-Up doesn't "reveal" the word — it **contextualizes** it by adding the quote, the challenge, and the wins.

**Example** (for mockup reference):

```
        FOCUS WORD
       STALWART

"Be sure you put your feet in the right place,
          then stand firm."
             — Abraham Lincoln
```

**Hard rules**:
- Only ONE Focus Word per day
- No definitions of the word
- No scrolling or rotating quotes
- Quote attribution is optional (depends on content source)

### 4.4 Section 3 — Mindset Challenge

**Purpose**: Convert intention into an action commitment. This is the first active choice of the day and feeds directly into the Evening Review.

| Attribute | Specification |
|-----------|---------------|
| **Label** | "MINDSET CHALLENGE" — small uppercase section label |
| **Content** | One sentence only. Specific, actionable behavior. Pre-generated, unique per day. |
| **Interaction** | Single checkbox. Must be tapped to acknowledge. **Required to enable Finish CTA.** |
| **Unchecked state** | Empty checkbox + full challenge text |
| **Checked state** | Filled checkbox with subtle check animation. Text weight may darken slightly (affirmed state). |

**Example** (for mockup reference):

> ☐ Hold your composure during one stressful moment today — no reaction.

**Evening Review connection**: When the athlete completes the Morning Tune-Up, the Mindset Challenge text is saved and surfaced during the Evening Review's Win The Day section as one of the 5 Yes/No questions: *"Did you [challenge text]? Yes / No"*. This creates a direct accountability loop — the athlete commits in the morning and evaluates in the evening.

**Data flow**:
```
Morning Tune-Up                     Evening Review (WTD)
┌─────────────────┐                 ┌─────────────────────────┐
│ Mindset Challenge│   ──saves──▶   │ WTD Question (dynamic): │
│ ☑ "Hold your     │                │ "Did you hold your      │
│    composure..." │                │  composure during one   │
│                  │                │  stressful moment?"     │
│                  │                │  [Yes] [No]             │
└─────────────────┘                 └─────────────────────────┘
```

**Hard rules**:
- Cannot be auto-checked or pre-checked
- Only ONE challenge (no multiple challenges)
- No vague language ("try harder", "be better") — must be specific and actionable
- Checking is an acknowledgment act, not a promise to complete

### 4.5 Section 4 — Quick Wins

**Purpose**: Lower resistance while reinforcing identity reps. These are acknowledgments, not tasks.

| Attribute | Specification |
|-----------|---------------|
| **Label** | "QUICK WINS" — small uppercase section label |
| **Content** | Exactly 3 items. **Static — do not change day to day.** |
| **Interaction** | 3 checkboxes. All must be acknowledged. Checking order does not matter. **All 3 required to enable Finish CTA.** |
| **Unchecked state** | Empty checkbox + win text |
| **Checked state** | Filled checkbox with calm visual feedback (checkmark only, no celebration) |

**Static content rationale**: Quick Wins are the same every day by design. The repetition is intentional — like a daily ritual or pre-game routine. The content is chosen during program setup and represents foundational behaviors the athlete commits to daily.

**Design consideration for daily ritual feel**: Since the content doesn't change, the interaction must feel intentional, not rote. Recommendations:
- Sufficient tap target size (48px+ height per item) to prevent mindless rapid-tapping
- Brief, subtle check animation that requires the eye to track (not instant state change)
- Visual separation between items (not a cramped list)

**Example** (for mockup reference):

> ☐ One deep breath before every drill or moment.
> ☐ One positive phrase when doubt creeps in.
> ☐ One decision today that favors growth over comfort.

**Hard rules**:
- No counters or progress indicators
- No journaling or text input
- Not optional — all 3 must be checked
- No "select all" or batch-check option

### 4.6 Section 5 — Closing Statement

**Purpose**: Shift the athlete from doing to being. Identity reinforcement and emotional glue.

| Attribute | Specification |
|-----------|---------------|
| **Content** | 1–2 sentences max. Pre-generated, unique per day. Identity-based language. |
| **Typography** | Smaller than Mindset Statement headline, larger than body text. Medium emphasis. |
| **Tone** | Coach-voice. Identity-affirming. Calm closure. |
| **Interaction** | None — read only |
| **Visual treatment** | Clean, uncontained text. May have subtle visual distinction (e.g., slight padding, different weight) to mark it as the closing moment. |

This section replaces the "I Am" identity declaration from the old mockup. The Closing Statement serves the same psychological function (doing → being) but in coach-voice rather than first-person declaration.

**Example** (for mockup reference):

> "You don't need to be perfect — you just need to keep showing up like someone who refuses to quit."

**Hard rules**:
- No performance metrics or scores
- No score language
- No comparison framing
- No streak references
- Different content from the Mindset Statement (Section 1) — the statement sets tone, the closing reinforces identity

### 4.7 Section 6 — Finish Tune-Up CTA

**Purpose**: Provide psychological closure and mark the activity as complete.

| Attribute | Specification |
|-----------|---------------|
| **Label** | "FINISH TUNE-UP" |
| **Style** | Full-width button. Calm, coach-like language. Primary button styling (amber/gold). |
| **Disabled state** | Shown at reduced opacity or muted styling when not all checkboxes are complete. Not tappable. |
| **Enabled state** | Full opacity, interactive. Appears when: Mindset Challenge (1 check) + Quick Wins (3 checks) = all 4 checkboxes checked. |
| **On tap** | Saves completion data → transitions to completion state (Section 4.8) |

**Alternative label options** (decided during development): "Lock It In", "I'm Ready". Recommend "FINISH TUNE-UP" for clarity and calm tone.

**Hard rules**:
- Only ONE CTA — no secondary buttons, no skip, no "do this later"
- Button must not be enabled until all 4 checkboxes are checked
- Minimum tap target: 48x48px (full-width button exceeds this)

### 4.8 Section 7 — Completion State

**Purpose**: Brief confirmation before returning to Home. Reinforces: "You showed up. That counts."

| Attribute | Specification |
|-----------|---------------|
| **Trigger** | Athlete taps enabled Finish CTA |
| **Visual** | Full-screen or overlay confirmation. Clean, minimal. |
| **Copy** | Brief confirmation message. 1 sentence. |
| **Transition** | Auto-return to Home screen after ~2 seconds |
| **Animation** | Subtle fade or slide transition. No celebration animations. |

**Example copy**:

> Tune-Up Complete.
> "Lead with intention today."

The confirmation message can be static (same every day) or can echo the day's Focus Word. Recommend static for simplicity.

**Behavior details**:
- The screen replaces the Tune-Up content (not an overlay on top of it)
- No CTA button — auto-return is the only exit
- If the athlete taps the back button during the ~2 second display, navigate to Home immediately
- Bottom nav remains visible and functional during the completion state

**Hard rules**:
- No streak language
- No score reveal
- No celebration animation (no confetti, no fireworks, no sound effects)
- No "You're on a X-day streak!" or similar
- Auto-return must not exceed 3 seconds

## 5. Content Model — Data Requirements

### 5.1 Pre-Generated Content Set (Per Day)

The Morning Tune-Up pulls **6 pre-generated components** from the database each day:

| # | Component | Uniqueness | Source |
|---|-----------|------------|--------|
| 1 | **Mindset Statement** | Unique per day | Coaching system / database |
| 2 | **Focus Word** | Unique per day | Coaching system / database |
| 3 | **Focus Word Quote** | Unique per day | Coaching system / database |
| 4 | **Mindset Challenge** | Unique per day | Coaching system / database |
| 5 | **Quick Wins** (3 items) | **Static** — same every day | Program configuration |
| 6 | **Closing Statement** | Unique per day | Coaching system / database |

### 5.2 Data Read on Screen Load

| Data Point | Source | Used For |
|------------|--------|----------|
| Today's Mindset Statement | Pre-generated content set | Section 1 display |
| Today's Focus Word | Pre-generated content set | Section 2 display (also shown on Home) |
| Today's Focus Word Quote | Pre-generated content set | Section 2 display |
| Today's Mindset Challenge | Pre-generated content set | Section 3 display |
| Quick Wins (3 items) | Program configuration (static) | Section 4 display |
| Today's Closing Statement | Pre-generated content set | Section 5 display |
| Morning Tune-Up completion status (today) | Activity log | Determines active vs. read-only review mode |
| Checkbox states (if partially completed) | Local state / activity log | Restores checked items on re-entry |

### 5.3 Data Written on Completion

| Data Point | Destination | Used By |
|------------|-------------|---------|
| Completion timestamp | Activity log | Home screen state, timing categorization (on-time vs. late) |
| Completion category | Activity log | "on-time" (before noon) or "late" (noon or later) |
| Mindset Challenge text | Carried forward to Evening Review | WTD question generation — becomes a Yes/No evaluation question |
| Mindset Challenge accepted = true | Activity log | Confirms athlete engaged with the challenge |

### 5.4 Data NOT Written

- No score
- No time-spent metric
- No individual checkbox timestamps
- No partial completion record (completion is all-or-nothing from the system's perspective, though UI state is preserved locally)

## 6. Interaction Specifications

### 6.1 Checkbox Behavior

All checkboxes (1 Mindset Challenge + 3 Quick Wins) follow the same interaction pattern:

| Aspect | Specification |
|--------|---------------|
| **Tap target** | Minimum 48px height. Tap area includes both the checkbox and the text label. |
| **Unchecked → Checked** | Subtle check animation (checkmark draws in or fades in, ~200ms). Text may darken slightly to indicate affirmed state. |
| **Checked → Unchecked** | Allowed. Athlete can uncheck and re-check. Instant state change (no animation on uncheck). |
| **Haptic feedback** | Optional light haptic on check (platform-dependent). |
| **Sound** | None. |
| **Visual feedback** | Checkmark only. No color burst, no confetti, no particle effects. |

### 6.2 CTA Enable/Disable Logic

```
finishButtonEnabled = (
  mindsetChallengeChecked === true
  AND quickWin1Checked === true
  AND quickWin2Checked === true
  AND quickWin3Checked === true
)
```

- The CTA transitions from disabled → enabled the moment the 4th checkbox is checked
- If any checkbox is subsequently unchecked, the CTA returns to disabled
- Transition between states should be smooth (opacity fade, ~200ms)

### 6.3 Focus Word Micro-Interaction (Optional)

| Aspect | Specification |
|--------|---------------|
| **Trigger** | Tap on the Focus Word text |
| **Effect** | Subtle pulse animation — word briefly scales up ~5% and returns. Amber glow effect optional. |
| **Purpose** | Decorative only. Draws attention back to the word. No functional effect. |
| **Frequency** | Can be triggered multiple times. No cooldown. |

This interaction is **optional for v1** — the developer may omit it without affecting functionality.

## 7. Navigation & Flow States

### 7.1 Flow State Machine

```
┌──────────────┐
│   INACTIVE   │ ← Before hard-out time OR after evening release
│  (not avail) │    (Morning Tune-Up not accessible)
└──────┬───────┘
       │ Hard-out time reached (default 6:00 AM)
       ▼
┌──────────────┐
│   AVAILABLE  │ ← Tune-Up ready but not started
│  (Home shows │    Athlete sees START on Home card
│   START CTA) │
└──────┬───────┘
       │ Athlete taps START or Dynamic Action
       ▼
┌──────────────┐
│  IN PROGRESS │ ← Athlete is in the Tune-Up flow
│  (active     │    Checkboxes available
│   session)   │    Can leave and return (state preserved)
└──────┬───────┘
       │ All 4 checks + Finish CTA tapped
       ▼
┌──────────────┐
│  COMPLETION  │ ← Brief confirmation (~2 sec)
│  (transient) │    Auto-returns to Home
└──────┬───────┘
       │ Auto-return fires
       ▼
┌──────────────┐
│  COMPLETED   │ ← Done for the day
│  (read-only  │    Home shows completed card with Focus Word
│   review)    │    Dynamic Action opens read-only review
└──────────────┘
```

### 7.2 Availability Window

| Parameter | Value |
|-----------|-------|
| **Available from** | Hard-out time (default 6:00 AM, configurable up to 10:00 AM) |
| **Available until** | Evening release time (default 7:00 PM, configurable up to 9:00 PM) |
| **On-time completion** | Before 12:00 PM (noon) local |
| **Late completion** | 12:00 PM to evening release time |
| **No backfill** | If not completed by evening release, it's missed. No way to go back. |

### 7.3 Bottom Navigation During Tune-Up

The bottom nav remains visible and functional while the athlete is in the Tune-Up flow:

| Nav Item | Behavior |
|----------|----------|
| Inbox | Navigates to Messages/Inbox. Tune-Up state preserved. |
| ☀️ Dynamic Action | No-op (already on this screen) or scrolls to top |
| Home | Navigates to Home. Tune-Up state preserved. |
| Bullseye | Navigates to Bullseye Reflection. Tune-Up state preserved. |
| Journal | Navigates to Journal. Tune-Up state preserved. |

## 8. Edge Cases & Special States

### 8.1 Loading State

| Aspect | Behavior |
|--------|----------|
| **Immediate** | Top bar renders with back arrow and title |
| **Skeleton** | Placeholder blocks for Mindset Statement, Focus Word, Challenge, Quick Wins, Closing |
| **Animation** | Subtle pulse/shimmer on skeleton elements (matches Home screen loading pattern) |
| **Content arrival** | Fade in real content, no jarring layout shift |
| **Failure** | Show retry message: "Couldn't load today's tune-up. Tap to retry." with RETRY button |

### 8.2 Mid-Flow Exit & Re-Entry

**Behavior**: Checked state is preserved. On re-entry, show a brief "Welcome back" nudge.

| Scenario | Behavior |
|----------|----------|
| **Athlete leaves via back button** | Navigates to Home. Checked items saved locally. |
| **Athlete leaves via bottom nav** | Navigates to destination. Checked items saved locally. |
| **Athlete returns same session** | Tune-Up loads with previously checked items restored. Brief "Welcome back" message shown at top (e.g., "Pick up where you left off.") — auto-dismisses after 2 seconds or on scroll. |
| **App backgrounded** | State preserved in memory. On foreground return, screen is exactly as left. |
| **App killed / force closed** | Checked state may be lost (depends on local persistence implementation). If lost, athlete re-engages from scratch. No error message — the flow simply shows unchecked. |
| **Checked state TTL** | Preserved until end of day (evening release time). After evening release, any partial state is discarded — the Tune-Up window has closed. |

**"Welcome back" nudge spec**:
- Small, non-intrusive text at the top of the content area (below the top bar)
- Copy: "Pick up where you left off." or "Welcome back."
- Auto-dismiss: fades out after 2 seconds or when athlete scrolls
- Does NOT block interaction — athlete can immediately tap checkboxes
- Only shown on re-entry, not on first visit

### 8.3 Network Error During Completion

| Scenario | Behavior |
|----------|----------|
| **Finish CTA tapped but save fails** | Show inline error below the CTA: "Couldn't save. Check your connection and try again." CTA remains enabled for retry. |
| **Retry succeeds** | Normal completion flow (completion state → auto-return). |
| **Persistent failure** | CTA stays enabled. Athlete can keep retrying or leave (checked state preserved). |

### 8.4 Content Not Yet Generated

If the pre-generated content for today hasn't been created yet (edge case — coaching system lag):

- Show the loading/error state from 8.1
- Do NOT show yesterday's content as a fallback
- Do NOT show placeholder/generic content
- The Home screen's START CTA should ideally not be shown if content isn't ready (handled by Home screen logic)

### 8.5 Multiple Completions (Same Day)

Not possible. Once completed, the screen transitions to read-only review mode. There is no "redo" or "undo completion" feature.

### 8.6 Timezone Edge Cases

- State transitions follow device local time (same as Home screen)
- If timezone change causes the athlete to re-enter the morning window after completion, the completed status holds — no re-prompting

## 9. Read-Only Review Mode (Post-Completion)

After the athlete completes the Morning Tune-Up, the screen becomes a passive reference accessible throughout the day.

### 9.1 Access Points

| Source | Trigger |
|--------|---------|
| Bottom Nav — Dynamic Action (☀️) | Tap during morning state after completion |
| Home — Completed Card | Tap the completed Morning Tune-Up card (optional — may just be non-tappable) |

### 9.2 Layout

Same layout as the active flow, with these changes:

| Element | Change |
|---------|--------|
| **Top Bar** | Same — back arrow + "Morning Tune-Up" title |
| **Mindset Statement** | Displayed as-is (read-only) |
| **Focus Word + Quote** | Displayed as-is (read-only) |
| **Mindset Challenge** | Checkbox shown as checked (filled). Not interactive. |
| **Quick Wins** | All 3 checkboxes shown as checked (filled). Not interactive. |
| **Closing Statement** | Displayed as-is (read-only) |
| **Finish CTA** | **Hidden** — replaced by a subtle completion indicator (e.g., "Completed at 7:42 AM" in muted text) |
| **Bottom nav** | Fully functional as usual |

### 9.3 Visual Treatment

- Slightly muted overall tone compared to the active flow (reduced contrast, lighter weight)
- Checkmarks are permanently filled — no interaction affordance (no checkbox border, just the checkmark icon)
- No CTA button — the completion indicator takes its place
- The Focus Word retains its prominent styling — it's the element most likely to be revisited

## 12. Hard Rules & Constraints

### 12.1 Design Philosophy Rules

| Rule | Enforcement |
|------|-------------|
| Commitments, not selections | Checkboxes are acknowledgment acts — the athlete is committing to pre-written content, not choosing from options |
| Calm, coach-led pacing | No urgency timers, no countdown, no "time remaining" indicators |
| Identity priming + ownership | Copy reinforces who the athlete IS, not what they need to DO |
| One completion outcome | No partial completion, no "save for later", no skip |
| Psychological closure | The flow ends with identity reinforcement and a calm exit |

### 12.2 Prohibited Elements

- No streak language or streak counters
- No celebration animations (confetti, fireworks, sounds)
- No score display or score language
- No progress bars or step indicators (it's a single scroll, not a wizard)
- No motivational hype copy ("Let's go!", "Crush it!", "You've got this!")
- No urgency language ("Don't miss your window!", "Time is running out!")
- No comparison to other athletes
- No guilt messaging if the athlete leaves mid-flow
- No "select all" or batch-check shortcuts
- No first-person "I Am" declaration (replaced by coach-voice Closing Statement)
- No skip button, no "do this later" button, no secondary actions
- No counter showing "2 of 4 checked" or similar progress tracking

### 12.3 Accessibility Requirements

- All text meets WCAG AA contrast ratios
- All checkboxes have accessible labels (the full text of the commitment)
- Checkbox tap targets: minimum 48px height, full width of the text row
- Focus Word text large enough to read at a glance (display/headline sizing)
- Screen reader: sections are announced in order with their labels
- Completion auto-return should be announced to screen readers ("Tune-Up complete. Returning to Home.")
