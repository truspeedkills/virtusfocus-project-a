ABI DESIGN DECISIONS TO RESOLVE
============================================================
The following decisions need to be presented, discussed, and
confirmed with the user during this session:

1. FORMULA RANGE VERIFICATION
   The pre-approved formula pillar_score = round((Q_a + Q_b) / 2 * 2)
   produces a range of 2-10 per pillar (not 0-10):
     Both 1s: round((1+1)/2 * 2) = round(2) = 2
     Both 5s: round((5+5)/2 * 2) = round(10) = 10
   Total range: 8-40 (not 0-40).

   Options:
   (A) Accept 2-10 range, keep the formula as-is (simplest)
   (B) Adjust formula to produce true 0-10:
       pillar = round(((Q_a-1) + (Q_b-1)) / 2 * 2.5)
       This maps (1,1)=0, (3,3)=5, (5,5)=10
   (C) Keep the formula, relabel documented range as 2-10

   Note: Option A means the "0-16 Needs Foundation" band can
   only be reached at total 8-16, so the band effectively
   starts at 8. This may or may not matter.

2. BAND THRESHOLD ADJUSTMENT
   With actual range 8-40 (if Option A above), should bands
   be adjusted?
   Current bands: 0-16 / 17-28 / 29-36 / 37-40
   Possible adjusted bands: 8-16 / 17-28 / 29-36 / 37-40

   The question is whether "Needs Foundation" should be 8-16
   (documenting the real minimum) or if the band definitions
   work fine as-is since 0-7 is impossible anyway.

3. PRIMARY EMPHASIS RULES
   "Primary Emphasis = lowest 1-2 dimensions." Needs precision:
   - Always identify the single lowest pillar?
   - When do you identify TWO lowest vs ONE?
   - What point-gap threshold separates 1 emphasis from 2?
   - What if all 4 pillars are equal?
   - What if 3 pillars tie for lowest?

   Options for the threshold:
   (A) Fixed gap: if the 2nd-lowest is within 2 points of the
       lowest, include both. Otherwise just the lowest.
   (B) Fixed gap: within 1 point
   (C) Always identify exactly the 1 lowest, plus any tied
   (D) Always identify the 2 lowest regardless of gap

4. ABI-GROWTH PHASE RELATIONSHIP
   How do ABI bands relate to Growth Phase at intake?

   (A) Soft input — ABI band informs the Snapshot Builder's
       Growth Phase judgment but doesn't constrain it. The
       agent can classify an athlete as Consistent even with a
       Developing ABI band if the narrative warrants it.
   (B) Floor constraint — ABI band sets a ceiling on Growth
       Phase at intake (e.g., ABI band = "Needs Foundation"
       caps Growth Phase at Developing). Weekly pipeline data
       can advance beyond this floor.
   (C) Independent — Growth Phase purely from narrative, ABI
       is a separate metric. No interaction.

   Note: The existing pipeline uses deterministic Growth Phase
   thresholds based on weekly data (VF_Interpretation_JSON_Rules
   Section 4C). ABI is a one-time intake score, not weekly. The
   Snapshot Builder assigns initial Growth Phase; Stage 2 then
   tracks it longitudinally.

5. ABI OUTPUT FORMAT
   Proposed:
   abi_scores: {
     ownership: N,
     composure: N,
     focus: N,
     structure: N,
     total: N,
     band: "...",
     primary_emphasis: ["..."] or ["...", "..."]
   }

   Questions:
   - Should coaching_strategy lines be in the ABI output or
     deferred to the Snapshot Builder?
   - Should the raw Q values (Q7-Q14) be stored alongside the
     computed pillar scores for audit trail?
   - Should individual pillar bands be computed (e.g., Ownership
     is "Low"/"Moderate"/"High") or just the total ABI band?

6. INDIVIDUAL PILLAR BANDS
   Should each pillar (0-10 or 2-10) have its own band labels
   in addition to the total ABI band?

   (A) No individual bands — only total ABI has a band
   (B) Individual bands: e.g., 2-4 Low / 5-7 Moderate / 8-10 High
   (C) Individual bands with descriptive labels per pillar

   Individual pillar bands would support: Primary Emphasis
   determination (lowest band, not just lowest score), dashboard
   visualization, and downstream coaching personalization.
