WHAT NEEDS AUDIT CRITERIA
============================================================

The existing Editorial Audit Agent (Stage 5) audits Stage 3
output (Coaching Messages + Deep Dives) using 13 criteria
across 3 tiers. Key question: does Stage 6 need equivalent
external audit, or is the built-in validation checklist
sufficient given the constrained output format?

Arguments FOR external audit:
- The 12 prohibited content rules need independent verification
  (same reason Stage 3 has Criterion 2 for clinical language —
  the generator shouldn't grade its own compliance)
- Pipeline data leakage (Criterion 13 equivalent) needs checking
- Missed day language rule violation is high-stakes (naming a
  miss is one of the worst possible athlete experiences)

Arguments FOR self-validation only:
- Output is 2-3 sentences / 30-60 words — many audit criteria
  are trivially verifiable (sentence count, word count, question
  mark presence, exclamation point presence)
- An external audit pass doubles the LLM cost per snippet
  (currently ~$0.01/snippet, would become ~$0.02)
- The validation checklist already covers the same ground
- Stage 3 needs external audit because it produces 250-450 word
  Coaching Messages and multi-section Deep Dives where subtle
  clinical language and data leakage can hide in long prose.
  A 30-60 word snippet has nowhere to hide.

DESIGN QUESTIONS TO RESOLVE:

1. Architecture: External Stage 5 audit, self-validation only,
   or hybrid (self-validation + periodic sampling audit)?

2. If external audit: Which of the existing 13 criteria apply
   as-is, which need adaptation, and which are not applicable?

3. If external audit: Same Opus model as weekly audit, or can
   Haiku handle snippet-length compliance checking?

4. If self-validation only: What quality monitoring mechanism
   catches systematic drift (e.g., the generator consistently
   producing borderline-clinical language that individually
   passes but collectively shows a pattern)?

5. Batch audit option: Could snippets be audited in weekly
   batches (7 at once) rather than individually, reducing cost
   while still catching systematic issues?

============================================================
KEY CONSTRAINTS
============================================================

1. Whatever audit mechanism is chosen, it must catch:
   - Clinical language in the snippet
   - Pipeline data leakage (execution signal terminology,
     composite score names, flag IDs, etc.)
   - Named misses (the most athlete-harmful violation)
   - Questions (the most common generation drift)
   - Surveillance framing

2. The audit mechanism must be proportional to the output size.
   A 30-60 word snippet should not cost more to audit than to
   generate.

3. If extending Stage 5, the existing 13 weekly criteria must
   remain completely unchanged. Daily criteria are additive.

4. The Daily Coach Insights Engine (Stage 7, planned for Tasks
   12-15) will also produce daily output. Whatever audit
   architecture is chosen here should be extensible to Stage 7.
