=========================================================
THE PROBLEM
=========================================================

An audit of all 33 Coaching Messages revealed two distinct format
populations:

GROUP A (FORMATTED — 22 messages):
  Grace Kindel (6 weeks), Mergim Bushati (19 weeks),
  Aidan Burkardt (5 weeks partially — need to verify all)

  Format characteristics:
  - "VIRTUSFOCUS — ATHLETE COACHING MESSAGE" header
  - Metadata line: athlete name, week number, date range
  - Schema/EI/Tier metadata line
  - ━━━ separator bars (full-width, ~73 characters)
  - Body with salutation ("Grace —")
  - Labeled "YOUR ACTION — [CONTEXT]" section with separators
  - Closing phrase + "— Coach Arron" signature
  - Labeled "NEXT WEEK FOCUS QUESTION" section with separators

GROUP B (UNFORMATTED — 11 messages):
  Tucker Lloyd (5 weeks), Angelo Allen (4 weeks),
  Jasmine Jones (7 weeks partially — need to verify all)

  Format characteristics:
  - NO header, NO metadata lines
  - Starts directly with salutation ("Tucker," or "Angelo —")
  - Body narrative (same quality content)
  - Action embedded in narrative ("Here is your action:" inline)
  - NO labeled "YOUR ACTION" section
  - Closing phrase + "— Coach Arron" (sometimes closing phrase absent)
  - NO "NEXT WEEK FOCUS QUESTION" section at all
  - NO separator bars anywhere

ROOT CAUSE:
The Coaching Message Framework spec (VF_Coaching_Message_Framework.txt)
defines the 5 content sections but says NOTHING about visual format —
no header, no separators, no "YOUR ACTION" label, no "NEXT WEEK FOCUS
QUESTION" section. The Master Prompt (V4) also does not specify output
format for the athlete-facing CM. The formatted version emerged
organically during early Grace Kindel/Mergim processing sessions and
was never codified. Later athletes (Tucker, Angelo, Jasmine) were
processed by parallel agents in fresh sessions that did not inherit
the format convention.

The Parent CM format IS defined (in Instructions v1.8 Section 23+)
with explicit header, sections, and separator specification. The
athlete-facing CM has no equivalent.

After reading all files, present the user with:

A. A PROPOSED FORMAT SPECIFICATION for the athlete-facing Coaching
   Message. This should cover:

   1. HEADER — Should the CM have a header? If so, what lines?
      Consider: the header contains pipeline metadata (schema version,
      EI classification, weekly tier). This is useful for internal
      pipeline tracking but may not be appropriate for athlete-facing
      output in the app. Present options:
        Option A: Full header (current Group A style — header + metadata + separator)
        Option B: Clean header (athlete name + week + date only, no pipeline metadata)
        Option C: No header (starts with salutation, like Group B)

   2. SEPARATORS — Should the CM use visual separator bars?
      Consider: the Deep Dive uses dashes (----) between sections.
      The Parent CM uses ━━━ separators. What should the athlete CM use?
        Option A: Full-width ━━━ bars (current Group A style)
        Option B: Dashes (---- style, matching Deep Dive)
        Option C: No separators (clean narrative flow, like Group B)

   3. "YOUR ACTION" SECTION — Should this be a labeled section?
      Consider: making the action visually distinct helps scannability
      in the app. The framework spec calls it "ONE Action" (Section 5).
        Option A: Labeled section with context ("YOUR ACTION — THIS WEEK")
        Option B: Labeled section without context ("YOUR ACTION")
        Option C: Inline with narrative ("Here is your action:")

   4. "NEXT WEEK FOCUS QUESTION" — Should this section exist?
      Consider: this section does not appear in the framework spec at
      all. Group A includes it; Group B omits it entirely. The JSON
      contains a next_week_focus_question field. If included, it adds
      a reflective prompt for the athlete. If excluded, it simplifies
      the message.
        Option A: Keep as labeled section after signature
        Option B: Integrate into the action section or body
        Option C: Remove entirely (the question stays in the JSON for
                  Stage 4/Stage 6 use but doesn't appear in the CM)

   5. SALUTATION FORMAT — Minor but inconsistent:
        "Grace —" (em dash) vs "Tucker," (comma) vs "Angelo —"
      Lock one format.

   6. CLOSING PHRASE — Should a closing phrase be mandatory?
      Group A always has one. Group B sometimes omits it (Angelo, Jasmine).
      Framework spec provides examples but doesn't say "mandatory."

   7. SIGNATURE PLACEMENT — Where does "— Coach Arron" go relative
      to the action section and focus question?
