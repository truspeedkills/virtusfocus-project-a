============================================================
WHAT "EXECUTION SIGNAL LEAKAGE" MEANS
============================================================

Leakage is any appearance of execution_behavior_signals raw data in athlete-facing (Stage 3) or coach-facing (Stage 4) output. This includes:

STAGE 3 LEAKAGE (Coaching Message + Deep Dive — athlete-facing):
1. Raw metrics cited: "You completed 5 out of 7 Morning Tune-Ups"
2. Completion rates: "Your evening review completion rate was 0.71"
3. Timing data: "3 on-time, 2 late submissions"
4. Streak counts: "You're on a 14-day streak"
5. Question-level rates: "Your adversity question yes rate dropped"
6. Composite score names: "Your Ownership Index shows..."
7. Composite score values: "scored 73/100"
8. Band labels as labels: "entered the Active band"
9. Coach flag references: "The drift_active flag..."
10. Severity tier names: "attention-level concern"
11. Self-ratings labels: "Your confidence alignment is Conflated"
12. Surveillance framing: "Your engagement metrics show..." / "Based on your app usage data..."

STAGE 4 LEAKAGE (Weekly Performance Insight + Team Snapshot — coach-facing):
All of the above, PLUS:
13. Score names in institutional context: "Rhythm Score band was Disrupted"
14. Flag IDs in institutional context: "compound_disengagement fired"
15. Internal mechanics exposed: "shifted from Building to Stable due to Conflated alignment"
16. Modifier sources named: "Readiness shifted due to action-severity coach flag"
17. Band percentages on team snapshots: "55% of athletes had Disrupted Rhythm Score bands"

WHAT IS NOT LEAKAGE (permitted):
- Composite score BAND values translated into behavioral language (the entire point of the translation layer)
- Coach flag PATTERNS translated into compliance-safe KPS labels
- Self-ratings EFFECT on trend direction (without naming the classification)
- Execution pattern summary LANGUAGE translated into neutral observations
- "Your consistency was volatile" (behavioral observation) vs. "Your Rhythm Score was Disrupted" (leakage)

DESIGN CONSTRAINTS
============================================================

1. THE AUDIT AGENT AUDITS STAGE 3 OUTPUT ONLY (NOT STAGE 4)
   The Editorial Audit Agent (Stage 5) currently audits Coaching Messages and Deep Dives produced by Stage 3. It does NOT audit Stage 4 (Coach Insights) output. The v1.0 specification is clear: inputs are Interpretation JSON + Coaching Message + Deep Dive.

   DESIGN DECISION REQUIRED: Should new execution signal leakage criteria ALSO cover Stage 4 output? Options:
   A) Stage 3 only (current scope) — add criteria for execution leakage in Coaching Messages and Deep Dives. Stage 4 compliance is self-enforced by v1.2 rules.
   B) Expand to include Stage 4 — add Stage 4 output (Weekly Performance Insight) as a fourth input document for execution signal leakage checks only. This would be the first expansion of the audit agent's scope.
   C) Create a separate Stage 4 audit — a new audit specification specifically for Coach Insights output, rather than expanding Stage 5.

   SURFACE THIS DECISION TO THE USER BEFORE PROCEEDING. Do not choose unilaterally.

2. TIER PLACEMENT MATTERS
   - Tier 1 (must pass, reject on failure) = compliance violations that cannot be fixed by editing
   - Tier 2 (should pass, auto-correct if possible) = content issues fixable by editing
   - Tier 3 (quality optimization, auto-correct silently) = polish issues

   Execution signal leakage is a COMPLIANCE violation (same classification as clinical language). Raw execution data surfacing is not a style issue — it is a business rule violation. Consider whether this warrants Tier 1 placement.

3. RELATIONSHIP TO EXISTING CRITERION 2 (CLINICAL LANGUAGE COMPLIANCE)
   Criterion 2 already catches clinical language. Execution signal leakage is a DIFFERENT compliance category — it's about surveillance/engagement data, not clinical terminology. The new criteria must be distinct from Criterion 2, not duplicative.

4. CORE FOUNDATION AWARENESS
   The audit must know that Core Foundation athletes should have NO execution signal language in their output at all. If a Core Foundation athlete's Coaching Message references "daily engagement patterns" or "behavioral execution consistency," that is leakage — because there is no execution data to derive those observations from.

5. ACCURACY STANDARD
   Every rule in this pipeline is written for national app launch. Do not add phantom prohibitions (rules against things that cannot happen given the data flow). Do not add redundant criteria. Every criterion must address a real, possible failure mode. If a leakage vector is already covered by an existing criterion, note that rather than creating a duplicate.

6. VERSIONING RULES
   - The Master Prompt and Project Instructions are VERSIONED files. If structural changes are needed, create NEW versioned files (v1.1). Preserve v1.0 on disk.
   - If the changes are additive (new criteria sections added without restructuring existing content), editing v1.0 in place is acceptable IF the changes are clearly marked with version annotations.
   - Surface this decision to the user: new version vs. in-place edit.
