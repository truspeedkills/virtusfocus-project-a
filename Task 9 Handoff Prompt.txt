2. NEW SECTIONS TO ADD:

   a. EXECUTION SIGNAL INTEGRATION RULE — The master rule for how execution signals are used. Key principles:
      - Execution signal sharpens existing sections; it does not add new sections or expand word targets
      - Only composite score BANDS, coach flag-driven behavioral patterns, self-ratings alignment classifications, and execution_pattern_summary language may inform output
      - Raw execution metrics (7 prohibited categories) NEVER appear
      - Core Foundation athletes: all execution integration skipped, output identical to v1.4 behavior
      - When execution signals and v9.3 fields converge on the same pattern, strengthen the coaching point (don't double-count). When they diverge, note the divergence as a coaching insight.

   b. COMPOSITE SCORE TONE CALIBRATION RULE — How each score's band value translates into coaching language. This is the HOW companion to the Message Map's WHERE. Key content:
      - For each of the 7 scores: what the band means in coaching terms (not technical terms), example language for positive bands, example language for concern bands
      - Score directionality awareness: higher-is-better (Ownership, Follow-Through, Rhythm, Review Quality, Recovery) vs higher-is-worse (Drift, Reactivity Risk)
      - Language transformation principle: translate band classifications into behavioral observations the athlete can recognize. "High Ownership" becomes "You're showing up on your own terms." "Active Drift" becomes "The way you've been engaging shifted this week."
      - NEVER reference the score name, numeric value, or band label in coaching output. Only the behavioral translation.

   c. COACH FLAG INTEGRATION RULE — How each severity tier changes the coaching approach. Key content:
      - monitor: Tone adjustment only. The coaching message acknowledges the pattern indirectly. Examples of subtle integration vs. examples of what NOT to do (naming the flag, citing metrics).
      - attention: Direct behavioral address. Name the pattern in athlete-appropriate language. The micro_commitment targets the flagged area. Priority rule when multiple attention flags fire (prioritize by relevance to core_pattern).
      - action: Primary direction shift. The coaching message leads with the flagged pattern. Recognition section pivots. Drift Identification becomes the central section. Coaching Focus Areas dominated. If compound_disengagement fires, address the broad shift ("the way you've been showing up changed"), not each individual score.
      - Flag-to-coaching-language translation examples for the most common flags (drift_active, ownership_low, follow_through_weak, compound_disengagement).
      - Interaction with emotional_intensity: when EI = High AND action flags fire, the empathetic acknowledgment lead (from Emotional Calibration Rule) takes priority over the flag-driven direction shift. Weave the flag pattern into the empathetic frame rather than leading with the flag.

   d. SELF-RATINGS ALIGNMENT INTEGRATION RULE — How Conflated/Undervalued/Aligned classifications become coaching language. Key content:
      - Aligned: no special integration needed.
      - Conflated: gently ground self-perception. Coaching language must NOT use "overestimate," "inflated," "unrealistic," or any language that invalidates the athlete's self-view. Instead use grounding language: "Let's look at what the week actually showed," "Your sense of where you are doesn't quite match what the execution shows." Place in Drift Identification (Coaching Message) and Mindset Summary (Deep Dive).
      - Undervalued: elevate with execution evidence. Language: "You're further along than you think," "The execution tells a stronger story than you gave yourself credit for." Place in Recognition + Identity Anchor (Coaching Message) and Mindset Summary (Deep Dive).
      - Distinction between confidence_alignment and habit_consistency_alignment — these are two separate self-ratings comparisons. An athlete can be Conflated on confidence but Aligned on habit consistency (or vice versa). Address each independently if both are non-Aligned.

   e. CORE FOUNDATION EXECUTION SKIP RULE — When execution_behavior_signals.input_source = "core_foundation", skip ALL execution signal integration. Output uses v9.3 fields only. Identical to v1.4 behavior. No execution references of any kind.

   f. RAW METRIC EXCLUSION RULE (COMPLIANCE) — Explicit prohibition with examples. List the 7 prohibited metric categories. Provide examples of WRONG output (citing completion rates, naming streak counts) and CORRECT output (behavioral observation language). This is a compliance rule, not a style preference — same framing as the clinical language prohibition in Section 7.
