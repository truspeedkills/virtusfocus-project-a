WHAT THIS SESSION COVERS
============================================================

Idea 2 introduces the FIRST DAILY OUTPUT in the VirtusFocus
pipeline. The entire pipeline currently operates on a weekly
batch cycle. A daily coaching snippet adds a new cadence,
new data consumption patterns, and new architectural
questions that don't exist in the current system.

The Daily Coaching Snippet is a short, personalized coaching
message delivered to the athlete daily — separate from and
complementary to the Morning Mindset Tune-Up.

Ideas 2 and 3 (Daily Coaching Snippet + Daily Coach Insights)
are architecturally coupled: both introduce daily processing
and should share daily data consumption infrastructure.
They can be designed in sequence but architectural decisions
in Idea 2 will constrain Idea 3.

KEY ARCHITECTURAL QUESTIONS TO ANALYZE
============================================================

These are the open questions that need collaborative analysis
before formal design begins. Present your understanding of
each and discuss with the user.

1. DAILY DATA SOURCE — WHAT DOES THE SNIPPET CONSUME?

   The weekly pipeline operates on interpreted data: Stage 2
   produces a comprehensive JSON, and Stages 3-5 consume it.
   A daily snippet can't wait for the weekly JSON — it needs
   to operate on whatever data exists TODAY.

   Options to analyze:
   a. Yesterday's raw app data only (Evening Review scores,
      journal entries, Bullseye, WTD answers)
   b. Yesterday's raw data + athlete baseline snapshot
   c. Yesterday's raw data + baseline + last weekly JSON
      (carries forward micro_commitment, growth_phase, etc.)
   d. A lightweight "daily interpretation" that produces a
      mini-JSON each day

   Tension: Option (a) has minimal context. Option (d) adds
   a new processing stage. Options (b) and (c) use existing
   artifacts but skip the interpretation layer's deterministic
   classifications.

2. RELATIONSHIP TO MORNING TUNE-UP

   The Morning Mindset Tune-Up is delivered each morning and
   includes: mindset challenge, focus word, quick win items.
   The daily coaching snippet would also be delivered daily.

   Questions:
   - Is the snippet delivered AT the same time as Tune-Up?
     Before? After? Evening?
   - Does the snippet reference the Tune-Up content?
     ("Your focus word today is [X] — here's how to use it
     when [situation from baseline_primary_derailer]")
   - Or is the snippet based on YESTERDAY's data, making it
     a reflective message rather than a forward-looking one?
   - Could the snippet BE the Tune-Up (replacing the current
     static format with a personalized daily coaching message)?
   - Risk of message fatigue: athlete gets Tune-Up + snippet
     + weekly coaching message. Too much?

3. PIPELINE STAGE PLACEMENT

   Where does the daily snippet live architecturally?

   Options:
   a. Sub-process of Stage 3 (Coaching Output Engine) — same
      persona (Coach Arron), same voice, lighter format
   b. New Stage 3B — separate from weekly Stage 3 but sharing
      source files (brand voice, style guide, message framework)
   c. Independent daily engine — fully separate specification
   d. Part of the app layer — not a pipeline stage at all,
      generated by app-side logic with pipeline data as context

   Tension: Option (a) keeps things simple but the Coaching
   Output Engine is designed for weekly data. Option (b) adds
   a new stage. Option (d) moves coaching generation outside
   the pipeline's quality control boundary.

4. COACHING QUALITY WITHOUT INTERPRETATION

   The weekly coaching message benefits from the full
   interpretation layer: composite scores, trend analysis,
   emotional intensity classification, narrative arc, coach
   flags, consistency signals. A daily snippet operating on
   raw data doesn't have ANY of this context.

   Questions:
   - Can a 2-4 sentence coaching snippet be meaningful without
     interpretation?
   - Does it need to be "coaching" at all, or is it more like
     a personalized reinforcement/reminder?
   - What is the minimum interpretation context needed for a
     coaching-quality daily message?
   - Does the daily snippet need access to execution behavior
     signals? (Those are computed weekly, not daily)

5. COMPLIANCE AND AUDIT

   The weekly pipeline has a full 5-stage compliance chain
   including Editorial Audit with 13 criteria. Daily output
   at scale (7 messages/week × N athletes) creates a much
   higher volume of content to quality-check.

   Questions:
   - Does each daily snippet need Stage 5 audit? (Cost and
     latency implications at scale)
   - Or is compliance handled by constrained templates +
     lighter audit rules?
   - Clinical language prohibition still applies — how is
     this enforced at daily cadence?
   - Competitive Level Calibration — does it apply daily?

6. IDEA 2 ↔ IDEA 3 COUPLING

   Both ideas introduce daily processing. Design decisions
   here constrain Idea 3 (Daily Coach Insights):

   - If Idea 2 creates a "daily interpretation" step, Idea 3
     can consume it too
   - If Idea 2 operates on raw data only, Idea 3 faces the
     same "no interpretation" problem
   - The daily data consumption path should be designed once
     and shared

7. CONTENT DIFFERENTIATION

   The athlete currently receives (or will receive):
   - Morning Tune-Up (daily, forward-looking)
   - Weekly Coaching Message (weekly, reflective + directive)
   - Weekly Deep Dive (weekly, analytical)
   - [NEW] Daily Coaching Snippet

   Each must occupy a distinct content niche. If the snippet
   overlaps with Tune-Up or duplicates the weekly message in
   shorter form, it adds noise, not value.

8. SCALE AND TIMING

   At app launch with potentially hundreds of athletes:
   - Daily snippet generation must be fast (morning delivery)
   - Each snippet must be personalized (not generic)
   - Generation failures can't block the athlete's morning
   - How does this interact with the hard-out time model
     from the App Input Format Specification?
