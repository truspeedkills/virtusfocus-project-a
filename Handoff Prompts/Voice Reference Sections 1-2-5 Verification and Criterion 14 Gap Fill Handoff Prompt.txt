6. PHASE 2 EXECUTION: CRITERION 14 ALIGNMENT AUDIT
============================================================

Read SOP_Editorial_Audit_Project_Instructions_v1.6.txt
Section 14 in full. It contains 12 fail conditions:

  14.1  Reusable Opening Template
  14.2  Telegraphing
  14.3  Untraceable Paragraph
  14.4  Category Coverage Over Response
  14.5  Brand Copy Close
  14.6  Same Construction 3-Week Window
  14.7  DD as Expanded CM
  14.8  "That is [Noun]." Standalone Overuse
  14.9  "Version of [X]" Construction
  14.10 Triple-Negative Setup
  14.11 Trendline Arc Recitation
  14.12 Pipeline-Adjacent Vocabulary

For each fail condition, find its teaching home in the
Voice Reference. A teaching home can be:

  TYPE A: Section 3 positive exemplar
    The principle that prevents this failure is taught
    positively through an exemplar that demonstrates the
    correct instinct. Acceptable home if the exemplar
    explicitly addresses the failure mode.

  TYPE B: Section 4 before/after pair
    The failure is taught directly through a BEFORE
    demonstrating the failure and an AFTER demonstrating
    the fix. Acceptable home with no caveat.

  TYPE C: Section 5 binary rule
    The failure is a mechanical rule (em dash ban,
    specific term prohibition). Acceptable home only for
    failures that reduce to binary mechanical checks.

  TYPE D: NO HOME — GAP
    The failure condition has no clear teaching location
    in the Voice Reference. The model reading the Voice
    Reference will not learn to avoid this failure except
    through the audit catching it after the fact.

Produce a table with one row per fail condition:

  Condition | Home Type | Location | Adequacy

"Adequacy" rating:
  STRONG — the teaching is concrete, uses a specific
    example, and explicitly addresses the failure mode.
  ADEQUATE — the teaching exists but is abstract or
    indirect.
  WEAK — the teaching exists but is mentioned only in
    passing without demonstrating the fix.
  GAP — no teaching home.

Any condition rated GAP or WEAK is a candidate for a new
Section 4 pair in Phase 3.

Present the audit table to the user before beginning
Phase 3. User approves which conditions get new pairs.

7. PHASE 3 EXECUTION: NEW SECTION 4 PAIRS (FABRICATED DATA)
============================================================

For each approved gap, design one new Section 4 pair.

PAIR STRUCTURE (consistent with existing Section 4):

  PAIR [N]: [Short descriptive name of the AI pattern]
  ------------------------------------------------------------

  BEFORE (Fabricated — [Athlete Name] Wk[N] [CM or DD]):
  "[Passage demonstrating the AI pattern cleanly]"

  THE PROBLEM:
  [1-3 sentence diagnostic explanation]

  AFTER:
  "[Passage demonstrating the corrective cleanly]"

  WHAT CHANGED:
  [Second-pass quality block: instinct-first opening,
  explicit Voice Principle reference, concrete mechanics,
  variable length]

FABRICATED DATA DESIGN CHECKLIST per pair:

  1. Athlete profile:
     - First name + last name (no real athlete names).
     - Sport (vary across pairs — basketball, wrestling,
       swimming, volleyball, golf, gymnastics, etc.).
     - Position/event.
     - Competitive level (vary — HS, College D1, College
       D2/D3, Pro, Masters).
     - Week number (specific, not "week N").
     - Week tier (Growth/Mixed/Reset).
     - Emotional intensity (Low/Moderate/High).
     - Growth phase (Developing/Consistent/Leadership).

  2. Situational details specific enough to feel real:
     - Competition or practice event.
     - WTD day counts (e.g., "4W/2N/1M").
     - Bullseye ring items (actual item names, not
       placeholders).
     - Journal quote if the pair needs one.
     - Prior week context if relevant.

  3. Targeted failure mode:
     - The BEFORE must cleanly demonstrate exactly one
       Criterion 14 failure mode. Not two. Not a
       combination. One.
     - No contamination from other AI patterns.

  4. Targeted fix:
     - The AFTER must cleanly demonstrate exactly one
       Voice Principle applied as the corrective.
     - The connection between principle and fix must be
       teachable in the WHAT CHANGED block.

  5. Sampling balance:
     - Across all new pairs, target diverse athlete
       profiles. If the current Section 4 is Grace-heavy
       (6 of 8 pairs), new pairs should avoid anything
       resembling Grace's situation.
     - Vary emotional intensity, growth phase, week tier,
       sport, competitive level.

  6. Zero cross-references to real athletes:
     - Do not invent a name that sounds like any of the
       6 existing athletes (Grace Kindel, Tucker Lloyd,
       Mergim Bushati, Aidan Burkardt, Angelo Allen,
       Jasmine Jones, John Tastinger).
     - Do not use real school names (Seton Hill, Pitt,
       etc.) or real team names.
     - Use plausible but generic names.

PAIR NUMBERING:
  Existing Section 4 has Pairs 1-8. New pairs start at
  Pair 9. If the audit identifies 3 gaps, you add Pair 9,
  Pair 10, Pair 11.

PRESENT THE FULL PAIR DESIGN TO THE USER BEFORE WRITING
PROSE. Include fabricated athlete profile, targeted
failure mode, targeted principle, and a brief description
of the BEFORE/AFTER strategy. User approves the design
before you write the WHAT CHANGED blocks.

Once design is approved, write the full pair (BEFORE,
THE PROBLEM, AFTER, WHAT CHANGED) and present the batch
for review. Apply on approval.
