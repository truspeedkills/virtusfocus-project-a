KEY DESIGN QUESTIONS TO RESOLVE
============================================================

1. STRUCTURED DATA vs NARRATIVE vs BOTH?
   The current snapshot is primarily narrative (9 sections of
   prose text). PPD and ABI are structured data objects (JSON-
   like). Should the expanded snapshot:
   (A) Keep everything narrative — PPD and ABI are woven into
       prose sections only (human-readable, loses structured
       data fidelity)
   (B) Add a structured data block alongside narrative — PPD
       and ABI are stored as structured objects, plus
       narrative sections reference them (both machine-readable
       and human-readable)
   (C) Hybrid — some fields stored as structured data (PPD
       scores, ABI scores), others woven into narrative
       (coaching emphasis, identity friction)

   CONTEXT: Stage 2 (Interpretation Engine) needs to READ
   the snapshot fields. Currently it reads the 4 baseline
   anchors as structured fields from the snapshot text file.
   PPD and ABI are data objects — storing them as narrative
   only would require Stage 2 to parse prose, which goes
   against the pipeline's deterministic architecture.

2. SECTION COUNT AND STRUCTURE
   Current: 9 sections in 300-500 words.
   The App intake form produces significantly more structured
   data. How should the section structure expand?
   Consider:
   - Word count may need to increase (300-500 was designed for
     the old CF form with limited structured data)
   - New sections could be added (PPD Summary, ABI Summary,
     Ecosystem Profile)
   - Existing sections could be enriched (Section 7 Stress &
     Trigger Profile now has structured adversity data)
   - Structured data blocks don't count toward narrative word
     count

3. BASELINE ANCHOR DERIVATION
   The 4 locked baseline fields currently come from agent
   synthesis of free-text intake answers. With the App form,
   they have much richer inputs:
   - baseline_identity_claim: Q17 + Q19 (was: agent-inferred
     from free-text goals/struggles)
   - baseline_hinge_habit: Q7-Q14 + Q20 + ABI (was: agent-
     inferred from narrative)
   - baseline_primary_derailer: PPD #1 + Q15 + Q16 + Q18
     (was: agent-inferred from struggles/thoughts)
   - baseline_growth_phase: Full profile + ABI soft input
     (was: agent-inferred from narrative)

   Should the derivation rules be deterministic or agent-
   synthesized? The PPD and ABI are deterministic, but the
   baseline anchors are narrative fields that require
   synthesis (e.g., identity_claim is a sentence, not a
   score).

4. ECOSYSTEM SECTION
   4 ecosystem questions (Q22-Q25) produce structured data
   about the family/support environment. This data feeds
   Idea 4 (Parent Output + Dashboard, Tasks 16-21). Should
   this be:
   (A) A new dedicated section (Section 10: Ecosystem Profile)
   (B) Woven into existing Section 7 (Stress & Trigger
       Profile) since ecosystem is a stress/support context
   (C) Both: structured data block for pipeline consumption +
       narrative integration in Section 7

5. SNAPSHOT FORMAT STANDARDIZATION
   Grace's snapshot uses structured field labels (e.g.,
   CONFIDENCE_TYPE:, ANCHOR_IDENTITY_CLAIM:) while Tucker's
   and Mergim's use more narrative prose. The expanded
   snapshot should standardize. Which format?
   (A) Grace-style structured labels (more machine-parseable)
   (B) Prose narrative (more human-readable)
   (C) Structured header/data blocks + narrative sections
       (machine-parseable data + human-readable context)
