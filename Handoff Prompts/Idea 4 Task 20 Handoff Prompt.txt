DESIGN QUESTIONS TO RESOLVE DURING SPECIFICATION
============================================================

DQ6. Stage 4 Parent Output Format
  Stage 4 currently produces a single Weekly Performance
  Insight document. For parent mode, should it produce:
  (a) A separate parent document alongside the coach
      document — two outputs per Stage 4 run
  (b) A single document that is EITHER coach-facing OR
      parent-facing depending on the audience (D2C athletes
      have no institutional coach, so the coach version may
      not be needed)
  (c) Both: coach version for future institutional use,
      parent version for D2C parent app

  Context: For D2C Premium athletes, there may be no
  institutional coach consuming Stage 4 output. The Parent
  App gives Stage 4 output its first consumer. However,
  designing for coach-only vs parent-only vs both affects
  how the Stage 4 prompt is structured.

DQ7. Parent Insights Length
  The coach-facing Weekly Performance Insight has no formal
  word target — it's structured by section. Should the
  parent version:
  (a) Same length — 10 sections with parent labels and
      adjusted framing
  (b) Condensed — fewer sections or shorter sections. The
      parent doesn't need the same analytical depth.
  (c) Selective — only include sections that are meaningful
      for parent context (e.g., drop Section 3 Self-Assessment
      if it doesn't serve the parent)

DQ8. Section 7 (How You Can Support) Relationship to
     Parent CM Section 4
  Both the Parent CM "How You Can Support" and the Parent
  Insights "How You Can Support" (rebranded from Coach
  Context Cue) serve similar functions. How should they
  relate?
  (a) Different depth: Parent CM gives one quick suggestion
      (35-65 words). Parent Insights gives the analytical
      context behind the suggestion (full section depth).
  (b) Complementary: Parent CM gives the "what to do."
      Parent Insights gives the "why this matters."
  (c) Independent: Each generates its own support content
      from the same JSON, may overlap.

DQ9. Parent Insights Document Delivery
  How does the parent access the Parent Insights?
  (a) Dashboard sections — each insight section appears as
      a card/section on the app dashboard (no document)
  (b) Document — a full Parent Insights document (like the
      coach Weekly Performance Insight) accessible via the
      app
  (c) Both — dashboard shows summary cards, tapping into
      any card shows the full section text

DQ10. Stage 7 Parent Signal Specification Location
  Where should the parent label mapping be documented?
  (a) Addendum to VF_Daily_Coach_Signal_Determination_Rules
      _v1.0.txt — keeps all signal definitions in one place
  (b) New document: VF_Parent_Signal_Label_Mapping.txt —
      clean separation
  (c) Embedded in the Stage 4 parent instructions — since
      the dashboard specification lives there

IMPORTANT CONTEXT
============================================================

STAGE 4 D2C CONTEXT:
For D2C Premium athletes, Stage 4 currently has NO consumer.
There are no institutional coaches for D2C athletes. The
Parent App is what gives Stage 4 output a purpose in the
D2C ecosystem. This means the parent-rebranded output may
be the ONLY Stage 4 output generated for D2C athletes
(no coach version needed if no coach exists).

However, Coach Arron (the VirtusFocus coach) may want to
review the coach-facing version internally for quality
assurance. Consider whether the coach version should still
be generated as an internal tool even when no institutional
coach consumes it.

STAGE 7 IS NOT AN LLM:
Stage 7 is a deterministic rules engine computed by the app
backend. The parent signal adaptation is a LABEL MAPPING
at the presentation layer — the app renders "Good Day"
instead of "Green." No new computation, no new rules, no
AI cost. The specification documents what the app needs to
display.

DASHBOARD IS NOT AN LLM:
The dashboard is a presentation layer. It renders data from
Stage 4 and Stage 7. The specification documents what data
sources feed each dashboard element, what format each
element displays in, and what warm placeholders appear in
early weeks.

PARENT INSIGHTS vs PARENT CM:
These are different outputs at different depths:
- Parent CM (Stage 3, Task 19 COMPLETE): Quick read. 150-250
  words, 4 sections. Delivered weekly as a push notification
  or message. The "here's what happened and how to help"
  summary.
- Parent Insights (Stage 4, this task): Deep read. Full
  analytical dashboard. 10 rebranded sections. Accessible
  anytime via app or browser. The "detailed view for parents
  who want to understand more."

The Parent CM and Parent Insights should complement each
other — not duplicate. The CM gives the action; the Insights
give the context.

COMPLIANCE CONSISTENCY:
All parent output uses the same compliance framework as
coach-facing output. The v1.7 Instructions (Task 19)
established the parent compliance rules in Section 29.
The Stage 4 parent expansion should reference the same
compliance framework (VF_Coach_Insights_Compliance_
Framework.txt) and apply identical prohibited categories.
