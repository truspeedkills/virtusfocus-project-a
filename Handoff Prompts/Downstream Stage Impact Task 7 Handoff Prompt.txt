============================================================
SPECIFIC DESIGN QUESTIONS TO RESOLVE
============================================================

These are the open design questions that Task 7 must answer:

Q1: COMPETITIVE LEVEL CALIBRATION — RULE STRUCTURE
Should the Competitive Level Calibration Rule be:
(a) A single lookup table (level → tone parameters), or
(b) A multi-section rule with per-level subsections and examples, or
(c) A modifier applied to existing rules (e.g., emotional_intensity + competitive_level = adjusted tone)?
Consider: Option (c) creates the most nuanced calibration (a High EI + Youth athlete vs a High EI + Professional athlete get very different coaching) but is the most complex to specify.

Q2: PPD COACHING IMPLICATIONS — ROUTING PRIORITY
When ppd_top_3.coaching_implication says "Reset reps + next-play language" but the current week's data suggests a different coaching direction (e.g., the athlete had a breakthrough week with no derailer activation), which takes priority?
Consider: PPD coaching implications are BASELINE recommendations. Weekly data may override them when the athlete's current state doesn't match the baseline pattern.

Q3: CRITERION 13 EXPANSION VS NEW CRITERION 14
Should intake profile leakage be folded into the existing Criterion 13 (Execution Signal Leakage Compliance) by expanding it to "Pipeline Data Leakage Compliance"? Or should it be a separate Criterion 14?
Consider: Criterion 13 already has 6-step check logic, 7 subcategories (E1-E7), and 12 WRONG/CORRECT examples specifically for execution signals. Adding PPD/ABI/intake terminology to the same criterion could make it unwieldy. But having two near-identical "leakage" criteria creates audit redundancy.

Q4: LIVE SEASON PHASE — CF ATHLETE HANDLING
Core Foundation athletes don't have the weekly check-in app. Their weekly input is a PDF recap. Options:
(a) CF athletes get current_season_phase = "not available" (same pattern as all other app-only fields)
(b) CF athletes get current_season_phase inferred from their recap content (if they mention "game today" = likely In-Season)
(c) CF athletes get current_season_phase set by the coach/admin as a program-level setting
Consider: Option (a) is simplest and most consistent. Option (b) adds classification complexity for a transitional population. Option (c) adds a non-athlete data source.

Q5: ADVERSITY RESPONSE PATTERN — COACHING LANGUAGE
The adversity_response_pattern enum values are descriptive ("Push harder," "Freeze and overthink"). Should Stage 3 use these as INTERNAL routing signals only (never in coaching text), or can the natural-language descriptions be reflected in coaching?
Example: "When things get hard, your instinct is to freeze and overthink — but this week you broke that pattern" — is "freeze and overthink" acceptable coaching language or a leakage violation?
Consider: Unlike PPD bucket names (which are pipeline jargon like "Mistake Recovery Lag"), the adversity response pattern descriptions are athlete-friendly language from Q20 of the intake form. The athlete selected this option themselves.
