WHAT HAS ALREADY BEEN DECIDED (DO NOT RE-OPEN)
============================================================

These decisions were made collaboratively and confirmed by the
user in the prior session. They are constraints, not options.

1. UNIFIED INTAKE FORM — One question set. The athlete answers
   once. The system derives three outputs from the same answers:
   (a) Athlete Snapshot (narrative baseline)
   (b) PPD Top 3 (deterministic problem ranking)
   (c) ABI 4-Pillar Scores (quantitative developmental baseline)
   No separate question sets. Shared inputs routed to multiple
   systems.

2. FORM LENGTH STRATEGY — Optimize for maximum signal per minute
   of athlete time, NOT minimum question count. Many structured
   questions (1-5 scales, multi-select) + few strategic free-text
   questions. Total 20-25 questions acceptable. Structured formats
   keep completion time fast despite higher question count.

3. PPD AS INTAKE + LONGITUDINAL — Self-report PPD computed at
   intake. Longitudinal PPD from execution data is a FUTURE Stage 2
   enhancement (out of scope for Task 1). BUT: the PPD output
   format must be designed NOW to support future comparison.

4. DUAL-USE ARCHITECTURE — All improvements are App-athlete only.
   Core Foundation athletes keep existing intake + baseline. The
   Snapshot Builder needs dual-mode detection. Same additive
   pattern as the Execution Signal upgrade.

5. 4 LOCKED BASELINE FIELDS — These exist and do not change:
   - baseline_identity_claim
   - baseline_hinge_habit
   - baseline_primary_derailer
   - baseline_growth_phase
   New fields sit ALONGSIDE these, never replacing them.

DESIGN APPROACH
============================================================

Recommended workflow for this session:

STEP 1: MAP EVERY QUESTION
Create a master table of all unique questions across both source
documents + the existing CF form. For each question, tag:
- Source (CF form / Redesigned / High-Value / New)
- Response format (free-text / multi-select / 1-5 scale / single-select)
- System routing (Snapshot / PPD / ABI / Ecosystem / Longitudinal)
- Priority (Essential / High / Nice-to-have)

STEP 2: RESOLVE OVERLAPS
Where the same question appears in multiple sources with different
formats, select the optimal format based on the routing needs.
Rule: 1-5 scale for ABI-feeding questions (granularity needed for
pillar math, maps to PPD amplifiers). Multi-select for categorical
PPD bucket assignments.

STEP 3: CONSOLIDATE REDUNDANCY
The CF form has Q15/16/17 all asking goal variants. Consolidate.
The High-Value doc has questions not in the redesigned form —
evaluate each for signal value and decide include/exclude.

STEP 4: SEQUENCE THE FORM
Order questions for optimal athlete experience:
- Start easy (context, demographics — low cognitive load)
- Build to self-assessment (scales — moderate load, fast)
- Peak with identity + friction (most important coaching data)
- End with goals + commitment (forward-looking, energizing)

STEP 5: TAG MULTI-SYSTEM ROUTING
For each question in the final list, document exactly which
systems consume it and how (field name, calculation role, or
narrative input).

STEP 6: PRESENT FOR APPROVAL
Show the complete form with all tags to the user for review.
Get approval before moving to Task 2 (PPD logic) and Task 3
(ABI logic).
