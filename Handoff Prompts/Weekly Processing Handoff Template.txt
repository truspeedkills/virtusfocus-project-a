2. CONTEXT GATHERING PROTOCOL
============================================================

After reading CLAUDE.md, execute the following steps IN
ORDER before generating any output. Do not skip any step.
Do not begin Stage 2 processing until all steps are done.

STEP 1 — READ THE BASELINE SNAPSHOT
Read the athlete's baseline snapshot file from the path
listed in CLAUDE.md. Internalize the full profile: sport,
position, team, years competing, mental game self-rating,
all 9 sections, all baseline anchors, growth phase
classification and justification.

STEP 2 — READ ALL PRIOR WEEK JSONS
Read EVERY prior week Interpretation JSON from the athlete's
Weekly JSONS directory. Not just the most recent — ALL of
them. You need the full longitudinal data set for:
  - narrative_arc construction
  - trend_analysis computation
  - longitudinal_metrics calculation
  - prior_commitment_result evaluation
  - consistency_signal and recommitment_signal tracking
  - growth_phase progression assessment
  - pattern_shift detection across the full program

STEP 3 — DEEP-READ THE MOST RECENT PRIOR WEEK JSON
Read the most recent prior week JSON with particular
attention to these fields, which directly feed Week N
processing:
  - narrative_arc.open_thread — these are the active
    developmental threads you must address or update
  - micro_commitment — this is the prior commitment you
    must evaluate for execution in the new week's recap
  - upcoming_context — this tells you what was anticipated
    for the current week
  - next_week_focus_question — may appear in the recap
  - coach_insights — informs Stage 3 coaching approach
  - growth_phase + growth_phase_movement — current phase
    and trajectory
  - emotional_intensity — prior week's EI for comparison
  - trend_analysis — all trend directions entering this week
  - longitudinal_metrics — rolling rates and stability

STEP 4 — READ FORMAT REFERENCES
Read the most recent prior week's Coaching Message, Deep
Dive, and Coach Insight files. These are your format
references — match their visual structure, section layout,
separator style, header format, and word count range. Format
consistency across weeks is a quality requirement.

STEP 5 — READ THE NEW WEEKLY REVIEW
Now read the weekly review file specified above. This is the
primary input to the Interpretation Engine. Read it as raw
data — apply the specification's classification rules to
what the athlete wrote, not to any pre-analysis or
assumptions.

STEP 6 — READ ALL SPECIFICATION FILES
Read the specification files listed in Section 5 of this
template. These contain the classification rules, output
formats, compliance requirements, and quality standards that
govern every output you produce. You must read them — do not
rely on summaries or prior knowledge. Rules change between
specification versions.

STAGE 2: INTERPRETATION JSON
------------------------------------------------------------
Input: Baseline snapshot + ALL prior week JSONs + new
  weekly review (raw text)
Process: Apply v9.7 Interpretation Engine specification
  rules directly to the raw recap data.

The raw weekly review is the PRIMARY data source. Apply
classification rules to what the athlete actually wrote:
  - Win the Day: Score each day from the recap. 4-5 = Win,
    2-3 = Neutral, 0-1 = Missed. If no numeric score is
    visible, use category-only classification from the
    weekly scorecard (Gold Streak = Growth tier).
  - Bullseye: Classify center/influence/outer ring quality
    and specificity. Detect noise fixation. Detect category
    confusion (controllables misplaced in wrong ring).
  - Reflection Quality: 1=Surface, 2=Emerging awareness,
    3=Clear pattern recognition, 4=Ownership + forward
    adjustment.
  - Emotional Intensity: Low=task-oriented/composed,
    Moderate=friction named but managed,
    High=significant emotional strain.
  - Alignment: Compare baseline identity claim, hinge habit,
    and primary derailer against this week's evidence.
  - Prior Commitment: Evaluate the most recent prior week's
    micro_commitment against evidence in the new recap.
  - Growth Phase: Check advancement/regression criteria
    against longitudinal data.
  - Narrative Arc: Thread this week into the full program
    arc. Name milestone and open threads.

Core Foundation fallback: Set input_source to
"core_foundation". Skip Steps 12-18 (execution signals,
longitudinal PPD). Populate all CF fallback values for
execution_behavior_signals, baseline_intake_profile,
current_season_phase, forward_anchor, and longitudinal_ppd
blocks.

Determine the week period dates from the weekly review
submission date (the recap covers the 7 days ending the
Saturday before submission, which is typically Monday).

STAGE 5: EDITORIAL AUDIT
------------------------------------------------------------
Input: Interpretation JSON + Coaching Message + Deep Dive
  (all three required). No Parent CM for CF athletes.
Process: Apply v1.4 Editorial Audit Master Prompt +
  Instructions.

Run ALL 13 audit criteria in order. Do not skip any.

TIER 1 — MUST PASS (reject on failure, never auto-correct):
  1. Emotional Intensity Calibration
     - High EI: empathy leads, no framework in first 2
       paragraphs
     - Low EI: no empathy framing, recognition-forward
  2. Clinical Language Compliance
     - ZERO clinical terms. Never auto-corrected.
     - Categories A-D: anxiety, trigger/triggered, emotional
       regulation, coping mechanism, regression, trauma,
       avoidance behavior, self-esteem, therapeutic,
       cognitive restructuring/reframing, self-efficacy,
       emotional dysregulation, maladaptive, behavioral
       intervention, psychoeducation
     - Replacements: anxiety→pressure/nerves, trigger→
       activated/derailed, emotional regulation→composure/
       staying centered, coping mechanism→response pattern/
       go-to move, regression→setback/dip/drift
  3. Data Fidelity
     - Every claim must trace to the JSON. No fabrication,
       no contradiction, no invention.
  4. Prior Commitment Bridge
     - Must acknowledge prior commitment BEFORE new one.
     - 1-3 sentences.
  13. Pipeline Data Leakage Compliance
     - No execution signal terminology (Category E: score
       names, band labels, flag IDs, self-ratings labels,
       raw metrics, surveillance framing, pipeline terms)
     - No intake profile terminology (Category F: PPD
       buckets, scoring artifacts, ABI terms, adversity
       classifications, field names, F6-F8 longitudinal PPD)
     - No forward anchor terminology (Category G: raw text,
       controllability labels, alignment labels, consistency
       labels, field names)
     - CF STRICTNESS: ANY execution/intake/forward-anchor-
       derived language is an automatic FAIL — no such data
       exists for CF athletes.

TIER 2 — SHOULD PASS (auto-correct if possible, else reject):
  5. Athlete Voice Reflection (mirror language once)
  6. Action Specificity (action + when + measurement)
  7. Upcoming Context Anchoring (connect to competition)
  8. Narrative Arc Presence (arc in DD when data exists)
  9. Message Length Constraint (CM 250-450 words)
  10. Micro-Commitment Realism (modality adaptation check)

TIER 3 — QUALITY CHECKS (auto-correct silently):
  11. Brand Voice Tonal Scan (no hype/cliches/coddling/guilt)
  12. Outer-Ring Language Discipline (redirect to center ring)

THREE POSSIBLE RESULTS:
  PASS — no issues detected
  AUTO-CORRECTED PASS — Tier 2/3 issues corrected in place,
    corrected outputs overwrite originals
  REJECT AND REGENERATE — Tier 1 failure, regenerate the
    failing output, then re-audit

If auto-correcting: fix the issue in the CM/DD file on
disk, then note the correction in the audit summary.

6. QUALITY STANDARDS
============================================================

The outputs you produce are delivered to real athletes and
real coaching staffs. Quality is not optional. Every output
must meet these standards — not just pass the audit, but
serve the athlete's development and the coach's decision-
making.

INTERPRETATION JSON (Stage 2):
- Every classification must be evidence-based and traceable
  to specific language in the raw recap.
- The narrative arc must thread this week into the full
  program story — not just describe the current week in
  isolation.
- Open threads must be specific enough that the next
  session can use them without re-reading the entire history.
- The micro-commitment must be realistic, specific, and
  connected to the athlete's upcoming context. It must be
  something the athlete can actually do this week.
- The next_week_focus_question must be a question the
  athlete would want to answer — grounded in something they
  said or did, inviting reflection, not a test.

COACHING MESSAGE (Stage 3):
- This is a message FROM Coach Arron TO the athlete. It
  must sound like a real coach talking to a real person —
  not a report, not a template, not a performance review.
- Recognition must be EARNED — grounded in specific
  evidence from the JSON. Never give generic praise.
- The identity thread must connect the athlete's baseline
  identity claim to what they demonstrated this week. This
  is the "who you are becoming" thread that makes coaching
  personal, not transactional.
- Growth areas must be named directly without softening,
  hedging, or apologizing. Coach Arron is direct.
- The prior commitment bridge shows the athlete their
  coaching is continuous, not episodic.

DEEP DIVE (Stage 3):
- This is the analytical depth document. It exists to give
  the athlete a detailed understanding of their week that
  the shorter CM cannot provide.
- Every Strength Signal and Growth Area must have a specific
  behavioral example and an italicized coaching observation
  that adds perspective the athlete may not see.
- The Weekly Trendline must place this week in the full arc
  — what changed, what continued, what is developing.
- Second person throughout. Coach Arron voice. No third-
  person reporting. No coach-to-coach language.

COACH INSIGHTS (Stage 4):
- This is an institutional document visible to coaching
  staff at every level (position coach → coordinator →
  head coach). Every word must be compliance-safe.
- No personal disclosures, journal quotes, emotional
  narrative, risk flags, or pipeline internal data.
- The Key Performance Signal (Section 8) is the most
  important section — it tells coaches the one thing worth
  paying attention to this week. Make it precise.
- The Coach Context Cue (Section 9) must use neutral
  coaching language: "Reinforce..." / "Monitor trend..." /
  "Recognize..." — never directive.

EDITORIAL AUDIT (Stage 5):
- Run all 13 criteria. Do not skip any.
- Every Tier 1 criterion is a MUST PASS. A single Tier 1
  failure means the output must be regenerated.
- Clinical language violations are the highest-severity
  issue. Scan every sentence in both CM and DD.
- Data fidelity means EVERY factual claim traces to the
  JSON. If you cannot point to the JSON field that supports
  a claim, the claim is fabricated and fails Criterion 3.

7. RULES AND EXPECTATIONS
============================================================

WHAT NOT TO DO:
- Do NOT modify any specification files.
- Do NOT use App-athlete features (execution signals, PPD,
  ABI, intake profile, forward anchor, parent output).
- Do NOT skip reading the specification files — they contain
  critical rules that changed in recent versions.

  downstream stages (Stage 3 reads JSON, not recaps).

  coach_insights, risk_flags, narrative_arc, upcoming_context,
  longitudinal_metrics, execution_behavior_signals,
  baseline_intake_profile, forward_anchor, longitudinal_ppd.
- Do NOT skip the Editorial Audit.
- Do NOT pre-interpret the raw weekly review before reading
  the specification files.
- Do NOT use the word "trigger" or "triggered" — use
  "activated" or "derailed."
- Do NOT use "cognitive reframing" — use "mental shift" or
  "reframing."
- Do NOT fabricate athlete quotes or paraphrase them
  inaccurately.
- Do NOT add pipeline metadata to the CM header (no schema
  version, no EI label, no tier classification).
- Do NOT include action items in the DD Growth Areas — those
  belong in the CM micro-commitment.
