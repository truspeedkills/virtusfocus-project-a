SECTION 4 — PITCH SCRIPT DESIGN PARAMETERS
=====================================================

Length:
  - 30 minutes spoken
  - ~3,000 words target (assumes ~100 words/minute spoken pace
    with pauses, slide transitions, demo gestures)
  - Hard ceiling: 3,500 words. Anything beyond runs over.

Frame (in order):
  - Hook / problem (3-5 minutes)
  - The vision / what we're building (8-10 minutes)
  - Proof / what's already produced (8-10 minutes)
  - Where we are / what we need from this room (4-7 minutes)

Voice and tone:
  - Coach Arron speaking in first person
  - Less tech/architecture up front (per user direction Session 120)
  - More emotional anchor on coach/athlete stories
  - Reference real athletes by anonymized labels (Athlete A, B, C)
    with sport + level
  - Calm authority, not hype
  - Allow for natural pauses; speakers reading scripts often
    underestimate this

Anchor stories to consider weaving in:
  - Grace's 12-week confidence arc (outcome-dependent intake ->
    outcome-INDEPENDENT self-recognition at Wk12; the "I did so

    keystone)
  - Angelo's preparation-system arc demonstrated across three
    timescales (retrospective Wk4, prospective Wks 5/9,
    in-competition Wk10) culminating in phase advancement at the
    final week of his collegiate career
  - Hunter's first-week return after season-ending injury (work-
    anchored "1% better everyday" goal translation, Wrestler of the

  - The "preparation-quality drives performance" principle

What the room needs to walk away with:

  - There is a coherent path from athlete intake to multi-audience

  - The work being asked of the room in the breakouts is

  - Compliance and clinical-vs-performance line is taken seriously

SECTION 6 — EXECUTION PHASES
=====================================================

PHASE 1: Outline structure (HIGH effort, ~30-45 min)
  - Read Tier 1 references
  - Spot-check Tier 2 (Mental Health vs Mental Performance
    explainer is mandatory; mockup gallery skim)
  - Draft top-level outline:
      * Section name
      * Time target (in minutes)
      * Theme / what this section accomplishes
      * Key anchor (story, quote, mockup reference, etc.)
  - Total time targets must sum to 28-30 minutes (leave buffer)
  - Stop. Present outline to user. Wait for go.

PHASE 2: Draft section content (HIGH effort, 60-90 min)
  - For each outlined section, draft full talking points
    paragraph by paragraph
  - Include athlete-language quotes where they land (verify
    quotes against athlete material rewrites in Focus Group
    Materials/Athlete Materials/)
  - Mark any reference to mockup / artifact / breakout question
    explicitly (e.g., "[reference Athlete A 12-week visual]")
  - Word count check at end of phase: target ~3,000, ceiling 3,500
  - No em dashes anywhere (use period, comma, or colon as
    appropriate)
  - Stop. Present full draft to user. Wait for revisions.

PHASE 3: User review + revisions (collaborative, 30-60 min)
  - User marks up sections that don't land or feel off
  - Iterate on specific sections rather than full rewrites
  - Time-box each revision pass (don't rabbit-hole on a single
    paragraph)
  - Flag if any revision pushes word count over 3,500

PHASE 4: Final pass + speaker annotations (MEDIUM effort, 20-30 min)
  - Read aloud and time it (or estimate at ~100 words/min)
  - Add speaker notes inline where helpful (pauses, transitions
    to mockups, emotional beats)
  - Save final pitch script to disk:
      Focus Group Materials/Pitch Script/Pitch_Script_v1.txt
      (create the Pitch Script subfolder if not present)
  - Optional: also generate a .docx via the same generator
    pattern (extend make_focus_group_docx.py if Arron wants a
    formatted version for printing speaker copy)

PHASE 5: Capture protocol (MEDIUM effort, 20-30 min)
  - Define:
      * 3 note-takers (one per breakout) — propose names; user
        confirms or assigns
      * Audio recording (NDA covers; confirm verbally at start)
      * Note-taker shorthand convention (capture verbatim quotes,
        mark which question, mark sentiment)
      * Post-session synthesis plan (who reviews, by when, what
        they produce)
      * Worksheet collection process (collect at end of breakout,
        attendee NAME field optional so respect anonymity)
  - Save to:
      Focus Group Materials/Day-of Operations/Capture_Protocol.txt
  - Create the Day-of Operations subfolder if not present.

PHASE 6: Welcome + NDA-at-door flow (MEDIUM effort, 20-30 min)
  - 10-minute welcome script (covered by the 12:00-12:15 block):
      * Greet, name introductions (consider quick go-around)
      * NDA reminder (signed at door, but reinforce confidentiality)
      * Why this group specifically (1-2 sentences per segment
        purpose)
      * Today's goal (NOT to validate; to extract what we're
        missing)
      * Format walk-through (briefly: pitch -> mockups -> lunch
        -> breakouts -> pricing -> synthesis)
  - NDA-at-door flow:
      * Check-in lead (someone non-presenting, available
        11:30-12:00)
      * QR code primary path (already set up in Slack canvas)
      * Printed NDA copies for less-tech attendees (need print
        run estimate; ~5 copies should be plenty)
      * Sign-in roster (track who attended for follow-up)
  - Save to:
      Focus Group Materials/Day-of Operations/Welcome_and_NDA_Flow.txt

PHASE 7: Commit + push + close (LOW effort, 5-10 min)

=====================================================
SECTION 10 — FAILURE MODES (HANDLE THESE)
=====================================================

If pitch goes long after Phase 2 draft:
  - First cut: tighten transitions and remove redundant framing
    sentences
  - Second cut: drop one anchor story (keep the strongest)
  - Third cut: tighten the "problem" section (it tends to
    overrun)
  - Do NOT cut "what we need from this room" — that's the most
    important closer

If a section doesn't land after 2 revision passes:
  - Document what's not working
  - Move on; come back at end of Phase 4
  - Fresh eyes after working on later sections often reveals the
    fix

If user wants to reopen a locked decision:
  - Pause and confirm. Locked decisions from Session 119/120
    drove a lot of downstream work (worksheets, packets, brand-
    name treatment). Reopening one ripples.

If you find missing context (something the pitch needs that's
not in any reference file):
  - Stop. Ask user. Don't fabricate.

If a quote you want to use doesn't appear verbatim in the
rewritten athlete materials:
  - Verify against the original (preserved at Athlete Data/...)
  - If discrepancy, use the original verbatim (athlete language
    doesn't change in audit-and-rewrite — only coach prose does)
