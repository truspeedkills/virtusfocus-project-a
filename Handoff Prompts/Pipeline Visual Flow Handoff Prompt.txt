2. THE PIPELINE — 7 STAGES + APP BACKEND
============================================================

The pipeline has 7 AI/computation stages plus the app backend
which handles data collection and deterministic extraction.

STAGE 0: APP BACKEND (Deterministic — No AI)
  Role: Collects raw athlete interactions and produces
        structured data objects for downstream stages.
  Produces:
    - Daily Mini-JSON (nightly, one per athlete per day)
      → consumed by Stage 6 + Stage 7
    - Weekly Input Object (Monday, assembles 7 daily events
      + weekly check-in into single structured package)
      → consumed by Stage 2
  Cadence: Continuous collection, nightly Mini-JSON generation,
           weekly input object assembly on Monday

STAGE 1: ATHLETE SNAPSHOT GENERATOR (Claude Opus — One-time)
  Role: Processes athlete intake form into structured baseline
        profile. Computes PPD (Primary Problem Detector) scores,
        ABI (Athlete Baseline Index) pillar scores, and writes
        10-section narrative snapshot.
  Input: App intake form (29 questions, 7 sections)
  Output: Athlete Snapshot file (structured data block +
          narrative sections + 5 locked baseline anchors)
  Cadence: ONE TIME at athlete enrollment
  Consumed by: Stage 2 (reads snapshot for baseline context),
               Stage 3 (reads snapshot for coaching persona),
               Stage 6 (reads snapshot for daily snippet context)

STAGE 2: INTERPRETATION ENGINE (Claude Opus — Weekly)
  Role: Reads all available data sources and produces a
        comprehensive weekly Interpretation JSON — the single
        source of truth for all downstream stages.
  Inputs:
    - Athlete Snapshot (from Stage 1)
    - Weekly Input Object (from App Backend — 7 daily events
      + weekly check-in including Weekly Recap, Motivation
      Inventory, season context, Forward Anchor)
    - All prior week Interpretation JSONs (for longitudinal
      analysis, trend detection, cross-week comparisons)
  Output: Interpretation JSON (v9.6 schema)
    Contains: weekly_tier, win_the_day scores, consistency
    signal, reflection_quality_score, growth_phase,
    emotional_intensity, micro_commitment,
    execution_behavior_signals (50+ fields, 7 composite
    scores, 15 coach flags), baseline_intake_profile
    (7 pass-through fields), forward_anchor (text +
    3 classification signals), narrative_arc,
    upcoming_context, risk_flags, coach_insights subfields
  Cadence: WEEKLY (Monday)
  Consumed by: Stage 3, Stage 4, Stage 5, Stage 6 (Premium)

STAGE 3: COACHING OUTPUT ENGINE (Claude Opus — Weekly)
  Role: Transforms Interpretation JSON into athlete-facing
        coaching content. Coach Arron persona.
  Input: Interpretation JSON (from Stage 2)
  Outputs (up to 3):
    1. Coaching Message — athlete-facing, 250-450 words
       (6 sections: Recognition, Drift, Identity, Strength,
       Growth, Micro-Commitment)
    2. Deep Dive — athlete-facing, detailed coaching analysis
       (5 sections: Mindset Summary, Trendline, Strength
       Signals, Growth Areas, What's Ahead)
    3. Parent Coaching Message — parent-facing, 150-250 words
       (4 sections: Weekly Context, What Stood Out, What's
       Ahead, How You Can Support)
       CONDITIONAL: Only generated when parent_inclusion=true
       (D2C Premium + Parent tier ONLY)
  Cadence: WEEKLY (Monday)
  Consumed by: Stage 5 (audit), Athlete (via app), Parent (via app)

STAGE 4: COACH INSIGHTS ENGINE (Claude Opus — Weekly)
  Role: Produces compliance-safe analytical dashboards for
        coaches and/or parents. NOT athlete-facing.
  Input: Interpretation JSON (from Stage 2)
  Outputs (up to 4, tier-dependent):
    1. Weekly Performance Insight — coach-facing, 10 sections
       (INSTITUTIONAL ONLY)
    2. Weekly Team Snapshot — coach-facing aggregate, 7 sections,
       requires 5+ athletes (INSTITUTIONAL ONLY)
    3. Parent Weekly Insight — parent-facing, 10 sections
       (rebranded labels), Coach Arron voice, third person
       (D2C PREMIUM + PARENT ONLY)
    4. Monthly Institutional Summary — not currently active
  Cadence: WEEKLY (Monday)
  Consumed by: Coach dashboard (institutional), Parent app/dashboard
               (D2C Premium + Parent)

STAGE 5: EDITORIAL AUDIT AGENT (Claude Opus — Weekly)
  Role: Quality gate. Audits Stage 3 output against 13 tiered
        criteria. Does NOT generate coaching content.
  Input: Interpretation JSON + Coaching Message + Deep Dive
         + Parent Coaching Message (conditional, if generated)
  Output: PASS / AUTO-CORRECTED PASS / REJECT AND REGENERATE
    - If PASS: outputs proceed to delivery
    - If AUTO-CORRECTED: corrected outputs overwrite originals
    - If REJECT: Regeneration Instructions written, Stage 3
      must re-run
  Cadence: WEEKLY (Monday, runs after Stage 3)
  13 Criteria across 3 tiers:
    Tier 1 (must-pass): JSON fidelity, clinical language,
      tone alignment, length compliance, pipeline data leakage
    Tier 2 (should-pass): growth phase accuracy, micro-commitment
      realism, emotional intensity calibration, etc.
    Tier 3 (quality): style, flow, repetition

STAGE 6: DAILY COACHING ENGINE (Claude Sonnet — Daily)
  Role: Produces a short daily coaching snippet for the athlete.
        2-3 sentences, 30-60 words, Coach Arron voice.
        Closes yesterday, bridges to today.
  Inputs (vary by tier):
    BASE: Mini-JSON + Athlete Snapshot + Previous Snippet
    PREMIUM: Mini-JSON + Athlete Snapshot + Previous Snippet
             + Weekly Interpretation JSON + Forward Anchor data
  Output: Daily Coaching Snippet (2-3 sentences, signed
          "— Coach Arron")
  Cadence:
    BASE: DAILY Mon-Sun (7 snippets/week)
    PREMIUM: DAILY Tue-Sun (6 snippets/week — Monday is
             full Coaching Message + Deep Dive delivery day)
  Compliance: Embedded 3-layer self-validation scan
              (no external audit stage for daily output)
  Consumed by: Athlete (via app, delivered alongside
               Morning Tune-Up)

STAGE 7: DAILY COACH INSIGHTS ENGINE (Deterministic — No AI)
  Role: Produces a structured categorical daily signal for
        each athlete. Pure backend computation from Mini-JSON.
        Zero AI cost. Zero generation latency.
  Input: Daily Mini-JSON + optional Weekly Calibration Cache
         (emotional_intensity, growth_phase from most recent
         Stage 2 JSON)
  Output: Daily Athlete Signal
    - Traffic Light: Green / Yellow / Orange / Gray
    - 4 Contributing Signals (coach-facing only):
      Day Outcome: Aligned / Building / Reset Day
      Focus Alignment: Centered / Mixed / Drifting
      Follow-Through: Demonstrated / Gap
      Weekly Trajectory: Building Week / Steady Week / Cooling Week
    - Team/Position Group Aggregation (institutional only):
      Strong Alignment / Positive Trend / Mixed Signals /
      Needs Attention
  Cadence: DAILY Mon-Sun (7 days/week, all athletes)
  Parent Label Mapping (D2C Premium + Parent only):
    Green → Good Day
    Yellow → Building Day
    Orange → Reset Day
    Gray → No Update Today
    (Contributing signals HIDDEN from parents — traffic
    light color + label only)
  Consumed by:
    INSTITUTIONAL: Coach dashboard (full signal + team tiers)
    D2C PREMIUM + PARENT: Parent app (traffic light with
                          parent labels only)

3. TIER-BY-TIER STAGE ACTIVATION MAP
============================================================

Use this as the definitive guide for which stages appear
in each diagram and what outputs they produce.

--------------------------------------------------------------
DIAGRAM 1: D2C BASE
--------------------------------------------------------------

Who is this athlete?
  - App-only athlete, no weekly coaching subscription
  - Gets daily coaching snippets only (standalone product)
  - No coach audience, no parent audience
  - Cheapest tier (~$0.06/athlete/week)

Active stages:
  Stage 0 (App Backend) → produces Mini-JSON daily
  Stage 1 (Snapshot Generator) → one-time at enrollment
  Stage 6 (Daily Coaching Engine) → 7 snippets/week Mon-Sun

Inactive stages (DO NOT RUN):
  Stage 2 (Interpretation Engine) — no weekly pipeline
  Stage 3 (Coaching Output Engine) — no weekly pipeline
  Stage 4 (Coach Insights Engine) — no coach/parent audience
  Stage 5 (Editorial Audit) — nothing to audit
  Stage 7 (Daily Coach Insights) — no coach/parent audience

Inputs consumed:
  Stage 6 reads: Mini-JSON (daily) + Athlete Snapshot (baseline)
                 + Previous Snippet (for cohesion)
  NO weekly Interpretation JSON available — Stage 6 operates
  in "Base mode" with limited context

Outputs delivered to athlete:
  - Daily Coaching Snippet (via app, 7 days/week)
  That's it. One output type, one audience.

Key visual elements:
  - Simple linear flow: App Backend → Mini-JSON → Stage 6 → Snippet → Athlete App
  - Snapshot feeds in as one-time baseline
  - Should look clean and minimal compared to other diagrams

5. VISUAL DESIGN GUIDANCE
============================================================

These are suggestions, not requirements. The fresh session
should use its judgment on the best visual approach.

SUGGESTED VISUAL ELEMENTS:
  - Rounded rectangles for pipeline stages (numbered 1-7)
  - Different colors for:
    - AI stages (Opus = darker shade, Sonnet = lighter shade)
    - Deterministic stages (App Backend, Stage 7 = distinct color)
    - Inactive/skipped stages (gray or absent)
  - Arrows showing data flow direction
  - Dashed arrows for "inform-only" data (data that shapes
    output but is never surfaced to the recipient)
  - Distinct output icons or shapes for each deliverable type
    (document for CM/DD, signal/badge for traffic light, etc.)
  - Audience labels at the terminus of each output arrow
    (Athlete App, Coach Dashboard, Parent App)
  - Cadence labels on each stage (One-time, Weekly, Daily)
  - A legend explaining all visual conventions

LAYOUT SUGGESTION:
  - Top: Data sources (Athlete interactions, Intake form)
  - Middle: Pipeline stages flowing left-to-right or top-to-bottom
  - Bottom: Output destinations (Athlete App, Coach Dashboard,
    Parent App) as distinct endpoint zones
  - Weekly pipeline and daily pipeline as visually distinct tracks
  - Stage 5 (Audit) shown as a gate/checkpoint between
    production and delivery

IMPORTANT DISTINCTIONS TO SHOW:
  - Stage 2 is the SINGLE SOURCE OF TRUTH — everything
    downstream reads from its JSON output
  - Stage 5 is a QUALITY GATE — it can reject and force
    Stage 3 to regenerate (show this feedback loop)
  - Stage 6 has two modes: Base (standalone, minimal inputs)
    and Premium (enriched by weekly JSON)
  - Stage 7 is NOT AI — it's pure computation
  - The App Backend is NOT a pipeline stage — it's infrastructure
    that feeds the pipeline
  - Parent Dashboard is a PRESENTATION LAYER — it renders
    data from Stage 4 + Stage 7, it doesn't compute anything

8. LOCKED DECISIONS — DO NOT CHANGE
============================================================

These architectural decisions are final. The visual diagrams
must accurately reflect them:

  - D2C Base = daily snippets ONLY (no weekly pipeline)
  - D2C Premium = weekly pipeline + daily snippets (no parent)
  - D2C Premium + Parent = full pipeline including parent outputs
  - Institutional = full pipeline with coach-facing analytics
    (no parent output, no Base/Premium distinction)
  - Stage 7 is deterministic (no AI model)
  - Stage 6 uses Sonnet (not Opus)
  - All other AI stages use Opus
  - Stage 5 audits Stage 3 output only (not Stage 4, not Stage 6)
  - Stage 6 has embedded self-validation (no external audit)
  - Parent data boundary = identical to coach data boundary
  - Premium snippets skip Monday (CM+DD delivery day)
  - Base and Institutional snippets run 7 days/week
  - Team Snapshot requires 5+ athletes
  - Position Group aggregation requires 3+ athletes per group
  - Parent sees traffic light only (contributing signals hidden)
  - All coach-facing output requires mandatory disclaimer
