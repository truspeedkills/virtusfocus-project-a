PURPOSE
=======
The May 9, 2026 focus group needs to demonstrate current-spec
quality output as part of attendees' evaluation. Some weeks
selected for the featured athletes were generated under older
spec versions (v1.3, v1.4, v1.8/V4, v2.1) and contain voice
quality issues that have been retired in v2.3/V5:
  - em dashes
  - retired phrase constructions ("the version of X", "Here's
    what I want you to...", "Not X. Y." parallel-clause refute,
    "The question is whether...", etc.)
  - telegraphing ("Now here's what I want to push you on...")
  - "That is [Noun]." standalone overuse
  - pipeline classification language ("Win days", "Growth
    weeks", named phase classifications, "the recap")
  - Other Criterion 14 fail conditions (12 total)

This session AUDITS the existing CM, DD, and Coach Insight
files for the specified weeks against current spec, and
REWRITES them to current spec. The underlying JSON
interpretations are NOT regenerated. Originals are preserved.
New files go to a separate folder so the original generation
history stays intact for future voice quality comparison work.


CRITICAL DISTINCTION FROM PIPELINE PROCESSING
==============================================
This is NOT pipeline processing. We do NOT:
  - Re-read the weekly recap PDF
  - Regenerate the JSON
  - Re-interpret the data
  - Change the coaching conclusions or analysis
  - Modify Audit Summary files

We DO:
  - Read the existing JSON (data source remains valid)
  - Read the existing CM, DD, Coach Insight
  - Audit each against current spec for voice quality issues
  - Rewrite each to current spec, preserving athlete-specific
    voice and content while fixing voice quality issues
  - Save new files to Focus Group Materials\Athlete Materials\
  - For Grace Wk12 only: generate fresh Parent CM + Parent
    Insight via current Stage 3 + Stage 4 specs

EXECUTION PHASES
================

PHASE 1: AUDIT
--------------
For each file (CM, DD, Coach Insight) for each week:

  1.1 Read the file in full.

  1.2 Mechanical scan:
        - C1 Em dashes: zero tolerance, must be removed
        - C2 Contractions: present where natural? Default
          register
        - C3 Retired phrases (full retired list in v2.3
          Section 35 C3)
        - C4 Pipeline classification language

  1.3 Criterion 14 scan (12 fail conditions, v1.6 Section 14):
        14.1   Reusable opening
        14.2   Telegraphing
        14.3   Untraceable paragraph
        14.4   Category coverage over response
        14.5   Brand copy close
        14.6   Same construction in 3-week window
        14.7   DD reads as expanded CM
        14.8   "That is [Noun]." standalone overuse
               (cap: 1/CM, 1/DD)
        14.9   "version of [X]" construction
        14.10  Triple-negative setup
        14.11  Trendline arc recitation
        14.12  Pipeline-adjacent vocabulary

  1.4 Document findings per file:
        - List specific violations with line numbers
        - Note mechanical fixes vs surrounding-sentence rewrites
        - Note passages that pass current spec — preserve
          verbatim


PHASE 2: REWRITE
----------------
  2.1 For each violation, determine the natural rewrite:
        - Consult Voice Reference exemplars (Sections 3-4)
          for guidance on the coaching move involved
        - Preserve athlete-specific content, phrases, moments
        - Preserve coaching arc and conclusions
        - Fix only what fails audit — don't rewrite what passes

  2.2 Apply rewrites in working copy.

  2.3 Re-run mechanical and Criterion 14 scans on the rewritten
      version. Iterate until clean.

FAILURE-MODE HANDLING
=====================
- Existing JSON missing fields required by current spec: do
  not regenerate. Note the schema gap, proceed with available
  data, flag in audit summary. Schema gaps don't affect coaching
  output rewrite.

- Passage genuinely cannot be rewritten without losing the
  coaching point: preserve it and flag for user review before
  final save.

- Athlete voice cannot be preserved cleanly through the
  rewrite: stop, document the issue, and ask user before
  proceeding.

- Coach Insight file passes audit cleanly with no issues:
  document as PASS in summary, no rewrite needed, do not
  duplicate the file to the new folder. The original at
  Athlete Data\[Athlete]\Coach Insights\ is the version of
