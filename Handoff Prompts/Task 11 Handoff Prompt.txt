THE CRITICAL STAGE 4 CONSTRAINT (unchanged from Task 10):

execution_behavior_signals is INFORM-ONLY for Stage 4.

- Raw execution data (completion rates, timing, backfill counts, reminder dependency, streak counts, component counts, question-level yes rates) NEVER reaches coaching staff in any form
- Composite scores and coach flags are consumed INTERNALLY by the engine to produce better compliance-safe sections
- The inform-only routing pattern is identical to how coach_insights subfields already work
- This aligns with VF_Coach_Insights_Compliance_Framework.txt prohibited metrics

Additionally, the following execution data categories must NEVER appear in any Stage 4 output (from the Execution Signal Compliance Filter in Task 10):
- Completion rates (decimal or percentage)
- Timing data (on-time counts, late counts, timeliness profiles)
- Backfill counts, reminder dependency counts
- Streak counts (current_streak, longest_streak)
- Component completion counts
- Question-level yes rates (q1-q5 rates)
- Self-initiated vs. reminder-dependent counts
- Composite score names (Ownership Index, Drift Score, etc.)
- Composite score numeric values (0-100)
- Composite score band labels (High, Active, Disrupted, etc.)
- Coach flag IDs, labels, or severity tier names
- Self-ratings alignment classification labels (Conflated, Undervalued, Aligned)
