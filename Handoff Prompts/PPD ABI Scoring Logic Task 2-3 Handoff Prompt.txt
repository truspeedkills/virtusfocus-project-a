PPD SCORING LOGIC — STARTING POINT AND DESIGN QUESTIONS
============================================================

The redesigned form document contains a PPD scoring proposal.
Here is what it proposes vs. what needs to change:

CONFIRMED FROM SOURCE (likely keep as-is):
- Selection Weight: +3 per friction selection (Q15) to matching bucket
- Trigger Amplifier weights: +2 primary / +1 secondary per trigger
- Tie-breaker: pressure thought text classification
- Output: Top 3 ranked problems

NEEDS ADAPTATION (inputs changed from source assumptions):

1. RESET SPEED AMPLIFIER — Source used 4 categorical options:
     Immediate = +0 / Next rep = +1 / Few minutes = +2 / Sticks = +3
   Now Q9 is a 1-5 scale (1 = sticks with me, 5 = reset immediately).
   Need to map 1-5 scale to amplifier weight for Mistake Recovery Lag.

   Suggested mapping (invert scale since low = slow reset):
     Q9 = 5 (immediate)    -> +0 amplifier
     Q9 = 4 (next rep)     -> +1 amplifier
     Q9 = 3 (moderate)     -> +2 amplifier
     Q9 = 2 (few minutes)  -> +2 amplifier
     Q9 = 1 (sticks)       -> +3 amplifier

   [DESIGN DECISION]: Confirm this mapping. Should 2 and 3 both
   be +2, or should there be 5 distinct values (+0/+1/+1/+2/+3)?

2. ROUTINE AMPLIFIER — Source used 3 categorical options:
     Consistent = +0 / Somewhat = +1 / None = +3
   Now Q13 is a 1-5 scale (1 = no routine, 5 = very consistent).
   Need to map 1-5 scale to amplifier weight for Structure Gap.

   Suggested mapping (invert scale since low = no routine):
     Q13 = 5 (very consistent) -> +0 amplifier
     Q13 = 4 (consistent)      -> +0 amplifier
     Q13 = 3 (somewhat)        -> +1 amplifier
     Q13 = 2 (minimal)         -> +2 amplifier
     Q13 = 1 (none)            -> +3 amplifier

   [DESIGN DECISION]: Confirm this mapping. Should 4 and 5 both
   be +0, or should 4 get +1?

3. ECOSYSTEM AMPLIFIER — Source said "if home conversations are
   advice-heavy/pressure: +3 to Ecosystem Friction."
   Now Q23 has 4 options. Need amplifier per option:
     "Mostly supportive"             -> +0 (no friction)
     "A lot of analysis or advice"   -> ? (mild friction)
     "Pressure to perform"           -> +3 (clear friction)
     "We don't discuss performance"  -> ? (disengagement signal)

   [DESIGN DECISION]: What weight for "analysis/advice" and
   "don't discuss"? Source only distinguished friction vs no-friction.
   The new form adds nuance. Options:
   (A) Binary: supportive = +0, all others = +3
   (B) Graduated: supportive = +0, advice = +2, pressure = +3, don't discuss = +1
   (C) Two-tier: supportive/don't discuss = +0, advice/pressure = +3

4. PRESSURE THOUGHT TIE-BREAKER — Source classifies free text
   into 3 categories:
     Avoidance ("Don't mess up") -> +1 Overthinking, +1 Pressure Reactivity
     Proving ("I need to prove")  -> +1 Confidence Volatility
     Obligation ("Can't let them down") -> +1 Pressure Reactivity, +1 Ecosystem
   Q18 is free text. The Snapshot Builder (or PPD computation
   system) must classify the text.

   [DESIGN DECISION]: Should this classification be:
   (A) Done by the Snapshot Builder agent (Claude classifies the text
       using the 3 categories above + an "Approach" category for
       positive framings like "Let's go")
   (B) Expanded to more categories
   (C) Made more deterministic with keyword matching rules

5. ADDITIONAL AMPLIFIER CANDIDATES — Two ABI scale questions
   could serve as PPD amplifiers beyond what the source document
   proposed:
   - Q10 (big-moment composure, 1-5) could amplify Pressure Reactivity
   - Q7 (development driver, 1-5) could amplify Discipline Gap

   [DESIGN DECISION]: Add these as amplifiers, or keep PPD inputs
   to the 6 already defined (Q9, Q13, Q15, Q16, Q18, Q23)?
   Adding more amplifiers increases scoring precision but also
   complexity. The source document intentionally kept it simple.

6. OUTPUT FORMAT — Source proposed:
     "1) Mistake Recovery Lag (9) — Trigger: after mistakes —
      Coaching: reset reps + next-play language"
   For pipeline consumption, we need a structured format:
     ppd_top_3: [
       { rank: 1, bucket: "Mistake Recovery Lag", score: 9,
         trigger: "After making a mistake",
         coaching_line: "..." },
       ...
     ]
   Plus supporting fields:
     ppd_all_scores: { bucket: score } (all 8 buckets for
                     longitudinal comparison)
     ppd_primary_problem: "Mistake Recovery Lag" (shorthand for
                          baseline_primary_derailer derivation)

   [DESIGN DECISION]: Confirm output format. Should all 8 bucket
   scores be stored (for future longitudinal PPD comparison), or
   only the Top 3?

7. EDGE CASES:
   - What if an athlete selects 0 friction items? (Q15 says "up to 3"
     — is 0 valid? Probably not — should be 1 minimum)
   - What if all 8 buckets score 0 after amplifiers? (Only possible
     if 0 selected AND no amplifiers fire — impossible if minimum 1)

ABI SCORING LOGIC — STARTING POINT AND DESIGN QUESTIONS
============================================================

The redesigned form document contains an ABI scoring proposal.
Here is what it proposes vs. what needs refinement:

CONFIRMED FROM SOURCE (likely keep as-is):
- Formula: pillar_score = round((Q_a + Q_b) / 2 * 2)
  This maps 1-5 paired averages to 0-10 scale:
    Both 1s → round(1/2*2) = round(1.0) = 1/10 (minimum non-zero)
    Both 3s → round(3/2*2) = round(3.0) = 3 -> wait...
  Actually let me recalculate:
    Q_a=1, Q_b=1: round((1+1)/2 * 2) = round(2/2 * 2) = round(2) = 2
    Q_a=3, Q_b=3: round((3+3)/2 * 2) = round(6/2 * 2) = round(6) = 6
    Q_a=5, Q_b=5: round((5+5)/2 * 2) = round(10/2 * 2) = round(10) = 10
    Q_a=1, Q_b=5: round((1+5)/2 * 2) = round(6/2 * 2) = round(6) = 6
    Q_a=2, Q_b=3: round((2+3)/2 * 2) = round(5/2 * 2) = round(5) = 5
  Range: 2-10 (not 0-10; minimum is 2 when both answers are 1)

  [DESIGN DECISION]: The formula produces a range of 2-10, not
  0-10 as stated. Options:
  (A) Accept 2-10 range (practical — no one scores 0 or 1)
  (B) Adjust formula to produce true 0-10: pillar = round(((Q_a-1)+(Q_b-1))/2 * 2.5)
      This maps (1,1)=0, (3,3)=5, (5,5)=10
  (C) Keep the formula, relabel range as 2-10
  Recommendation: Option A is simplest and the bands already
  accommodate it (0-16 total is still "Needs Foundation" even
  though minimum total is 8).

- Total ABI: Sum of 4 pillar scores (range 8-40 with current formula)
- Bands: 0-16 Needs Foundation / 17-28 Developing /
         29-36 Consistent / 37-40 Leadership

  [DESIGN DECISION]: With actual range 8-40, the "Needs Foundation"
  band (0-16) can only be reached when average pillar is 4 or below
  (out of 10). Should bands be adjusted for the real range?

NEEDS DESIGN:

1. PRIMARY EMPHASIS RULES
   Source says: "Primary Emphasis = lowest 1-2 dimensions."
   Need precise rules:
   - Always identify the single lowest pillar?
   - Identify TWO lowest if they're within X points of each other?
   - What threshold separates "one emphasis" from "two"?
   - What if all 4 pillars are equal?
   - What if 3 pillars tie for lowest?

   [DESIGN DECISION]: Define deterministic Primary Emphasis rules.

2. ABI ↔ GROWTH PHASE RELATIONSHIP
   Source says bands "support, not replace" Growth Phase.
   The Snapshot Builder assigns Growth Phase from behavioral
   assessment. ABI is a quantitative input that may:
   (A) Inform the Growth Phase decision (soft input)
   (B) Set a floor/ceiling (hard constraint — e.g., ABI < 17
       cannot be classified higher than Developing)
   (C) Be completely independent (Growth Phase from narrative
       only, ABI separate metric)

   [DESIGN DECISION]: What is the relationship? Previous pipeline
   design chose deterministic Growth Phase thresholds based on
   weekly data (see VF_Interpretation_JSON_Rules.txt Section 4C).
   ABI is a one-time intake score, not weekly. Should it set an
   initial constraint that weekly data can then advance?

3. ABI ↔ PPD INTERACTION
   - Low ABI pillar scores correlate with certain PPD buckets:
     Low Composure ↔ Mistake Recovery Lag, Pressure Reactivity
     Low Structure ↔ Structure Gap, Discipline Gap
     Low Ownership ↔ Discipline Gap
     Low Focus ↔ Focus Drift
   - Should PPD and ABI cross-validate? (e.g., if ABI Composure
     is low but athlete didn't select composure-related friction,
     should that flag a gap in self-awareness?)
   - Or should they be independent assessments that happen to
     share some inputs?

   [DESIGN DECISION]: Independent or cross-validating?
   Recommendation: Independent at intake. The future PPD Alignment
   Signal (intake PPD vs execution-derived PPD) handles the
   self-awareness gap detection longitudinally.

4. ABI OUTPUT FORMAT

       coaching_strategy: {
         "Structure": "Build a 3-step routine...",
         "Composure": "Reset language + Bullseye..."
       }
     }

   [DESIGN DECISION]: Confirm output format. Should coaching_strategy
   lines be included in the ABI output, or deferred to the
   Snapshot Builder (which synthesizes coaching emphasis from
