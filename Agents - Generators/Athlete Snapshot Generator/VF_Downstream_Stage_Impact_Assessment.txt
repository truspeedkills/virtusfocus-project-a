THE 7 FIELDS AND THEIR DOWNSTREAM INTENT:

  competitive_level — Stage 3 formal coaching calibration rule.
    Determines vocabulary complexity, directness level, accountability
    framing, and concept naming per competitive level. Scalability-
    critical for Youth through Professional.

    indicator. Renamed from season_phase to prevent confusion. Stale
    after athlete progresses through seasons.

  ppd_primary_problem — The #1 ranked PPD bucket name. Backs the
    baseline_primary_derailer in baseline_profile. Informs Stage 3
    coaching theme selection by naming the specific friction pattern.

  ppd_tie_breaker_classification — Avoidance/Proving/Obligation/
    Approach (or dual). Informs Stage 3 tone calibration: how the
    athlete frames pressure internally.

  ppd_top_3 — Array of top 3 PPD buckets with scores, trigger
    contexts, and static coaching implication lines. Coaching
    implications inform Stage 3 coaching direction as baseline
    recommendations.

  abi_primary_emphasis — The 1-2 ABI pillars scoring lowest (or
    "Balanced"). Informs Stage 3 pillar-based coaching calibration:
    which behavioral domains need the most coaching attention.

  adversity_response_pattern — Self-reported adversity style from
    intake Q20. Complements baseline_primary_derailer for tone
    calibration during derailer-activation weeks.

2A. COMPETITIVE LEVEL CALIBRATION RULE (NEW SECTION)
------------------------------------------------------------

DESIGN DECISION (Q1): Modifier-based system. A base calibration table
defines per-level tone parameters. These parameters interact with
existing contextual modifiers (emotional_intensity, coach flags,
self-ratings alignment) to produce nuanced calibration. This is Option
(c) from the design questions — it creates the most athlete-appropriate
output because the same data pattern requires different coaching for a
Youth athlete versus a Professional.

RULE STRUCTURE:

The Competitive Level Calibration Rule is a MODIFIER applied to all
coaching output sections. It does not add new sections or change the
mandatory structure. It adjusts HOW content is expressed within
existing sections.

The rule has two components:
  (1) Base Calibration Table — 5 parameters per competitive level
  (2) Interaction Rules — how competitive level modifies existing
      contextual rules (emotional intensity, coach flags, self-ratings)

EXAMPLE PHRASES — SAME COACHING POINT ACROSS LEVELS:

Coaching point: Athlete showed drift in daily execution this week.

  Youth:
    "You had some days this week where you didn't bring your best
    effort. That happens. What matters is what you do tomorrow.
    Pick one thing you can do better and go do it."

  Middle School:
    "Your effort shifted this week — some days were strong, others
    slipped. That's a pattern worth noticing. Tomorrow, bring the
    same focus you had on your best day this week."

  High School:
    "Execution dipped midweek. Your Win the Day consistency dropped
    from where it was. The gap isn't about ability — it's about
    showing up the same way every day. Close it this week."

  College:
    "The way you showed up shifted this week. Not one off day —
    engagement pulled back across the board. Your preparation
    standard is higher than what the week delivered. Address it
    directly."

  Professional:
    "Execution pattern compressed this week. You know your standard.
    The week didn't meet it. Identify what shifted and correct it
    before it becomes a trend."

COMPONENT 2: INTERACTION RULES

Competitive level modifies HOW existing contextual rules are applied.
It does not override them.

EMOTIONAL INTENSITY + COMPETITIVE LEVEL:

  When emotional_intensity = "High":
    Youth/Middle School: Lead with WARMTH. Extended empathetic
      acknowledgment (2-3 sentences). Reduce all framework language
      to zero. Use only concrete behavioral descriptions. Do not
      name the emotion clinically — describe it in the athlete's
      own language or simple terms ("that was a hard week").
    High School: Lead with EMPATHY. Standard empathetic lead
      (1-2 sentences) per existing rule. Light framework language
      acceptable after the empathetic lead.
    College: Lead with ACKNOWLEDGMENT. Current rule (1-2 sentences).
      Framework language reduced but not eliminated.
    Professional: Lead with BRIEF ACKNOWLEDGMENT. One sentence.
      Then address the pattern directly. Professionals expect
      honesty quickly. Extended empathy reads as patronizing.

  When emotional_intensity = "Low":
    All levels: Standard tone per base calibration table.
    No emotional preamble at any level.

COACH FLAGS + COMPETITIVE LEVEL:

  Action-severity flags:
    Youth: Frame as "something shifted" — name the behavioral
      change without alarm. Micro-commitment stays simple and
      concrete. Do not use language suggesting the athlete is
      failing ("your engagement collapsed").
    Middle School: Frame as "a pattern worth addressing" —
      slightly more direct than Youth but still encouraging.
    High School: Current rule applies. Direct address, clear
      naming of the pattern.
    College: Current rule applies. Pattern leads the message.
    Professional: Most direct framing. "This is a trend. Here's
      what needs to change." Expectation of self-correction.

  Attention-severity flags:
    Youth: Weave into encouragement. "You did great on X. Now
      let's build Y too."
    Middle School: Name the gap gently alongside the strength.
    High School/College/Professional: Current rules apply with
      increasing directness per base table.

  Monitor-severity flags:
    All levels: Subtle integration per current rule. No level-
    specific modification needed — monitor flags are already
    designed as subtle tone adjustments.

SELF-RATINGS ALIGNMENT + COMPETITIVE LEVEL:

  Conflated (athlete rates self higher than execution shows):
    Youth: Do NOT address directly. Youth athletes rating
      themselves highly is developmentally normal. Gently redirect
      to specific controllable actions without naming the gap.
      "Here's what to focus on this week" (not "your week didn't
      match what you thought").
    Middle School: Very gentle bridge. "You're feeling good about
      where you are — and there's room to make the execution match
      that feeling."
    High School: Standard grounding language per existing rule.
    College: Standard grounding language per existing rule.
    Professional: More direct. "Self-assessment ran ahead of
      execution this week. Close the gap."

  Undervalued (athlete rates self lower than execution shows):
    All levels: Current elevation language works across all levels.
    Youth may benefit from even simpler framing: "You did better
    than you think you did this week. Look at what you actually
    did." No level-specific override needed.

CORE FOUNDATION HANDLING:
When baseline_intake_profile.input_source = "core_foundation",
competitive_level = "not available — core foundation intake."
Stage 3 defaults to College calibration (current behavior). All
existing Core Foundation athletes receive identical output to
current v1.5. Zero degradation.

DEFAULT WHEN COMPETITIVE LEVEL IS MISSING:
If competitive_level = "not available" for ANY reason, default to
College calibration. This is the safest default — it matches current
pipeline behavior and avoids accidentally patronizing an older athlete
or overwhelming a younger one.

2B. PPD / ABI / ADVERSITY FIELD ROUTING
------------------------------------------------------------

PPD COACHING IMPLICATIONS — MESSAGE MAP ADDITIONS:

DESIGN DECISION (Q2): Weekly data takes priority over PPD baseline.
PPD coaching implications are PERSISTENT BASELINE GUIDANCE — they
represent what the intake data says the athlete's friction patterns
are. But weekly signals reflect the athlete's CURRENT STATE, which
may diverge from baseline patterns.

THREE-TIER ROUTING PRIORITY:

  Priority 1: WEEKLY SIGNALS
    Weekly tier, core_pattern, behavioral_root_cause, emotional_
    intensity, coach flags, composite score bands. These reflect
    what actually happened THIS WEEK.

  Priority 2: PPD COACHING IMPLICATIONS
    ppd_top_3.coaching_implication lines. These are baseline
    recommendations that apply when weekly signals are CONSISTENT
    with the baseline pattern or when weekly signals are neutral
    (no strong directional signal).

  Priority 3: DEFAULT COACHING APPROACH
    Standard coaching approach based on growth_phase, consistency_
    profile, and existing v9.3 field routing.

WHEN PPD IMPLICATIONS ACTIVATE (Priority 2):
  - Weekly data shows derailer activation or Partially Aligned status
    AND the derailer pattern matches the PPD primary problem
  - Weekly data is neutral (no strong positive or negative signal)
    and PPD implications provide useful baseline direction
  - Coaching Focus Areas in Deep Dive — PPD implications can
    always inform focus area selection as background context

WHEN PPD IMPLICATIONS ARE OVERRIDDEN (Priority 1 takes over):
  - Athlete had a breakthrough week with no derailer activation —
    coaching leads with recognition, not baseline friction patterns
  - Weekly signals point to a DIFFERENT pattern than PPD predicted
    (e.g., PPD says Overthinking Loop but this week shows Discipline
    Gap behavior) — coach to the current week, not the baseline
  - Emotional intensity is High — empathy leads, and PPD coaching
    implications are secondary to emotional acknowledgment

PPD FIELD ROUTING TO COACHING MESSAGE:

  ppd_primary_problem →
    Does NOT appear as named text. Informs the coach's internal
    understanding of the athlete's primary friction pattern.
    Routes to: Drift Identification (when derailer activates,
    the coaching language addresses the specific friction pattern
    rather than generic drift language)

  ppd_top_3.coaching_implication →
    Routes to: Coaching Focus Areas subsection targets
    Example: If coaching_implication = "Reset reps + next-play
    language" and the week shows derailer activation, the micro-
    commitment targets reset-based behavioral actions.
    Does NOT appear as quoted text. Informs coaching direction.

  ppd_tie_breaker_classification →
    Routes to: Tone calibration across entire message
    Avoidance-framed athlete: Coaching emphasizes safety of
      controllables. "This is something you can control" framing.
      Reduce high-stakes language. Frame forward actions as
      low-risk, high-return.
    Proving-framed athlete: Coaching separates worth from outcomes.
      "Your value isn't determined by this result" framing. Anchor
      identity in preparation quality, not external validation.
    Obligation-framed athlete: Coaching redirects from others'
      expectations to self-defined standards. "What do YOU want
      from this?" framing. Reduce references to coaches, parents,
      team expectations.
    Approach-framed athlete: Standard coaching tone. Athlete
      already frames pressure productively. No special calibration
      needed — reinforce existing approach framing.
    Dual classification: Apply both calibrations. When they
      conflict (e.g., Avoidance + Proving), prioritize the
      classification matching the current week's behavioral
      pattern.

PPD FIELD ROUTING TO DEEP DIVE:

  ppd_primary_problem →
    Routes to: Growth Areas — when derailer activates, the growth
    section can reference the specific friction type without
    naming the PPD bucket. Example: instead of "Mistake Recovery
    Lag was activated," write "The pattern of compounding errors
    appeared again this week."

  ppd_top_3 (all three) →
    Routes to: Coaching Focus Areas — all three PPD coaching
    implications are available as background context for focus
    area selection. The #1 implication has highest priority for
    derailer-activation weeks. #2 and #3 may inform secondary
    focus areas.

  ppd_tie_breaker_classification →
    Routes to: Mindset Summary — the athlete's pressure framing
    (Avoidance/Proving/Obligation/Approach) contextualizes the
    growth phase narrative. Example: "Grace's pressure framing
    has shifted from avoidance-oriented to approach-oriented over
    five weeks" would be expressed as: "The way Grace frames
    pressure has shifted — from protecting against failure to
    preparing for opportunity."

ABI PRIMARY EMPHASIS — ROUTING:

  abi_primary_emphasis →
    Routes to: Strength Signals + Drift Identification +
    Coaching Focus Areas

    When abi_primary_emphasis pillars ALIGN with current week's
    strength signals: Reinforce. "Your [pillar behavior] this
    week was exactly the kind of [pillar concept] that builds
    consistency."

    When abi_primary_emphasis pillars ALIGN with current week's
    drift pattern: Name directly. "The area where you have the
    most room to grow — [pillar behavior description] — is
    exactly where this week's drift showed up."

    When abi_primary_emphasis pillars are UNRELATED to current
    week's data: Background context only. Do not force a pillar-
    based coaching point that doesn't connect to the week's data.

    Pillar-to-behavior translations (NEVER use pillar names):
      Ownership → self-directed action, driving your own process,
        initiating without being told
      Composure → staying steady under pressure, recovering from
        setbacks, keeping your response consistent
      Focus → staying locked in during performance, filtering out
        distractions, maintaining attention
      Structure → building routines, maintaining daily rhythms,
        following preparation patterns

    Example: abi_primary_emphasis = ["Composure", "Ownership"]
      WRONG: "Your ABI shows Composure and Ownership as emphasis
      areas."
      CORRECT: "The two areas where you have the most room to
      grow — how you respond under pressure and how much you
      drive your own process — both showed up this week."

ADVERSITY RESPONSE PATTERN — ROUTING:

DESIGN DECISION (Q5): Adversity response pattern descriptions ARE
permitted in coaching language. Unlike PPD bucket names (pipeline
jargon), these are athlete-selected natural language from intake Q20.
The athlete chose "Freeze and overthink" or "Reset and refocus" as
their own self-description. Reflecting this language back is coaching,
not leakage.

  adversity_response_pattern →
    Routes to: Drift Identification + Growth Areas (ONLY during
    derailer-activation weeks or Partially Aligned status)

    NOT routed during non-derailer weeks. The adversity response
    pattern is relevant only when adversity is present.

    Routing logic:
      "Push harder" →
        When derailer activates: Coach toward measured response.
        "Your instinct when things get hard is to push harder —
        and sometimes that works. But this week, pushing harder
        without adjusting direction burned energy without changing
        the result. Channel that intensity into your controllables."

      "Pull back and protect" →
        When derailer activates: Coach toward re-engagement.
        "When things got tough, you pulled back. That's a
        protection instinct — it makes sense. But pulling back
        means you're not in the game anymore. The goal isn't to
        fight harder. It's to stay present."

      "Freeze and overthink" →
        When derailer activates: Coach toward action.
        "When things got hard, your instinct was to freeze and
        overthink — replay it, analyze it, try to figure it out
        before acting. This week, the goal is to move first and
        process second. Action breaks the loop."

      "Reset and refocus" →
        When derailer activates: Reinforce existing pattern.
        "Your instinct is to reset and refocus — and this week
        that's exactly what showed up. Even under pressure, you
        came back to your controllables."

    When adversity_response_pattern is "not available" (CF):
      No routing. Standard derailer coaching language per v1.5.

3A. FIELD CLASSIFICATION
------------------------------------------------------------

INFORM-ONLY FIELDS (raw content never reproduced, but informs
better institutional output):

  competitive_level →
    Informs Section 1 (Weekly Performance Classification) as
    contextual enrichment. Example: "This classification reflects
    performance patterns at the collegiate competitive level."
    Does NOT change any determination logic or thresholds.

  abi_primary_emphasis →
    Informs Section 8 (Key Performance Signal) and Section 9
    (Coach Context Cue). When KPS selection is ambiguous between
    two equally strong signals, abi_primary_emphasis breaks the
    tie toward the pillar-aligned signal.
    Does NOT appear as "ABI Primary Emphasis is Composure."
    Appears as: "Self-directed preparation consistency" (when
    Ownership is emphasis), "Pressure response stability" (when
    Composure is emphasis), etc.

  adversity_response_pattern →
    Informs Section 9 (Coach Context Cue) during derailer-
    activation or Partially Aligned weeks. Helps the coach
    context cue name the athlete's likely response pattern
    without exposing the pipeline classification.
    Example: "Athlete's typical adversity response involves
    increased effort intensity rather than strategic adjustment"
    (translates "Push harder" without naming it)

PROHIBITED FIELDS (NEVER surface in institutional output in any
form, even translated):

  ppd_primary_problem — PPD bucket names are pipeline-internal
    classification labels. They are not coaching vocabulary. The
    baseline_primary_derailer in baseline_profile already carries
    the athlete-facing version of this information.
    PROHIBITED: "Primary problem: Mistake Recovery Lag"
    PROHIBITED: "PPD indicates overthinking patterns"
    REASON: Coaching staff should not see named problem
    classifications. The Weekly Performance Insight already
    addresses behavioral patterns through compliance-safe language.

  ppd_tie_breaker_classification — Internal scoring methodology
    artifact. "Avoidance-framed" or "Proving-framed" are pipeline
    labels applied to athlete pressure language by the scoring
    agent. Not appropriate for institutional output.
    PROHIBITED: "Athlete's pressure framing is Avoidance-type"
    REASON: This is a clinical-adjacent classification that
    could influence how coaching staff perceive the athlete.

  ppd_top_3 — The full array including bucket names, scores, and
    coaching implications. Scores are internal pipeline data.
    Bucket names are pipeline classifications. Coaching
    implications are Stage 3 routing directives, not institutional
    content.
    PROHIBITED: "PPD top 3: Pressure Reactivity (7), Overthinking
    Loop (6), Confidence Volatility (4)"
    REASON: Named problem rankings with numeric scores would
    fundamentally alter how coaching staff perceive the athlete.

  baseline_season_phase — Point-in-time intake data that goes
    stale. Surfacing it would present potentially outdated season
    information to coaching staff. The live current_season_phase
    (once implemented) is the appropriate source for Stage 4
    Season Phase Overlay.
    PROHIBITED: "Baseline season phase: Pre-Season"
    REASON: Stale data presented as current context creates
    misinformed coaching decisions.

CONTEXT-ONLY FIELD (used for internal routing, never surfaced):

  input_source — Used for dual-mode detection (App vs CF). Never
    mentioned in output. Same status as execution_behavior_signals.
    input_source.

3B. PER-SECTION ROUTING UPDATES
------------------------------------------------------------

Section 1 (Weekly Performance Classification):

  the executive summary sentence may include competitive level

Section 3 (WTD + Bullseye):
  No baseline_intake_profile routing. These are weekly behavioral

Section 4 (Recommitment & Response Signal):
  No baseline_intake_profile routing. Recommitment is event-

Section 5 (Confidence Momentum):
  No baseline_intake_profile routing. Confidence determined by
  weekly signals and self-ratings alignment.

Section 6 (Stress Load):
  No baseline_intake_profile routing. Stress Load is weekly
  contextual assessment.

Section 7 (Growth Phase):
  No baseline_intake_profile routing. Growth Phase is
  longitudinal behavioral classification.

Section 8 (Key Performance Signal):
  abi_primary_emphasis MAY serve as KPS tie-breaker when two
  signals are equally strong. Priority: existing KPS selection
  logic > abi_primary_emphasis tie-break > default selection.
  The tie-break translates pillar emphasis into compliance-safe
  signal labels:
    Ownership emphasis → "Self-Directed Engagement Consistency"

4A. LEAKAGE AUDIT CRITERIA DESIGN

DESIGN DECISION (Q3): EXPAND Criterion 13 from "Execution Signal

RATIONALE:

  - The 6-step check logic structure accommodates expansion
  - Adding a Category F (Intake Profile Terminology) alongside
    existing Category E (Execution Signal Terminology) keeps the
    audit taxonomy clean
  - Two near-identical "leakage" criteria would create audit
    redundancy and confusion about which criterion catches which
    violation

CRITERION 13 — EXPANDED SCOPE:

Current title: "Execution Signal Leakage Compliance"
New title: "Pipeline Data Leakage Compliance"

Current scope: Execution behavior signals only
New scope: Execution behavior signals + baseline intake profile data

The 6-step check logic expands as follows:

  Step 1: Read input_source (unchanged)

  Step 2: Core Foundation check (EXPANDED)
    Current: Scan for execution-signal-derived language
    Added: Also scan for intake-profile-derived language.
    Core Foundation athletes have no PPD/ABI computed data.
    Any reference to PPD-style friction patterns, ABI-style
    pillar assessments, or adversity response classifications
    is leakage because there is no intake profile data to
    derive it from.
    NOTE: Existing baseline_profile fields (identity claim,
    hinge habit, primary derailer) are NOT affected — these
    exist for CF athletes and are properly routed.

  Step 3: Raw Metric Scan (unchanged — Category E5-E6)

  Step 4: Composite Score/Flag/Alignment Scan (EXPANDED)
    Current: Scan for Category E1-E4, E7
    Added: Also scan for Category F1-F5 (see 4C below)

  Step 5: Surveillance Framing Scan (EXPANDED)
    Current: 10 surveillance framing patterns
    Added: Pipeline classification framing patterns:
      "Your primary problem is..."
      "Based on your intake assessment..."
      "Your baseline index shows..."
      "According to your intake profile..."
      "Your pillar scores indicate..."
      "Your friction pattern suggests..."
    These expose the athlete to the existence of internal
    classification systems.

  Step 6: Context-Dependent Judgment (EXPANDED)
    Added to PERMITTED list:
      a) Pillar concepts used as coaching language:
        "Take ownership of your routine" (coaching)

      b) Named ABI references:
        "Your Composure pillar is your weakest area"
        "Ownership Index is Moderate"
      c) Named classification references:
        "Your adversity response pattern is Freeze and overthink"
        (as a named pipeline classification, not coaching language)
        "Your tie-breaker classification is Avoidance"

    Reasonable reader test (expanded):
      Current: "Could a reasonable reader identify that the
      observation is derived from quantified app usage data?"
      Added: "Could a reasonable reader identify that the
      observation is derived from a formal intake assessment
      scoring system?"
      If yes on either → leakage. If no on both → permitted.

6. DESIGN QUESTION RESOLUTIONS
============================================================

Q1: COMPETITIVE LEVEL CALIBRATION — RULE STRUCTURE

  ANSWER: Option (c) — Modifier-based system.

  A base calibration table defines 5 parameters (vocabulary
  complexity, directness level, accountability framing, concept
  naming, sentence complexity) across 5 competitive levels (Youth
  through Professional). These parameters interact with existing
  contextual modifiers (emotional_intensity, coach flags, self-
  ratings alignment) to produce nuanced calibration.

  RATIONALE: A High EI + Youth athlete needs warm empathy with
  simple language. A High EI + Professional athlete needs brief
  acknowledgment with direct pattern-naming. A simple lookup table
  cannot capture these interactions. The modifier approach reuses
  existing interaction infrastructure (the emotional_intensity,
  coach flag, and self-ratings alignment rules already define
  behavioral patterns — competitive level adjusts HOW those
  patterns are expressed).

  DEFAULT: College (matches all current athlete output).
  CF FALLBACK: College (zero degradation).

  See Section 2A for full design.

------------------------------------------------------------

Q2: PPD COACHING IMPLICATIONS — ROUTING PRIORITY

  ANSWER: Weekly data takes priority.

  Three-tier priority system:
    Priority 1: Weekly signals (what happened THIS WEEK)
    Priority 2: PPD coaching implications (baseline guidance)
    Priority 3: Default coaching approach (standard rules)

  PPD coaching implications activate when weekly signals are
  consistent with baseline patterns or when weekly signals are
  neutral. They are overridden when the athlete had a breakthrough
  week, when weekly signals point to a different pattern, or when
  emotional intensity requires empathy-first coaching.

  RATIONALE: PPD represents what the athlete THOUGHT their
  problems were at intake. Weekly data represents what ACTUALLY
  happened. An athlete whose PPD says "Mistake Recovery Lag" but
  who had a perfect week with no errors should receive recognition
  coaching, not mistake-recovery coaching. The baseline is a
  starting point, not a permanent label.

  See Section 2B for full routing design.

------------------------------------------------------------

Q3: CRITERION 13 EXPANSION VS NEW CRITERION 14

  ANSWER: Expand Criterion 13. No new criterion.

  Criterion 13 title changes from "Execution Signal Leakage
  Compliance" to "Pipeline Data Leakage Compliance." Category F
  (Intake Profile Terminology) added alongside Category E
  (Execution Signal Terminology).

  RATIONALE: The core question is identical — "does internal
  pipeline data appear in athlete-facing output?" PPD/ABI leakage
  and execution signal leakage are the same category of compliance
  concern. Two near-identical criteria create audit redundancy.
  Category F integrates cleanly into the existing 6-step check
  logic with minimal structural change.

  See Section 4A for full design.

------------------------------------------------------------

Q4: LIVE SEASON PHASE — CF ATHLETE HANDLING

  ANSWER: Option (a) — "not available"

  Consistent with all other app-only fields. CF athletes do not
  have structured weekly check-in data. No inference from recap
  content (adds classification complexity for a transitional
  population). No coach-provided setting (adds non-athlete data
  source complexity).

  RATIONALE: Consistency. Every other app-only field uses the
  same "not available" pattern for CF athletes. Adding a special
  case for season phase would break the clean dual-mode
  architecture. CF is transitional (<20 athletes, phasing out).

  See Section 5E for full handling.

------------------------------------------------------------

Q5: ADVERSITY RESPONSE PATTERN — COACHING LANGUAGE

  ANSWER: Adversity response descriptions ARE permitted in
  coaching language.

  Unlike PPD bucket names (pipeline jargon like "Mistake Recovery
  Lag"), adversity response descriptions are athlete-selected
  natural language from intake Q20. The athlete chose "Freeze and
  overthink" or "Reset and refocus" as their own self-description.
  Reflecting this language is coaching, not leakage.

  BOUNDARY: Using the description AS coaching language ("When
  things get hard, your instinct is to freeze and overthink")
  is permitted. Using it AS a named pipeline classification
  ("Your adversity response pattern is Freeze and overthink")
  is prohibited. The distinction is framing: coaching observation
  vs. named classification.
