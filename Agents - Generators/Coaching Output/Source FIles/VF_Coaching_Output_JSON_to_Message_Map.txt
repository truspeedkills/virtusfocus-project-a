
VIRTUSFOCUS – COACHING OUTPUT
JSON → MESSAGE MAPPING LOGIC

PURPOSE
Standardizes how Interpretation JSON fields are translated into coaching outputs.
Covers both v9.3 fields and execution_behavior_signals routing.

------------------------------------------------------------
COACHING MESSAGE (ATHLETE-FACING)
------------------------------------------------------------

Recognition:
Pull from athlete_state + positive signals in coach_insights.
Execution enrichment (App athletes):
If ownership_index.band = High → recognize self-driven engagement.
If follow_through_score.band = Strong → recognize challenge execution.
If recovery_score.band = Strong (calculated, not fixed) → recognize
  post-disruption resilience.

Prior Commitment Check:
If prior_commitment_result.execution_status exists and is not "No Prior Week":
- Include 1 sentence acknowledging the prior commitment and its result.
- If Completed: reinforce with brief recognition.
- If Partial/Not Attempted: name it without judgment, then bridge
  to the current week's micro_commitment.
- Do not dwell. One sentence maximum.

Strength Signals:
Use weekly_tier + consistency_profile + control_quality.
Execution enrichment (App athletes):
ownership_index (High) → self-initiated engagement evidence.
follow_through_score (Strong) → Mindset Challenge execution evidence.
rhythm_score (Stable) → consistent daily engagement timing.
review_quality_score (High) → thorough, balanced review engagement.
recovery_score (Strong, calculated) → post-disruption execution resilience.

Drift Identification:
Use core_pattern + bullseye primary_focus_ring + recommitment_signal.
Execution enrichment (App athletes):
drift_score (Early) → name emerging engagement compression. Subtle, not direct.
drift_score (Active) → directly address behavioral disengagement.
reactivity_risk_score (Moderate) → note volatile execution subtly.
reactivity_risk_score (Elevated) → directly address event-driven disruption.
recovery_score (Low, calculated) → address post-disruption execution gap.
recovery_score (Pending Data) → name the open thread from end-of-week disruption.
self_ratings_alignment (Conflated) → gently ground self-perception
  against execution evidence.

Identity Anchor:
Use growth_phase to frame identity development.
Execution enrichment (App athletes):
self_ratings_alignment (Undervalued) → elevate identity with execution evidence.
Positive execution bands → ground identity in behavioral evidence.

Weekly Action:
Must directly equal micro_commitment.action + when + measurement.
Do not modify action language.
Coach flag impact: attention/action flags may redirect micro_commitment
  targeting (see Coach Flag Routing in Execution Signal Routing Rules).

Identity Thread (2–3 sentences):
If baseline_profile exists, connect:
baseline_identity_claim + baseline_hinge_habit → core_pattern → most_important_controllable.
Execution enrichment (App athletes):
When positive execution bands reinforce the identity thread, ground
  the baseline → current week connection in execution evidence.

Athlete Voice Mirror:
If athlete_voice.key_phrase or athlete_voice.achievement_statement
contains a significant statement:
- Include 1 sentence that mirrors the athlete's own language back.
- Use their exact words or a close paraphrase.
- Connect it to the identity thread or the most_important_controllable.
Do not use more than 1 athlete-voice mirror per message.

Upcoming Context Anchor:
If upcoming_context.scheduled_competition is not "insufficient data":
- Include 1 sentence connecting the micro_commitment to the
  specific upcoming event.
- Frame as: "Here is your tool for [event]."

Trendline Decision (1–2 sentences):
If longitudinal_metrics exists, reference:
rolling_win_rate_4wk + volatility_index_4wk + fatigue_load_indicator
to frame: "keep building" vs "tighten guardrails".

------------------------------------------------------------
DEEP DIVE ANALYSIS
------------------------------------------------------------

Weekly Overview:
Use weekly_tier + consistency_profile + recommitment_signal.
Execution enrichment (App athletes):
Include execution_pattern_summary as behavioral context in the
  weekly theme paragraph.
If drift_score.band != None → frame engagement quality alongside
  consistency_profile.

Strength Sections:
Draw from coach_insights.key_habit_signal + win_the_day signals + control_quality.
Execution enrichment (App athletes):
ownership_index (High) → self-driven engagement subsection.
follow_through_score (Strong) → challenge execution subsection.
rhythm_score (Stable) → daily timing consistency subsection.
review_quality_score (High) → thorough engagement subsection.
recovery_score (Strong, calculated) → post-disruption resilience subsection.
Use composite score bands as behavioral evidence. Do not cite numeric scores.

Growth Sections:
Use behavioral_root_cause + drift indicators + recommitment_signal.
Execution enrichment (App athletes):
drift_score (Early/Active) → engagement erosion pattern.
rhythm_score (Variable/Disrupted) → timing inconsistency pattern.
reactivity_risk_score (Moderate/Elevated) → volatile execution pattern.
review_quality_score (Low/Moderate) → review engagement gap.
follow_through_score (Weak/Moderate) → challenge execution gap.
ownership_index (Low/Moderate) → self-initiation gap.
recovery_score (Low, calculated) → post-disruption execution gap.
Use composite score bands as coaching cue evidence. Do not cite numeric scores.

Mindset Summary:
Use growth_phase + reflection_quality_score.
Execution enrichment (App athletes):
self_ratings_alignment:
  Conflated → perception-reality assessment. Ground self-view
    against execution evidence. No clinical language.
  Undervalued → perception-reality assessment. Elevate self-view
    with execution evidence.
  Aligned → no special treatment.

Narrative Arc:
If narrative_arc fields are populated (not "insufficient data"):
- Include arc_summary as a 2-3 sentence developmental summary.
- Name milestone_this_week as a specific callout.
- Reference open_thread in coaching focus areas if relevant.

Coaching Focus Areas:
Must include most_important_controllable + micro_commitment.
Execution enrichment (App athletes):
coach_flags (attention severity) → micro_commitment targets the
  flagged behavioral area.
coach_flags (action severity) → coaching focus areas lead with the
  flagged pattern as dominant priority.
If drift_score = Active or reactivity_risk_score = Elevated →
  include as specific coaching focus priority.
If recovery_score = Pending Data → include the open thread from
  end-of-week disruption as a forward-looking focus.

Upcoming Preparation:
If upcoming_context.scheduled_competition is not "insufficient data":
- Include one coaching focus area that specifically prepares
  the athlete for the named upcoming event.
- Connect the micro_commitment to the competition context.

Emotional Calibration Rule:
If emotional_intensity = "High":
- Lead coaching message with empathetic acknowledgment.
- Name the specific emotional experience from the recap.
- Reduce framework language; increase human language.
If emotional_intensity = "Low":
- Standard coaching tone. No emotional preamble needed.

Baseline vs Now:
Use baseline_alignment.alignment_status + alignment_notes
to explain whether this week matches the athlete's baseline standards.

Trendline:
Use longitudinal_metrics to describe:
- pace (win rate)
- stability (volatility + controllable stability)
- load (fatigue indicator)
Then prescribe focus areas already in JSON:
most_important_controllable + micro_commitment.

------------------------------------------------------------
PARENT COACHING MESSAGE (4 SECTIONS)
------------------------------------------------------------

The Parent Coaching Message is generated AFTER the Athlete
Coaching Message and Deep Dive. It is a standalone output
for the parent — thematically aligned with the Athlete CM
(same JSON, same week), but NEVER directly references the
Athlete CM, Deep Dive, or micro-commitment by name.

Voice: Coach Arron, third-person register. Informational +
support guidance. Does NOT coach the parent. Updates and
suggests how to support.

Length: 150-250 words total (4 sections, ~35-65 words each).

Generation trigger: D2C Premium athletes with
parent_inclusion = true ONLY.

----

SECTION 1 — Weekly Context:
Purpose: What the athlete is building this week. High-level
theme, tier-level framing without naming the tier.

Pull from:
- weekly_tier → frame as "building," "steady," or "working
  through a tough week" (never name Growth/Mixed/Reset)
- core_pattern → one-sentence behavioral theme
- consistency_profile → stability framing
- growth_phase → developmental framing (subtle, no phase
  name — see DQ4 rule below)
- current_season_phase → seasonal context when natural
  ("heading into competition," "in the offseason")

Do NOT include:
- Weekly tier names (Growth, Mixed, Reset)
- Growth phase names (Developing, Consistent, etc.)
- Consistency classifications (Stable Positive, Variable, etc.)
- Any longitudinal metric values

----

SECTION 2 — What Stood Out:
Purpose: One specific strength or development signal the
parent can recognize and reinforce at home. Behavioral,
not technical.

Pull from:
- coach_insights.key_habit_signal → translate to parent-
  recognizable behavioral description
- control_quality → what the athlete is controlling well
- athlete_voice.key_phrase or achievement_statement → if the
  athlete said something the parent would value hearing,
  include it as a direct quote. Use sparingly — only when
  it adds genuine warmth.
- win_the_day.daily_distribution → frame effort level
  without citing numbers ("full effort all week" or
  "pushed through a harder stretch")

Do NOT include:
- Win/Neutral/Missed day counts
- WTD scores or daily breakdowns
- Bullseye ring analysis
- Reflection quality scores
- Coach flag language

----

SECTION 3 — What's Ahead:
Purpose: Forward-looking context so the parent can create
the right environment. Not the micro-commitment itself.

Pull from:
- upcoming_context.scheduled_competition → name specific
  upcoming events ("NC trip this week")
- upcoming_context.anticipated_stressor → thematic framing
  only ("back-to-back travel," "competition environment").
  Do NOT reproduce stressor language verbatim.
- micro_commitment.action → THEMATIC REFERENCE ONLY. Frame
  the direction of work without quoting the commitment.
  WRONG: "Grace's goal is to complete a deliberate warmup
  before each game."
  CORRECT: "The coaching focus this week is on preparation
  quality heading into competition."
- narrative_arc.open_thread → if a longitudinal thread is
  relevant to what the parent should know about upcoming
  context, reference it thematically.

Do NOT include:
- The micro-commitment text itself
- Narrative arc technical language
- Derailer terminology
- Prior commitment results

DERAILER-ACTIVATION WEEKS (DQ3 — indirect):
When baseline_alignment.alignment_status = "Partially
Aligned" or "Not Aligned":
- "What's Ahead" notes the athlete is working on something
  specific without naming the derailer.
  CORRECT: "The coaching focus this week is on staying steady
  in a challenging stretch."
  WRONG: "Grace's primary derailer — loss of confidence —
  activated this week."
- "How You Can Support" adjusts to provide space/stability.
  The parent should sense the coaching is targeting something
  without knowing the specifics.

----

SECTION 4 — How You Can Support:
Purpose: One concrete, actionable support suggestion for the
parent. Drawn from the week's data, not generic. This is the
anchor section and differentiator.

PRIMARY CALIBRATION — Q23 home_conversation_pattern:
  "Mostly supportive":
    Standard support suggestion. Reinforce what the athlete
    is doing well. Suggest specific recognition language the
    parent can use at home.
    Tone: "Here's something you can say..."
  "A lot of analysis or advice":
    Redirect toward listening. Suggest the parent ask instead
    of advise. Frame support as creating space for the athlete
    to process independently.
    Tone: "This week, try asking instead of advising..."
  "Pressure to perform":
    Buffer-oriented suggestion. Protect the athlete's mental
    space. Suggest reducing performance conversation at home.
    Frame support as NOT discussing results.
    Tone: "The most helpful thing this week is to let the
    results go unmentioned..."
  "We don't discuss performance much":
    Open-door suggestion. Suggest one low-pressure way to
    show interest without initiating a performance
    conversation. Frame support as presence, not conversation.
    Tone: "You don't need to bring up the game. Just being
    there is the signal..."

SECONDARY MODIFIER — Q25 support_network_primary:
  "A parent" or "No one":
    The parent may be the athlete's primary emotional resource
    (or the athlete has none). Support suggestions can include
    emotional availability language — "being someone they can
    talk to" type guidance.
  "A teammate" / "My coach" / "A mentor":
    The athlete's primary support is outside the home.
    Support suggestions focus on environment — creating space,
    reducing pressure, being present without needing to be the
    processing partner.

ADDITIONAL PULL FROM:
- growth_phase → adjust suggestion complexity.
  Developing: simpler, routine-oriented suggestions.
  Consistent: reinforce the athlete's independence.
  Leadership: celebrate and stay out of the way.
- emotional_intensity → see EI handling below.
- core_pattern → ground the suggestion in the week's theme.

EMOTIONAL INTENSITY HANDLING (DQ2 — light touch):
When emotional_intensity = "High":
- ONLY "How You Can Support" adjusts.
- Support suggestion becomes warmer and more protective.
- Frame: "This was a harder week emotionally. The best
  support right now is..." + specific protective action.
- Other sections (1-3) stay informational. Do NOT signal
  emotional weight in Weekly Context, What Stood Out, or
  What's Ahead.

When emotional_intensity = "Low" or "Moderate":
- Standard support tone per Q23/Q25 calibration.
- No emotional modifier applied.

GROWTH PHASE ADVANCEMENT (DQ4 — subtle):
When growth_phase advances from one level to the next:
- Do NOT name the phase transition ("Developing to
  Consistent").
- DO reflect the milestone in tone and language.
  CORRECT: "Grace has shown real consistency in her routine
  over the past several weeks — that kind of steadiness is
  worth recognizing."
  WRONG: "Grace advanced from Developing to Consistent this
  week."
- Phase advancement may appear in Section 1 (Weekly Context)
  or Section 2 (What Stood Out), whichever fits the week's
  theme more naturally.

INSUFFICIENT DATA / EARLY WEEKS:
Week 1:
- Section 1: Welcome tone. "This is Grace's first week in
  the VirtusFocus program."
- Section 2: Frame initial engagement as the signal.
  "Showing up and completing the first week is itself a
  meaningful step."
- Section 3: "The early weeks are about building a baseline
  and getting comfortable with the daily routine."
- Section 4: Support suggestion focused on encouraging the
  process. "The most helpful thing right now is to let Grace
  settle into the routine without asking how it's going."
Weeks 2-3:
- All sections functional but longitudinal data limited.
- Avoid longitudinal language ("over several weeks," "the
  pattern shows") until Week 4+.

----

PARENT CM — GLOBAL EXCLUSIONS:
Everything prohibited in the Athlete CM is also prohibited
in the Parent CM. Additionally:

Do NOT include in any Parent CM section:
- Weekly tier labels (Growth, Mixed, Reset)
- Growth phase names (Emerging, Developing, Consistent,
  Leadership)
- Consistency classifications
- Scoring bands or composite score language
- Coach flag language or flag-derived patterns
- Self-ratings alignment labels
- Execution behavior data (raw metrics, composite scores,
  flags, alignment)
- Baseline intake profile field names (PPD buckets, ABI
  pillar names, adversity classifications)
- The micro-commitment text (thematic reference only)
- Prior commitment results
- Drift identification language directed at the parent
- Accountability language directed at the parent
- Any reference to what the Athlete CM contains
- Any language that frames the parent as a monitor,
  tracker, or compliance agent
- Clinical language (same VirtusFocus-wide prohibition)

EXECUTION SIGNAL ROUTING (App athletes):
Same prohibited categories as Athlete CM — composite scores,
flags, raw metrics, self-ratings alignment labels NEVER
appear in Parent CM. Execution data may subtly inform the
support suggestion tone (e.g., if broad disengagement flags
fire, the support suggestion may lean toward "create space
and reduce pressure") but no execution terminology surfaces.

INTAKE PROFILE ROUTING (App athletes):
Same prohibited categories as Athlete CM — PPD bucket names,
ABI pillar names, adversity classifications NEVER surface.
Ecosystem fields (Q22-Q25) are used directly for Section 4
calibration (see above). Other intake fields inform direction
only, same as Athlete CM routing.

CORE FOUNDATION SKIP:
When input_source = "core_foundation":
- Skip all execution signal and intake profile routing in
  all Parent CM sections.
- Section 4 uses default support suggestion (no Q23/Q25
  calibration available — use generic supportive guidance).
- All other sections function normally using v9.3 fields.

------------------------------------------------------------
EXECUTION SIGNAL ROUTING RULES
------------------------------------------------------------

Core Foundation Athlete Rule:
When execution_behavior_signals.input_source = "core_foundation":
All execution signal routing is skipped. Use v9.3 fields only.
Output is identical to pre-v9.4 behavior.

Raw Metric Exclusion Rule:
Raw execution metrics NEVER appear in coaching output:
- Completion rates (decimal or percentage)
- Timing data (on-time counts, late counts)
- Backfill counts, reminder dependency counts
- Streak counts (current_streak, longest_streak)
- Component completion counts
- Question-level yes rates (q1-q5)
- Self-initiated vs. reminder-dependent counts
Only the following may inform coaching output:
- Composite score BANDS (categorical only, never numeric scores)
- Coach flag-driven behavioral patterns
- Self-ratings alignment classifications
- Execution pattern summary language

Coach Flag Routing:

  monitor flags:
    Coaching Message: Subtle tone adjustment in Drift Identification
      or Strength Signals. No flag naming. No metric citation.
    Deep Dive: Subtle framing adjustment in Growth Sections.
      No dedicated subsection. No flag naming.
    No impact on micro_commitment targeting.

  attention flags:
    Coaching Message: Directly address the behavioral pattern in
      Drift Identification. Name the pattern in athlete-appropriate
      language (not the flag_id or label).
    Deep Dive: Dedicated Growth subsection addressing the behavior.
    Coaching Focus Areas: micro_commitment targets the flagged area.
    If multiple attention flags fire, prioritize by relevance to
      core_pattern.

  action flags:
    Coaching Message: Leads with the flagged pattern as the
      dominant theme. Recognition pivots to acknowledge the
      pattern before moving to strengths. Drift Identification
      becomes the central section.
    Deep Dive: Weekly Overview leads with the flagged pattern.
      Growth Sections organized around the action flag.
      Coaching Focus Areas dominated by the flagged pattern.
    micro_commitment directly targets the action-flag behavior.
    If compound_disengagement fires: address the broad behavioral
      shift, not each individual score.

Self-Ratings Alignment Routing:
  Aligned: No special routing.
  Conflated:
    Coaching Message: Drift Identification gently grounds
      self-perception.
    Deep Dive: Mindset Summary includes perception-reality
      assessment.
    Language: "Let's look at what the week actually showed" or
      "Your sense of where you are doesn't quite match what
      the execution shows."
    Do NOT use: "overestimate," "inflated," "unrealistic."
  Undervalued:
    Coaching Message: Recognition and Identity Anchor elevate
      with execution evidence.
    Deep Dive: Mindset Summary includes perception-reality
      assessment.
    Language: "You're further along than you think" or
      "The execution tells a stronger story than you gave
      yourself credit for."

Execution Pattern Summary Routing:
  Routes to: Deep Dive Weekly Overview only.
  Use as behavioral context in the weekly theme paragraph.
  Do NOT use in Coaching Message (summary tone does not match
    direct coaching voice).

Moderate-Band Routing Heuristic:
  For higher-is-better scores (Ownership, Follow-Through, Rhythm,
  Review Quality) when band = Moderate:
    weekly_tier = Growth → route as Strength evidence.
    weekly_tier = Mixed or Reset → route as Growth evidence.
  For higher-is-worse scores (Drift, Reactivity):
    Moderate bands always route to Drift Identification / Growth.

------------------------------------------------------------
BASELINE INTAKE PROFILE ROUTING
------------------------------------------------------------

The baseline_intake_profile block contains 7 fields extracted
from the athlete's intake snapshot plus 1 live seasonal field.
These provide baseline coaching context that informs — but does
not override — weekly signal-driven coaching.

ppd_primary_problem:
  Routes to: Drift Identification (when derailer activates,
    informs specific friction pattern language).
  Use: When baseline_alignment.alignment_status = "Partially
    Aligned" or "Not Aligned" AND the week's core_pattern
    connects to the PPD primary problem, sharpen derailer
    language with intake-informed friction framing.
  Do NOT quote as text. Do NOT name bucket. Inform direction only.

ppd_top_3.coaching_implication:
  Routes to: Coaching Focus Areas (baseline coaching direction,
    overridden by weekly signals per 3-tier priority).
  Use: When weekly signals do not point to a clear coaching
    direction, use coaching implications from PPD top 3 as
    baseline guidance. When weekly signals are clear, PPD
    implications serve as background context only.
  Do NOT surface bucket names, scores, or "coaching_implication"
    as a field reference.

ppd_tie_breaker_classification:
  Routes to: Tone calibration across entire message.
  Use: Avoidance-framed = emphasize controllables and safety.
    Proving-framed = separate worth from outcomes.
    Obligation-framed = redirect from external expectations.
    Approach-framed = standard coaching tone.
  Applied globally — not to a specific section. Informs how
    coaching points are framed, not what they contain.
  Do NOT name classification in output.

abi_primary_emphasis:
  Routes to: Strength Signals + Drift Identification + Coaching
    Focus Areas.
  Use: Translate pillar emphasis to behavioral descriptions.
    Ownership → self-directed action language.
    Composure → staying steady under pressure language.
    Focus → maintaining attention language.
    Structure → building routines language.
  When emphasis aligns with strength signals: reinforce.
  When emphasis aligns with drift: name directly.
  When unrelated to current week: background context only.
  NEVER surface pillar names as labels.

adversity_response_pattern:
  Routes to: Drift Identification + Growth Areas.
  Use: Derailer-activation weeks ONLY. Informs per-pattern
    coaching language:
    "Blame the situation" → coach toward ownership.
    "Get frustrated" → coach toward channeling emotion into action.
    "Try to adjust right away" → reinforce adaptive problem-solving.
    "Reset and refocus" → reinforce existing reset pattern.
  Adversity response descriptions ARE permitted in coaching
    language (athlete-selected natural language).
  Named classifications ARE NOT permitted.
  When no derailer activation: no routing. Background only.

competitive_level:
  Routes to: All sections (modifier applied globally).
  Use: References Competitive Level Calibration Rule in
    Instructions (Section 19). Adjusts vocabulary complexity,
    directness, accountability framing, concept naming, and
    sentence complexity per competitive level.
  Default: College. CF fallback: College.

current_season_phase:
  Routes to: Contextual framing across all coaching sections.
  Use: Provides seasonal context:
    In-Season → competition-ready controllables.
    Pre-Season → preparation quality, habit building.
    Off-Season → skill development, self-directed growth.
    Post-Season → reflection, forward-looking goals.
    Year-Round Training → no seasonal contextualization.
    "not available" (CF) → no seasonal framing.

ROUTING RULES:
  - 3-tier priority: weekly signals > PPD baseline > default
    coaching direction.
  - PPD fields NEVER quoted as text — they inform direction.
  - ABI pillar names NEVER appear as labels — behavior
    translations only.
  - Adversity response descriptions ARE permitted in coaching
    language (athlete-selected natural language). Named
    classifications ARE NOT.
  - CF skip: when input_source = "core_foundation", skip ALL
    baseline_intake_profile and current_season_phase routing.

FIELDS THAT DO NOT APPEAR (intake profile):
  - ppd_top_3.score (numeric values)
  - ppd_all_scores (not in JSON — stays in snapshot)
  - baseline_intake_profile.input_source (routing field only)
  - baseline_season_phase (stale data — use current_season_phase)


============================================================
LONGITUDINAL PPD ROUTING
============================================================

TIER 2 REPLACEMENT LOGIC:

When longitudinal_ppd.status == "established":
  Tier 2 coaching implications sourced from execution_derived_top_3
  instead of static baseline_intake_profile.ppd_top_3.
  Each execution-derived bucket's coaching implication replaces
  the corresponding intake PPD implication.
  Execution-derived evidence_tier (Strong/Moderate/Weak) informs
  coaching confidence — stronger evidence = more direct coaching.

When longitudinal_ppd.status == "emerging":
  Tier 2 remains static intake PPD (UNCHANGED).
  Execution-derived signal is preliminary — used for subtle tone
  lean only.

When longitudinal_ppd.status == "insufficient data":
  Tier 2 remains static intake PPD (UNCHANGED).
  No longitudinal PPD influence.

TONE MODIFICATION ROUTING:

alignment_signal.overall == "Blind Spot":
  Coaching Message:
    → Drift Identification: grounding language for the unseen
      pattern. Behavioral description only — never name the bucket
      or comparison mechanism.
    → Growth Area: lean into blind spot bucket's behavioral domain.
    → Micro-commitment: target blind spot area when actionable.
  Deep Dive:
    → Mindset Summary: may reference behavioral shift direction
      without naming PPD/comparison.
    → Growth Areas: execution-derived primary bucket focus.

alignment_signal.overall == "Gap":
  Coaching Message:
    → Recognition: acknowledge athlete's partial awareness.
    → Growth Area: elevate underestimated area.
  Deep Dive:
    → Mindset Summary: may note widening or shifting emphasis areas.

alignment_signal.overall == "Aligned":
  → No routing change. Continue current direction.
  → Optional: subtle confirmation language in Coaching Message
    Recognition section.

EMERGING STATUS ROUTING:
  Tone modification available but SUBTLE only.
  No explicit grounding or elevation language.
  Micro-commitment MAY lean toward emerging pattern.
  No Tier 2 change.

PARENT COACHING MESSAGE:
  Longitudinal PPD does NOT route to Parent CM.
  Parent CM uses the same Tier 2 as Coaching Message (which may
  be execution-derived when established), but alignment-specific
  tone modification (grounding/elevation) does NOT apply.
  Parent sees the coaching direction without the tone mechanics.

CF SKIP RULE:
  When longitudinal_ppd.core_foundation_fallback == true:
  Skip all longitudinal PPD routing. Output identical to
  pre-longitudinal-PPD behavior.

RAW DATA EXCLUSION (NEVER SURFACE IN ANY COACHING OUTPUT):
  - Evidence tier names (Strong / Moderate / Weak / None /
    Context Insufficient)
  - primary_signals array content
  - Alignment signal labels (Aligned / Gap / Blind Spot)
  - Bucket names as named pipeline classifications
  - overlap_count, new_entries, dropped_from_intake
  - intake_score values
  - weeks_accumulated, measurement_window_start
  - Status labels (emerging / established / insufficient data)
  - "execution-derived PPD", "longitudinal PPD"
  - Any intake-to-execution comparison framing


============================================================
FORWARD ANCHOR ROUTING
============================================================

Source: forward_anchor block (top-level JSON field)
Privacy: Raw forward_anchor.text is athlete-private. NEVER
reproduce verbatim in any output. Same protection as journal
content. Derived classifications inform output through
behavioral translation only.

Applies ONLY when input_source = "app" AND
forward_anchor.submitted = true.

When input_source = "core_foundation" OR submitted = false:
  ALL forward anchor routing is skipped.

------------------------------------------------------------
CONTROLLABILITY CLASSIFICATION ROUTING
------------------------------------------------------------

Controllable:
  → Coaching Message Recognition: acknowledge the athlete's
    ability to identify controllable focus areas
  → Deep Dive Strength Signal: note center-ring thinking
    pattern as developmental indicator

Outcome-Oriented:
  → Coaching Message Drift Identification: gentle redirection
    toward controllable elements within the stated outcome
  → micro_commitment: steer toward controllable version of
    the athlete's stated intention (DQ1: Anchor-Informed)

Mixed:
  → Coaching Message Growth: acknowledge the controllable
    element while coaching toward greater specificity
  → micro_commitment: sharpen the controllable component
    (DQ1: Anchor-Informed)

------------------------------------------------------------
FORWARD ANCHOR ALIGNMENT ROUTING (DEFERRED SIGNAL)
------------------------------------------------------------

prior_forward_anchor.alignment_status:

Aligned:
  → Coaching Message Recognition: "The focus you set heading
    into this week showed up in your actions."
  → Deep Dive Strength Signal: note intention-to-action
    consistency as behavioral strength

Partial:
  → Coaching Message Growth: acknowledge intent while noting
    the gap — "You started strong with the focus you set."
  → Deep Dive Trendline: note partial follow-through as
    growth area

Misaligned:
  → Coaching Message Drift Identification: gentle observation
    that stated intention and behavior diverged — frame as
    awareness opportunity, not failure
  → micro_commitment: target the gap area (DQ1: Anchor-Informed
    — if athlete anchored on X but didn't follow through,
    commitment should help bridge that gap)

No Prior Week / insufficient data:
  → No routing. Skip forward anchor alignment references.

------------------------------------------------------------
FOCUS CONSISTENCY ROUTING
------------------------------------------------------------

Consistent:
  → Deep Dive Trendline: note sustained thematic focus as
    developmental stability indicator

Shifting:
  → Deep Dive Weekly Overview: contextual note only. Shifting
    is not inherently negative — may reflect appropriate
    adaptation.

Scattered:
  → Coaching Message Drift Identification: note that weekly
    focus has been unfocused — frame as opportunity to build
    sustained intention-setting habits
  → micro_commitment: may target intention-setting itself as
    the behavioral focus area

insufficient data:
  → No routing. Skip focus consistency references.

------------------------------------------------------------
MICRO-COMMITMENT INTERACTION (DQ1: ANCHOR-INFORMED)
------------------------------------------------------------

When forward_anchor.submitted = true:
  The micro_commitment BUILDS ON the athlete's Forward Anchor
  when the anchor is controllable and specific.

  Priority system (expanded from 3-tier to 4-tier):
    Priority 1 (Always): Weekly signals (coach flags, composite
      score concern bands, weekly core_pattern)
    Priority 1.5 (NEW): Forward Anchor — when no flags fire
      and anchor is controllable, anchor informs commitment topic
    Priority 2: PPD baseline coaching implications
    Priority 3: Default coaching direction per core_pattern

  When flags fire: flags drive topic, but anchor can influence
  framing even when flags dictate direction.

  Controllable anchor: Coach uses it as foundation and adds
  behavioral precision + measurement.
    Example: Anchor = "Respond calmly to mistakes."
    Commitment = "After your first mistake in practice, take
    one breath before your next rep."

  Outcome-Oriented anchor: Coach redirects toward a controllable
  element related to the athlete's stated outcome.
    Example: Anchor = "Win my next match."
    Commitment = "Before each match, complete your full warmup
    with deliberate focus on the first 3 drills."

  Mixed anchor: Coach sharpens the controllable component.
    Example: Anchor = "Play confident in big moments."
    Commitment = "In the first big moment of each game, focus
    on your breathing for 3 seconds before acting."

  When submitted = false: micro_commitment uses existing
  3-tier priority system unchanged.

------------------------------------------------------------
FORWARD ANCHOR IN PRIOR COMMITMENT BRIDGE
------------------------------------------------------------

The 1-3 sentence prior commitment bridge (existing rule) may
acknowledge BOTH the prior micro_commitment result AND the
prior Forward Anchor alignment when both exist.

Bridge priority:
  1. Prior micro_commitment result (always leads — coach-assigned)
  2. Prior Forward Anchor alignment (secondary — one sentence
     maximum, only when Aligned or Misaligned, skip for Partial
     or insufficient data to avoid bridge bloat)

Bridge must NOT exceed 3 sentences total including both elements.

------------------------------------------------------------
RAW TEXT EXCLUSION
------------------------------------------------------------

The following forward_anchor data NEVER appears in any
coaching output (Coaching Message, Deep Dive, Parent CM):
  - Verbatim forward_anchor.text (never quoted directly)
  - "Controllable" / "Outcome-Oriented" / "Mixed" as named
    classification labels
  - "Aligned" / "Partial" / "Misaligned" as named labels
  - "Consistent" / "Shifting" / "Scattered" as named labels
  - "forward_anchor" as a pipeline field name
  - Numeric alignment scores or rates

Permitted: Behavioral translation of classifications.
  WRONG: "Your forward anchor was classified as Outcome-Oriented."
  CORRECT: "The focus you set is reaching for a result — let's
  ground it in something you can control every day."

------------------------------------------------------------
PARENT COACHING MESSAGE — FORWARD ANCHOR HANDLING
------------------------------------------------------------

Forward Anchor MAY inform Parent CM Section 3 (What's Ahead)
thematically — same rule as micro_commitment (thematic
reference only, never direct quote).

When forward_anchor.submitted = true AND controllability =
Controllable:
  Section 3 may reference the direction of the athlete's
  self-set focus for the week without naming the anchor text.
  WRONG: "[Athlete] wrote 'respond calmly to mistakes' as
  their focus."
  CORRECT: "The coaching focus this week builds on [athlete]'s
  own intention around composure."

When Outcome-Oriented or Mixed:
  Do NOT reference in Parent CM. The redirection is a coaching
  interaction (Stages 3 athlete CM), not a parent concern.

When submitted = false or not available:
  No forward anchor reference in Parent CM.

All other Parent CM exclusion rules (Section 3 global
exclusions) continue to apply.


------------------------------------------------------------
GLOBAL RULES
------------------------------------------------------------

- Do not reinterpret JSON classifications.
- Do not introduce new themes not grounded in JSON.
- Maintain brand voice (calm, direct, structured).
- Reinforce controllables consistently.
- Execution signal sharpens existing sections. It does not add
  new sections or expand word targets.
- Baseline intake profile informs coaching direction. It does
  not add new sections or override weekly signals.
