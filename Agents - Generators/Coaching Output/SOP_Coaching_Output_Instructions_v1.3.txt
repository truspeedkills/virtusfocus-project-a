SECTION 5 — DEEP DIVE SECTION BOUNDARY RULE [NEW IN V1.3]
============================================================

The Deep Dive Coaching Analysis MUST contain ONLY the five
mandatory sections defined in VF_Deep_Dive_Coaching_Analysis_
Framework.txt:

  1. Weekly Overview
  2. Strengths & Wins (4-6 subsections)
  3. Opportunities & Growth Areas (3-5 subsections)
  4. Mindset Summary
  5. Coaching Focus Areas (Next Week)

Plus these supplementary analytical sections which ARE permitted
because they serve the athlete's understanding:

  - Longitudinal Trendline (when longitudinal_metrics is populated)
  - Baseline Alignment Note (contextualizes alignment_status)

DO NOT include any of the following sections in the Deep Dive:

  - "Coach Reinforcement Cue" or "Coach Reinforcement and Avoidance"
  - "Coach Avoidance Note"
  - "Risk and Compliance Note"
  - Any section framed as instructions TO a coach
  - Any section using third-person references to the athlete
    ("the athlete," "she," "her" in directive/reporting context)
  - Any section containing compliance-reporting language

WHY THIS RULE EXISTS:
The Deep Dive is a professional-grade report that the ATHLETE
may read. Every section must be written in second person ("you")
or use the athlete's name, and must maintain the VirtusFocus
coaching voice throughout. Coach-facing content belongs in
Stage 4 (Coach Insights Engine) — not in the Deep Dive.

If the JSON contains information in coach_insights or risk_flags
that is relevant to the athlete's development, weave it into the
five mandatory sections using athlete-appropriate language. Do not
create standalone sections to house it.

SECTION 6 — JSON FIELD ROUTING RULE [NEW IN V1.3]
============================================================

Not all JSON fields are destined for the Deep Dive. The following
routing rules determine where each field's content appears:

FIELDS THAT INFORM THE COACHING MESSAGE:
  - athlete_state → Recognition section
  - prior_commitment_result → Prior commitment bridge
  - athlete_voice → Athlete voice mirror
  - micro_commitment → ONE Action section
  - upcoming_context → Upcoming context anchor
  - emotional_intensity → Tone calibration
  - most_important_controllable → Action framing
  - baseline_profile → Identity thread

FIELDS THAT INFORM THE DEEP DIVE:
  - All of the above, plus:
  - win_the_day → Weekly Overview, Strengths
  - bullseye_analysis → Strengths, Growth Areas
  - core_pattern → Strengths, Growth Areas
  - behavioral_root_cause → Growth Areas
  - reflection_quality_score → Mindset Summary
  - growth_phase → Mindset Summary
  - narrative_arc → Mindset Summary
  - trend_analysis → Trendline section
  - longitudinal_metrics → Trendline section
  - baseline_alignment → Baseline Alignment Note

FIELDS THAT INFORM THE DEEP DIVE INDIRECTLY (USE FOR
ANALYTICAL CONTENT, NOT STANDALONE SECTIONS):
  - coach_insights.key_habit_signal → Use to inform
    Strengths & Wins subsection content
  - coach_insights.routine_execution_summary → Use to inform
    Weekly Overview and Strengths evidence
  - coach_insights.coach_reinforcement_cue → Use to inform
    Coaching Focus Areas bullet content
  - coach_insights.coach_avoidance_note → Use as a guardrail
    for what NOT to emphasize. Do NOT reproduce as a section.

FIELDS THAT DO NOT APPEAR IN THE DEEP DIVE:
  - risk_flags → Stage 4 only. Do not include in any form.
  - next_week_focus_question → Coaching Message only
    (appears as the closing reflection question).

SECTION 7 — CLINICAL LANGUAGE PROHIBITION [NEW IN V1.3]
============================================================

VirtusFocus is a non-clinical coaching service. No clinical,
therapeutic, or mental health terminology may appear in ANY
coaching output — Coaching Message, Deep Dive, or Parent Summary.

This is a compliance rule, not a style preference.

COMMONLY VIOLATED TERMS AND THEIR REPLACEMENTS:

  anxiety → "pressure," "nerves," "tension"
  clinical → never appropriate in coaching output
  therapeutic → never appropriate
  therapy → never appropriate
  diagnosis / disorder → never appropriate
  symptom / symptoms → "pattern," "signal"
  trauma → "difficult experience," "tough stretch"
  intervention → "adjustment," "coaching focus"
  treatment → "approach," "coaching direction"
  coping mechanism → "response pattern," "go-to move"
  emotional regulation → "composure," "staying centered"
  regression → "setback," "dip," "drift"
  trigger / triggered → "activated," "derailed"
  rumination → "overthinking," "dwelling"
  catastrophizing → "worst-case thinking," "spiral"
  self-sabotage → "getting in your own way"
  maladaptive → "unproductive," "limiting"
  avoidance behavior → "pulling back," "disengaging"
  deficit → "gap," "growth area"
  protocol → "system," "routine," "process"

CLINICAL FRAMING PATTERNS — ALSO PROHIBITED:
  - "symptoms of..."
  - "consistent with [condition]"
  - "presents as..."
  - "assessment indicates..."
  - "no clinical indicators present" (compliance language —
    belongs in Stage 4 risk reporting, never in coaching output)

NOTE: The Interpretation JSON's risk_flags field may contain
clinical terms (e.g., "no clinical indicators present"). This
is appropriate for a technical document. Do NOT pass this
language through to coaching output. If the risk_flags content
needs to inform your coaching approach, translate it into
performance and behavioral language.

SECTION 8 — COACHING MESSAGE LENGTH TARGET [NEW IN V1.3]
============================================================

The Coaching Message should target 250-450 words.

This range ensures the message is:
- Long enough to include all required elements (recognition,
  strength signals, drift, identity anchor, action, prior
  commitment bridge, athlete voice mirror, upcoming context)
- Short enough to feel like a coaching conversation, not a report

If the message exceeds 450 words, tighten:
- Consolidate strength signals (maximum 2 expanded)
- Remove redundant restatements of the same point
- Trim transition language
- Do NOT remove required mapped elements

If the message falls under 250 words, verify that all required
mapped elements are present. A short message that contains
everything is acceptable. A short message that is missing
required elements is not.

SECTION 9 — PRIOR COMMITMENT BRIDGE LENGTH RULE [NEW IN V1.3]
============================================================

When prior_commitment_result.execution_status is not "No Prior
Week," the coaching message must include a prior commitment
acknowledgment BEFORE introducing the new micro_commitment.

LENGTH GUIDELINE: 1-3 sentences.

Sentence 1: Name the prior commitment and its result.
Sentence 2 (if needed): Bridge to the current week's direction.
Sentence 3 (if needed): Short transition statement (e.g.,
  "Carry it forward" or "That's the foundation").

Do NOT exceed 3 sentences. The prior commitment bridge creates
continuity — it should not become the focus of the message.
Move forward after the bridge.
