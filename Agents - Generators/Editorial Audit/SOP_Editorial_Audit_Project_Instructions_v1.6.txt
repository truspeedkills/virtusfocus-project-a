PURPOSE
============================================================

This document defines the executable audit logic for the
VirtusFocus Editorial Audit Agent (Pipeline Stage 5).

It contains:
- The detailed check logic for all 14 audit criteria
- Pass/fail definitions for each criterion
- Auto-correction rules for Tier 2 and Tier 3 criteria
- Regeneration instruction format for Tier 1 failures
- The clinical language and pipeline data exclusion list
  (including execution signal, intake profile, and forward
  anchor terminology)
- The complete decision tree for PASS / AUTO-CORRECT / REJECT

This document is the companion to the Editorial Audit Master
Prompt v1.6, which defines the agent's identity, position,
inputs, outputs, and operating principles.


============================================================
AUDIT EXECUTION SEQUENCE
============================================================

PRE-CRITERIA MECHANICAL CHECKS:
Before running the 14 content criteria, run the following
mechanical scan:

  EM DASH SCAN: Search the full text of the Coaching Message,
  Deep Dive, and Parent Coaching Message (if present) for any
  occurrence of the em dash character. This is a simple
  character search (Unicode U+2014). If any em dash is found,
  it is a Tier 2 auto-correctable violation. See Section 13A
  for replacement rules. Log all instances for auto-correction
  in the Audit Summary.

  This scan runs before content analysis because it is a
  mechanical character check that does not require contextual
  judgment.

CONTENT CRITERIA:
Run all 14 criteria in order. Do not skip criteria.
Do not stop at the first failure.

Complete the full audit even if a Tier 1 failure is found
early. The Regeneration Instruction must report ALL failures,
not just the first one.

After all 14 criteria and mechanical checks are evaluated,
apply the decision tree (Section 17) to determine the final
result.

SECTION 1 - CRITERION 1: EMOTIONAL INTENSITY CALIBRATION
============================================================
Tier: 1 - MUST PASS
Applies to: Coaching Message
Failure action: REJECT AND REGENERATE

PURPOSE:
When an athlete is in significant emotional distress, the
coaching message must lead with human acknowledgment before
any framework language. Getting this wrong damages trust.

CHECK LOGIC:

Step 1: Read emotional_intensity from the Interpretation JSON.

Step 2: Apply the appropriate rule:

  IF emotional_intensity = "High":
    a) The first two paragraphs of the Coaching Message MUST
       name or acknowledge the specific emotional experience
       described in the JSON (from athlete_state, core_pattern,
       or behavioral_root_cause fields).
    b) The first two paragraphs MUST NOT contain any of the
       following framework terms:
      - "center ring" / "middle ring" / "outer ring"
      - "Bullseye"
      - "Win the Day"
      - "Growth tier"
      - "weekly tier"
      - "consistency profile"
      - "reflection quality"
      - "growth phase"
      - "controllables" (as a standalone noun)
    c) The overall message must have reduced framework density
       compared to a Low-intensity week. Framework terms should
       appear only in the second half of the message.
    d) The emotional acknowledgment must be specific, not
       generic. "This was a tough week" is insufficient.
       The message must name what was tough.

  IF emotional_intensity = "Moderate":
    a) The message may lead with recognition that includes
       both emotional context and strength signals.
    b) Framework language is permitted in the first paragraph
       but should not dominate.
    c) No specific structural requirement beyond the standard
       Coaching Message Framework.

  IF emotional_intensity = "Low":
    a) Standard coaching tone. No emotional preamble required.
    b) No additional calibration checks.

  IF emotional_intensity = "insufficient data":
    a) Treat as Low. Standard coaching tone applies.

PASS: Message calibration matches the emotional_intensity level.
FAIL: Message calibration does not match. Any High-intensity
      message that leads with framework language FAILS.

WHY THIS IS TIER 1:
A message that ignores emotional distress and leads with
"Your center ring stayed strong" when the athlete described
a breakdown episode creates emotional dismissal. This cannot
be fixed by editing a few words. The entire opening must be
restructured. Reject and regenerate.

SECTION 2 - CRITERION 2: CLINICAL LANGUAGE COMPLIANCE
============================================================
Tier: 1 - MUST PASS
Applies to: Coaching Message AND Deep Dive
Failure action: REJECT AND REGENERATE

PURPOSE:
VirtusFocus is a non-clinical coaching service. Clinical,
therapeutic, or mental health terminology in athlete-facing
content is a compliance violation. This is a business rule.

CHECK LOGIC:

Step 1: Scan the full text of the Coaching Message for any
        term on the Clinical Language Exclusion List (Section 15,
        Categories A-D).

Step 2: Scan the full text of the Deep Dive for any term on
        the Clinical Language Exclusion List (Categories A-D).

Step 3: Check for clinical framing even when specific terms
        are absent. Clinical framing includes:
       - Diagnostic language patterns ("symptoms of...",
          "consistent with...", "presents as...")
       - Treatment language ("intervention", "protocol",
          "treatment plan")
       - Pathologizing normal athlete behavior ("avoidance
          behavior", "maladaptive coping")
       - Implying a clinical condition exists

PASS: Zero clinical terms found. No clinical framing detected.
FAIL: Any clinical term or clinical framing found anywhere
      in either document.

NOTE ON AUTO-CORRECTION:
Clinical language is NEVER auto-corrected. Even if the fix
appears simple (swap one word), the presence of clinical
language suggests a systemic framing issue in the output.
The entire output must be regenerated to ensure the non-
clinical frame is maintained throughout.

WHY THIS IS TIER 1:
VirtusFocus operates in the non-clinical coaching space.
Clinical language, even a single term, crosses a professional
boundary. There is no context in which it is acceptable.

SECTION 3 - CRITERION 3: DATA FIDELITY
============================================================
Tier: 1 - MUST PASS
Applies to: Coaching Message AND Deep Dive
Failure action: REJECT AND REGENERATE

PURPOSE:
Every factual claim in the coaching output must be traceable
to the Interpretation JSON. Fabricated or contradicted data
means the athlete receives coaching based on something that
did not happen.

CHECK LOGIC:

Step 1: Identify all factual claims in the Coaching Message.
        A factual claim is any statement that:
       - Names a specific event, behavior, or achievement
       - References a score, tier, or classification
       - States what the athlete did, felt, or experienced
       - Quotes or paraphrases the athlete

Step 2: For each factual claim, verify ONE of the following:
        a) The claim directly matches a JSON field value
        b) The claim is a reasonable coaching restatement of
           a JSON field value (same meaning, different words)
        c) The claim is drawn from the baseline snapshot
           (for identity thread content)

Step 3: Check for FABRICATION: claims that cannot be traced
        to any JSON field. Examples:
       - "You scored 4 out of 5 on three days" when the JSON
          shows different daily scores
       - "Your recovery after Wednesday was impressive" when
          the JSON shows no Missed day followed by recovery
       - "You mentioned wanting to lead the team" when no
          such statement exists in athlete_voice

Step 4: Check for CONTRADICTION: claims that directly
        conflict with JSON field values. Examples:
       - Calling the week "Mixed" when weekly_tier = Growth
       - Saying the athlete struggled with consistency when
          consistency_profile = Stable Positive
       - Describing a derailer as not activated when
          baseline_alignment shows derailer_activated = true

Step 5: Repeat Steps 1–4 for the Deep Dive.

Step 6: Check for INVENTION: coaching themes, strength
        signals, or growth observations not grounded in any
        JSON field. The coaching output may restate JSON data
        in coaching language, but it may not introduce entirely
        new themes.

PASS: All factual claims traceable to JSON. No fabrication,
      contradiction, or invention detected.
FAIL: Any fabricated claim, contradicted data, or invented
      theme found.

WHY THIS IS TIER 1:
An athlete receiving coaching about something that did not
happen, or contradicting something that did, undermines
the entire coaching relationship. This cannot be fixed by
editing; the output was built on false premises.

SECTION 4 - CRITERION 4: PRIOR COMMITMENT BRIDGE
============================================================
Tier: 1 - MUST PASS
Applies to: Coaching Message
Failure action: REJECT AND REGENERATE

PURPOSE:
When a prior week's micro-commitment exists, the coaching
message must acknowledge it before introducing the new one.
Skipping this makes the coaching feel disconnected. The
athlete executed (or did not execute) a specific action and
deserves recognition of that continuity.

CHECK LOGIC:

Step 1: Read prior_commitment_result from the JSON.

Step 2: Apply the appropriate rule:

  IF execution_status = "No Prior Week":
    No check needed. PASS automatically.

  IF execution_status = "Completed":
    The Coaching Message must contain at least 1 sentence
    that acknowledges the prior commitment and recognizes
    completion. The acknowledgment must appear BEFORE the
    new micro_commitment is introduced.

  IF execution_status = "Partial":
    The Coaching Message must contain at least 1 sentence
    that names the prior commitment, acknowledges partial
    progress without judgment, and bridges to the current
    week's micro_commitment.

  IF execution_status = "Not Attempted":
    The Coaching Message must contain at least 1 sentence
    that names the prior commitment, acknowledges it was
    not attempted without guilt or criticism, and bridges
    to the current week's micro_commitment.

  IF execution_status = "Unknown":
    The Coaching Message should not reference a specific
    prior commitment result. A general continuity statement
    is acceptable. If the message fabricates a result
    (claims completed/partial/not attempted when the JSON
    says Unknown), that is a Data Fidelity violation
    (Criterion 3) in addition to this criterion failure.

Step 3: Verify the acknowledgment does not exceed 3 sentences.
        The mapping rule targets 1 sentence for the prior
        commitment reference. Up to 3 sentences is acceptable
        if the additional sentences bridge to the new commitment
        or provide a short transition (e.g., "That's the
        standard. Carry it forward."). More than 3 sentences
        means the message is dwelling on the prior week rather
        than moving forward.

Step 4: If carry_forward = "Yes", verify that the current
        week's micro_commitment reflects a continuation or
        evolution of the prior commitment (this connects to
        the Action Specificity check in Criterion 6).

PASS: Prior commitment acknowledged appropriately and
      positioned before the new micro_commitment.
FAIL: Prior commitment exists but is not acknowledged, or
      is acknowledged after the new commitment, or the
      acknowledgment fabricates a result.

WHY THIS IS TIER 1:
The prior commitment bridge creates week-to-week continuity.
Without it, each week feels isolated rather than part of a
developmental arc. An athlete who executed their commitment
and receives no recognition will disengage. This is a
structural issue that requires regeneration. Inserting an
acknowledgment into an existing message disrupts the flow.

SECTION 5 - CRITERION 5: ATHLETE VOICE REFLECTION
============================================================
Tier: 2 - SHOULD PASS
Applies to: Coaching Message
Failure action: AUTO-CORRECT if possible, else REJECT

PURPOSE:
When the athlete's own language is captured in the JSON,
mirroring it back creates resonance and trust. The athlete
should feel heard in their own words.

CHECK LOGIC:

Step 1: Read athlete_voice from the JSON.

Step 2: Check if key_phrase or achievement_statement contains
        a substantive statement (not "insufficient data" or
        blank).

Step 3: If substantive athlete language exists:
  a) The Coaching Message must contain at least 1 sentence
     that mirrors or closely paraphrases the athlete's own
     language from the athlete_voice field.
  b) The mirrored language must be connected to either the
     identity thread or the most_important_controllable.
  c) No more than 1 athlete-voice mirror per message.

Step 4: If no substantive athlete language exists (all fields
        are "insufficient data" or blank), this criterion
        PASSES automatically. There is nothing to mirror.

AUTO-CORRECTION RULE:
If the Coaching Message contains the identity thread section
but does not mirror athlete language:
- Insert one sentence that uses the athlete's key_phrase or
  achievement_statement, connected to the identity anchor or
  the most_important_controllable.
- Place the sentence within the identity thread section.
- Do NOT add a new section. Weave it into existing text.

If the identity thread section is missing entirely, this
becomes a structural gap. REJECT AND REGENERATE.

PASS: Athlete language mirrored once, connected to identity
      or controllable.
CORRECTED: Mirror sentence inserted into existing text.
FAIL: No athlete voice reflection and no viable insertion
      point (structural gap requiring regeneration).

SECTION 6 - CRITERION 6: ACTION SPECIFICITY (THREE-PART CHECK)
============================================================
Tier: 2 - SHOULD PASS
Applies to: Coaching Message
Failure action: AUTO-CORRECT if possible, else REJECT

PURPOSE:
The micro-commitment must be specific enough for the athlete
to execute without additional interpretation. A vague action
creates confusion rather than direction.

CHECK LOGIC:

Step 1: Read micro_commitment from the JSON. It contains:
       - action (what to do)
       - when (when to do it)
       - measurement (how to know it was done)

Step 2: Locate the action/commitment section in the Coaching
        Message (this is the "ONE Action" from the Coaching
        Message Framework).

Step 3: Verify THREE elements are present in the message:

  Part 1 - ACTION:
    The message must state the specific action from
    micro_commitment.action. The language must match or
    closely paraphrase the JSON. The action must not be
    generalized (e.g., "stay focused" instead of the
    specific action from the JSON).

  Part 2 - WHEN:
    The message must specify when the action is to be
    performed, drawn from micro_commitment.when.

  Part 3 - MEASUREMENT:
    The message must include how the athlete will know
    the action was completed, drawn from
    micro_commitment.measurement.

Step 4: If carry_forward = "Yes" from prior_commitment_result,
        verify the current micro-commitment reflects a
        continuation or evolution of the prior commitment,
        not an entirely unrelated action.

AUTO-CORRECTION RULE:
If one element is missing but the other two are present:
- Insert the missing element using the exact JSON field value.
- Place it adjacent to the existing action content.

If two or more elements are missing, or the action section is
absent entirely, REJECT AND REGENERATE.

PASS: All three elements present and match JSON.
CORRECTED: One missing element inserted from JSON.
FAIL: Two+ elements missing or action section absent.

SECTION 7 - CRITERION 7: UPCOMING CONTEXT ANCHORING
============================================================
Tier: 2 - SHOULD PASS
Applies to: Coaching Message AND Deep Dive
Failure action: AUTO-CORRECT if possible, else REJECT

PURPOSE:
When the athlete has a known upcoming competition or event,
the coaching output should connect the micro-commitment to
that specific context. This makes the coaching feel relevant
and forward-looking.

CHECK LOGIC:

Step 1: Read upcoming_context from the JSON.

Step 2: If scheduled_competition = "insufficient data" or is
        absent:
        This criterion PASSES automatically. No upcoming
        context exists to anchor.

Step 3: If scheduled_competition contains a specific event:

  FOR THE COACHING MESSAGE:
  a) The message must contain at least 1 sentence connecting
     the micro_commitment to the upcoming event.
  b) The sentence should frame the commitment as preparation
     for the specific event (e.g., "This is your tool for
     [event]" or similar forward-looking language).

  FOR THE DEEP DIVE:
  c) The Coaching Focus Areas section must include at least
     one priority that specifically references the upcoming
     event.
  d) The priority must connect the micro_commitment to the
     competition context.

AUTO-CORRECTION RULE:
FOR THE COACHING MESSAGE:
If the micro_commitment is present but not connected to the
upcoming event:
- Add one sentence connecting them, placed after the action
  statement.
- Use the pattern: "[Micro-commitment]. This is your
  preparation for [scheduled_competition]."

FOR THE DEEP DIVE:
If the Coaching Focus Areas section exists but does not
reference the upcoming event:
- Add one bullet point connecting the micro_commitment to
  the upcoming event.

If the Coaching Focus Areas section is absent, REJECT AND
REGENERATE (structural gap).

PASS: Upcoming event referenced and connected to commitment.
CORRECTED: Connection sentence/bullet added to existing text.
FAIL: Coaching Focus Areas section absent (structural gap).

SECTION 8 - CRITERION 8: NARRATIVE ARC PRESENCE
============================================================
Tier: 2 - SHOULD PASS
Applies to: Deep Dive
Failure action: AUTO-CORRECT if possible, else REJECT

PURPOSE:
When longitudinal data exists, the Deep Dive must tell the
athlete's developmental story, not just report on a single week.

CHECK LOGIC:

Step 1: Read narrative_arc from the JSON.

Step 2: If all narrative_arc fields = "insufficient data":
        This criterion is N/A. Not enough longitudinal data
        exists. Mark as N/A and move on.

Step 3: If narrative_arc fields are populated:

  a) The Deep Dive Mindset Summary section must include a
     developmental summary drawn from arc_summary.
     (2-3 sentences placing this week in the longer story.)

  b) milestone_this_week must be named as a specific callout
     somewhere in the Deep Dive (Strengths section or Mindset
     Summary).

  c) If open_thread is populated, it should appear in the
     Coaching Focus Areas as a forward-looking reference.

AUTO-CORRECTION RULE:
If the Mindset Summary section exists but does not include
arc content:
- Insert a 2-3 sentence arc summary drawn from
  narrative_arc.arc_summary at the beginning of the Mindset
  Summary section.
- Insert the milestone_this_week as a callout in the
  Strengths section or Mindset Summary.

If the Mindset Summary section is absent entirely, REJECT
AND REGENERATE (structural gap).

PASS: Arc summary, milestone, and open thread (if present)
      all reflected in the Deep Dive.
N/A: narrative_arc fields all "insufficient data."
CORRECTED: Arc content inserted into existing Mindset Summary.
FAIL: Mindset Summary section absent.

SECTION 9 - CRITERION 9: MESSAGE LENGTH CONSTRAINT
============================================================
Tier: 2 - SHOULD PASS
Applies to: Coaching Message
Failure action: AUTO-CORRECT

PURPOSE:
The Coaching Message is athlete-facing and must be concise.
Excessive length reduces impact and makes the message feel
like a report rather than a coaching conversation.

CHECK LOGIC:

Step 1: Count the total words in the Coaching Message.

Step 2: Apply the length rule:

  Target range: 250-450 words.

  250-450 words: PASS.
  451-550 words: CORRECTABLE. Trim without structural change.
  551+ words: REJECT AND REGENERATE. Message is too long for
              editing to fix without losing coherent structure.
  Under 250 words: FLAG as NOTE in audit summary. Short
              messages are acceptable if all required elements
              are present. If required elements are missing,
              the failure falls under the relevant criterion
              (Action Specificity, Identity Anchor, etc.),
              not this one.

AUTO-CORRECTION RULE (451-550 words):
 - Remove redundant restating of the same point
 - Trim excessive transition language
 - Consolidate strength signals if more than 2 are expanded
 - Remove any sentences that repeat the identity anchor
    without adding new information
 - Do NOT remove: the prior commitment bridge, the athlete
    voice mirror, the action statement, or the upcoming
    context anchor
 - After trimming, re-verify Criteria 4-7 still pass

PASS: 250-450 words.
CORRECTED: Trimmed from 451-550 to within range.
FAIL: 551+ words. Regenerate with instruction to target
      350-word range.

SECTION 10 - CRITERION 10: MICRO-COMMITMENT REALISM
============================================================
Tier: 2 - SHOULD PASS
Applies to: Coaching Message
Failure action: FLAG for revision (auto-correct if possible)

PURPOSE:
The micro-commitment should be realistically executable
within the athlete's weekly routine. Commitments that repeat
a modality the athlete has not executed reduce follow-through
and signal the pipeline is not adapting to the athlete's
demonstrated patterns.

CHECK LOGIC:

Step 1: Read micro_commitment from the JSON.

Step 2: MODALITY ADAPTATION CHECK.
        Read the current week's AND the prior 2+ weeks'
        prior_commitment_result entries.

        Classify each commitment's primary modality:
       - WRITTEN: write, log, record, journal, checklist
       - ORAL: say, state, tell yourself, repeat, aloud
       - BEHAVIORAL: do, perform, complete, warmup, breathe

        If 2+ consecutive prior weeks show execution_status
        = "Partial" on the SAME modality, AND the current
        week's micro-commitment uses the SAME modality:
        → FAIL: "Modality adaptation required. Prior
          commitment modality [WRITTEN/ORAL/BEHAVIORAL] has
          shown Partial execution for [N] consecutive weeks.
          The new commitment must shift modality."

Step 3: STRUCTURAL SIMPLICITY CHECK.
        Evaluate the commitment against simplicity markers:

  PASS markers:
 - Maximum 2 action steps
 - Single trigger moment
 - Binary measurement (yes/no)
 - Tied to a context the athlete already inhabits
    (practice, pre-game, post-game, daily routine)

  FAIL markers:
 - More than 2 action steps in a single commitment
 - Measurement requires logging, tallying, or tracking
 - Volume targets (5+ resets, 6+ checkmarks)
 - Multi-item checklists

Step 4: Evaluate how the coaching output presents the
        commitment in the Coaching Message. Even if the JSON
        commitment is structurally sound, the coaching output
        may add complexity in presentation (extra steps,
        compound framing) or may simplify a complex
        commitment effectively.

AUTO-CORRECTION RULES:
- If the modality adaptation check fails: The Audit Agent
  CANNOT rewrite the micro-commitment (it is a JSON field
  from Stage 2). Issue a REJECT AND REGENERATE with a
  Regeneration Instruction specifying the required modality
  shift.
- If the structural simplicity check fails on presentation
  only (coaching output adds complexity the JSON doesn't
  have): AUTO-CORRECT by simplifying the presentation to
  match the JSON's action/when/measurement format.
- If the structural simplicity check fails on the JSON
  itself: Log as a NOTE for upstream Stage 2 review. Do not
  reject on this basis alone.

PASS: Modality adapts to prior execution patterns AND
      commitment is structurally simple.
FAIL: Modality repeats despite 2+ weeks of Partial
      execution on the same modality → REJECT.

SECTION 11 - CRITERION 11: BRAND VOICE TONAL SCAN
============================================================
Tier: 3 - QUALITY OPTIMIZATION
Applies to: Coaching Message AND Deep Dive
Failure action: AUTO-CORRECT silently

PURPOSE:
All output must sound like VirtusFocus: calm, direct,
identity-based, performance-oriented. This check catches
tonal drift that does not rise to a compliance violation
but reduces brand consistency.

CHECK LOGIC:

Step 1: Scan the Coaching Message for tonal violations:

  PROHIBITED TONAL PATTERNS:
  a) Hype language: "You're unstoppable!", "Amazing work!",
     "Incredible week!", excessive exclamation marks (more
     than 2 total in the Coaching Message)
  b) Cliché motivation: "Believe in yourself!", "You've got
     this!", "Sky's the limit!", "Never give up!"
  c) Coddling language: Excessive reassurance that avoids
     naming real patterns ("Everything is going great" when
     drift is present)
  d) Guilt-driven tone: "You need to try harder", "This
     isn't good enough", "You're falling behind"
  e) Academic/intellectual tone: Overly analytical language
     that reads like a research paper rather than coaching
  f) Preachy tone: Lecturing, moralizing, "You should know
     by now that..."

Step 2: Scan the Deep Dive for the same tonal violations.

Step 3: Additionally, check the Deep Dive for:
  g) Coach-facing technical language that belongs in
     Stage 4 (Coach Insights), not in the Deep Dive.
     Examples of coach-facing language:
    - "From a coaching perspective..."
    - "The data suggests that the athlete..."
    - "Programmatic recommendation..."
    - "Assessment indicates..."
    - Third-person references to the athlete ("the athlete"
       instead of "you" or the athlete's name) in sections
       that should be athlete-facing
     NOTE: The Deep Dive is a professional-grade report that
     the athlete may read. It should use second person ("you")
     or the athlete's name, and should maintain the VirtusFocus
     voice throughout, even in analytical sections.

AUTO-CORRECTION RULE:
 - Replace hype language with calm, direct recognition
 - Replace clichés with specific, evidence-based statements
 - Reduce exclamation marks to a maximum of 2 per message
 - Replace coddling with honest, supportive acknowledgment
 - Replace guilt-driven language with ownership-based framing
 - Replace academic tone with clear, direct language
 - Replace coach-facing language with athlete-appropriate
    language (second person, VirtusFocus voice)
 - Do NOT change the meaning or remove content. Only
    adjust the tonal register

PASS: Voice consistent with VirtusFocus brand throughout.
CORRECTED: Tonal adjustments applied (list specific changes
           in Audit Summary).

SECTION 12 - CRITERION 12: OUTER-RING LANGUAGE DISCIPLINE
============================================================
Tier: 3 - QUALITY OPTIMIZATION
Applies to: Coaching Message AND Deep Dive
Failure action: AUTO-CORRECT silently

PURPOSE:
The VirtusFocus Bullseye Method anchors all coaching in the
center ring (controllables). When coaching language drifts
to outer-ring content without redirecting back to the center
ring, it reinforces the wrong focus pattern.

CHECK LOGIC:

Step 1: Scan both documents for outer-ring references:
 - Lineup decisions
 - Playing time
 - Coach behavior / coach decisions
 - Referee/umpire calls
 - Opponent actions
 - Weather / field conditions
 - Scores / outcomes (when presented as the primary focus
    rather than context)
 - Parental pressure
 - Team politics

Step 2: For each outer-ring reference found, verify that:
  a) It is used as CONTEXT, not as a coaching focus point
  b) It is followed (within the same paragraph or the next)
     by a redirection to a center-ring controllable
  c) The coaching message does not validate or dwell on
     the outer-ring concern

Step 3: Exception: the Deep Dive may reference outer-ring
        factors when analyzing behavioral patterns, but must
        always connect them to a center-ring redirection.

AUTO-CORRECTION RULE:
If an outer-ring reference exists without a center-ring
redirect:
 - Add a redirect sentence after the outer-ring reference
 - Use the pattern: "What you control is [center-ring
    controllable from most_important_controllable]."
 - Do NOT remove the outer-ring reference. Just ensure
    the redirect follows

If the outer-ring reference is the primary coaching focus
(multiple paragraphs dwelling on an uncontrollable without
redirect), this becomes a structural issue. REJECT AND
REGENERATE under Tier 1: Data Fidelity (the coaching
output is introducing a theme not grounded in center-ring
JSON fields).

PASS: All outer-ring references are contextualized and
      redirected.
CORRECTED: Redirect sentences added after unredirected
           outer-ring references.

SECTION 13 - CRITERION 13: PIPELINE DATA LEAKAGE COMPLIANCE
============================================================
Tier: 1 - MUST PASS
Applies to: Coaching Message AND Deep Dive
Failure action: REJECT AND REGENERATE

PURPOSE:
Internal pipeline data, including execution behavior signals
(composite scores, coach flags, raw app metrics, self-ratings
alignment), baseline intake profile data (PPD bucket names, PPD
scoring artifacts, ABI pillar scores, adversity response
classifications, intake profile field names), longitudinal PPD
data (PPD Alignment Signal labels, evidence tiers, intake-to-
execution comparison framing), and forward anchor data
(controllability classifications, alignment status labels,
focus consistency labels, raw forward anchor text, forward
anchor field names), is backend-only data consumed internally
by the pipeline to produce better coaching output. Raw pipeline
data NEVER appears in athlete-facing content. Surfacing it,
even in paraphrased form, crosses a professional boundary by
exposing surveillance-style engagement metrics, internal
assessment scoring systems, or intention-tracking
classifications to the athlete.

This criterion is the pipeline data counterpart to Criterion 2
(Clinical Language Compliance). Criterion 2 prohibits clinical/
therapeutic terminology. Criterion 13 prohibits pipeline data
terminology: execution signals (Category E), intake profile
classifications (Category F), and forward anchor terminology
(Category G). All are compliance violations, not style issues.

RELATIONSHIP TO EXISTING CRITERIA:
- Criterion 2 (Clinical Language): Different exclusion
  category. Clinical = therapeutic/diagnostic terms.
  Pipeline data = surveillance/engagement data terms AND
  intake assessment scoring terms AND longitudinal PPD
  comparison terms AND forward anchor classification terms.
  No overlap.
- Criterion 3 (Data Fidelity): Different concern. Criterion 3
  asks "is this claim true?" Criterion 13 asks "should this
  truth appear in athlete-facing output at all?" A statement
  can be factually accurate per the JSON yet still fail
  Criterion 13 because the data category is prohibited from
  appearing in coaching output.

CHECK LOGIC:

Step 1: Read execution_behavior_signals.input_source from

NOTE ON AUTO-CORRECTION:
Pipeline data leakage is NEVER auto-corrected. Same
rationale as Clinical Language (Criterion 2): the presence
of raw pipeline data (whether execution metrics, intake
profile classifications, or forward anchor terminology)
suggests the engine built coaching direction around surfaced
internal data rather than behavioral translations. Removing
the offending sentences would leave structural gaps in the
coaching narrative. The entire output must be regenerated
with proper pipeline-data-to-behavioral translation.

WHY THIS IS TIER 1:
Raw pipeline data in athlete-facing content crosses the
same professional boundary as clinical language. An athlete
reading "Your Ownership Index was Low at 35/100" or "You
needed reminders 4 out of 7 days" experiences surveillance,
not coaching. An athlete reading "Your Motivation and
Resilience Loss score suggests..." or "Your ABI Composure
pillar is Low" experiences being scored and classified, not
coached. An athlete reading "Your forward anchor alignment
was Partial" or "Your intention was classified as Outcome-
Oriented" experiences being tracked and categorized. An
athlete reading "Your intake identified overthinking but
the real pattern is structure" or "There's a blind spot in
your self-assessment" experiences being compared and
evaluated by a system, not coached. All damage trust,
violate the voluntary participation principle, and cannot
be fixed by editing. They indicate a systemic translation
failure requiring full regeneration.

PURPOSE:
The em dash character is banned from all athlete-facing and
parent-facing output. Em dashes are an AI-pattern marker that
reduces voice authenticity. This is a total ban with zero
tolerance, not a frequency cap.

        occurrence of the em dash character (Unicode U+2014).

Step 2: Scan the full text of the Deep Dive for any occurrence
        of the em dash character.

Step 3: If a Parent Coaching Message is present (conditional
        4th input), scan it for any occurrence of the em dash
        character.

Step 4: Count all instances found.

PASS: Zero em dashes found in any document.
FAIL: One or more em dashes found in any document.

AUTO-CORRECTION RULE:

alternative based on its function in the sentence:

  Parenthetical aside: Replace with comma or period + new

    NOTE: The above uses a hyphen. The em dash version would
    be the same with the longer dash character.
    BEFORE: "Your consistency held all week long, even when
    the schedule shifted" [with em dash before "even"]
    AFTER: "Your consistency held all week long. Even when
    the schedule shifted, it stayed."

  Contrast or reframe: Replace with period or comma + "but."
    BEFORE: "Not the outcome you wanted" [em dash] "the
    process you needed."
    AFTER: "Not the outcome you wanted, but the process you
    needed."

  Mid-sentence pivot: Replace with colon or comma.
    BEFORE: "One thing stood out" [em dash] "your preparation."
    AFTER: "One thing stood out: your preparation."

  List introduction: Replace with colon.
    BEFORE: "Three things defined this week" [em dash]
    "consistency, composure, and ownership."
    AFTER: "Three things defined this week: consistency,
    composure, and ownership."

  Dramatic pause: Replace with period + new sentence.
    BEFORE: "That is discipline" [em dash] "earned, not
    given."
    AFTER: "That is discipline. Earned, not given."

NOTE: Do not replace em dashes with en dashes or double
hyphens. Use the replacement alternatives listed above.
Regular hyphens in compound words (e.g., "self-directed,"
"center-ring") are not em dashes and are permitted.

VERIFICATION:
After correction, re-scan to confirm zero em dashes remain.


============================================================
SECTION 13B - PIPELINE CLASSIFICATION LANGUAGE CHECK
============================================================
Tier: 2 - SHOULD PASS (auto-correctable when behavioral
          replacement is clear)
Tier: 1 - MUST PASS (when the term is central to a coaching
          claim that would change meaning if reworded)
Applies to: Coaching Message AND Deep Dive AND Parent Coaching
            Message (if present)
Failure action: AUTO-CORRECT if behavioral replacement is clear,
                else REJECT AND REGENERATE

PURPOSE:
In addition to the existing Criterion 2 (Clinical Language)
and Criterion 13 (Pipeline Data Leakage) checks, this section
catches core v9.7 Interpretation Engine field names and
classification labels that leak into coaching prose. These
terms overlap partially with existing Category E (Execution
Signal Terminology) and Category F (Intake/Longitudinal PPD)
but also cover base-level pipeline field names that may appear
in coaching output when the generation engine fails to
translate classifications into behavioral language.

CHECK LOGIC:

Step 1: Scan the Coaching Message, Deep Dive, and Parent
        Coaching Message (if present) for the following
        pipeline classification terminology used as labels
        or direct classifications:

  a) Emotional intensity used as a label:
    - "Emotional intensity [Low/Moderate/High]"
    - "EI: Low" / "EI: Moderate" / "EI: High"
    - "emotional intensity level"
     NOTE: "emotional intensity" as a natural description
     of the athlete's week is permitted. "Your emotional
     intensity was High this week" is a pipeline label
     and is prohibited.

  b) Growth phase or tier used as direct classification
     statements:
    - "[Growth/Developing/Consistent/Leadership] phase
       maintained"
    - "Growth tier" / "Mixed tier" / "Reset tier" as
       named tier classifications
    - "weekly_tier" / "growth_phase" / "growth_phase_movement"
       as field names
     NOTE: Referencing the athlete's growth in natural
     coaching language is permitted. "You're growing" is
     coaching. "Growth tier maintained" is a pipeline label.
     "Your growth phase is Developing" is a pipeline label.

  c) Reflection Quality Score referenced directly:
    - "RQS [number]" / "RQS=3" / "RQS: 4"
    - "Reflection Quality Score"
    - "reflection_quality_score" as field name
     NOTE: Discussing the quality of the athlete's
     reflections in coaching language is permitted.
     "Your reflections went deeper this week" is coaching.
     "Your RQS improved to 3" is a pipeline label.

  d) Consistency signal labels used as named states:
    - "Stable Positive" / "Stable Neutral" /
       "Variable Improving" / "Variable Declining" /
       "Insufficient Data" as named consistency
       classifications
    - "consistency_profile" / "consistency_signal"
       as field names
     NOTE: Describing consistency patterns in coaching
     language is permitted. "Your consistency has been
     steady" is coaching. "Consistency signal: Stable
     Positive" is a pipeline label.

  e) Alignment status used as named states:
    - "Partially Aligned" / "Aligned" / "Not Aligned"
       as named baseline alignment classifications
    - "baseline_alignment" as field name
     NOTE: "aligned" in natural English is permitted
     (covered by existing edge case 10 and Category F6
     context rules). "Partially Aligned" as a formal
     two-word classification is prohibited.

  f) Technical framework terms used as labels:
    - "approach-framed" as a term (pipeline classification
       language for tie-breaker or pre-performance thinking)
    - "pattern_shift_detected" as a term
    - "center-ring dominant" / "outer-ring dominant" as
       named Bullseye classifications
    - "noise_fixation_present" as field name
     NOTE: "center ring" and "outer ring" as Bullseye
     Method concepts in coaching are permitted (standard
     VirtusFocus framework language). "Center-ring dominant"
     as a named classification label is prohibited.

Step 2: Determine severity:

  IF the term can be replaced with a behavioral translation
  without changing the coaching claim:
    Tier 2 - AUTO-CORRECT.
    Replace with behavioral language.

  IF the term is central to the coaching claim and removing
  it would change the meaning of the paragraph or section:
    Tier 1 - REJECT AND REGENERATE.
    The coaching direction was built around a pipeline
    classification rather than a behavioral observation.

AUTO-CORRECTION EXAMPLES:

  BEFORE: "Growth tier maintained for the fourth consecutive
  week."
  AFTER: "Four weeks in a row of strong execution."

  BEFORE: "Your RQS improved from 2 to 3 this week."
  AFTER: "Your reflections went deeper this week, naming
  specific patterns instead of surface-level observations."

  BEFORE: "Stable Positive consistency signal held."
  AFTER: "Your consistency held steady again this week."

  BEFORE: "Partially Aligned this week, with derailer
  activation but hinge habit holding."
  AFTER: "The derailer showed up this week, but your
  hinge habit held."

  BEFORE: "Your center-ring dominant Bullseye shows strong
  controllable focus."
  AFTER: "Your Bullseye stayed anchored in what you can
  control."

  BEFORE: "Pattern shift detected in weekend engagement."
  AFTER: "Something shifted in your weekend approach."

  BEFORE: "Approach-framed pre-performance thinking
  continued."
  AFTER: "You kept your pre-game mindset focused on what
  you wanted to do, not what you were afraid of."

RELATIONSHIP TO EXISTING CRITERIA:
This check complements Criterion 2 and Criterion 13. It does
not replace either. A single output can fail Criterion 2
(clinical language), Criterion 13 (pipeline data leakage),
and this check (pipeline classification language)
independently. All three are reported separately in the
Audit Summary or Regeneration Instruction.


============================================================
SECTION 14 - CRITERION 14: VOICE QUALITY REGRESSION
============================================================

Tier: 2 (SHOULD PASS, auto-correct where possible, reject
if not)
Applies to: Coaching Message AND Deep Dive (and Parent
Coaching Message when present)

PURPOSE:
Detects AI-pattern regressions in voice and rhetorical
moves that the Voice Calibration Reference is designed to
prevent. Complements the Voice Reference by catching
patterns that slipped through generation. The Voice
Reference is the quality benchmark this criterion measures
against. Read VF_Coach_Arron_Voice_Reference.txt before
running this criterion to understand what the target
quality looks like.

CHECK LOGIC:
Scan the Coaching Message and Deep Dive (and Parent CM when
present) for the following fail conditions. Flag any
detected and log them in the Audit Summary.

------------------------------------------------------------
FAIL CONDITION 14.1: REUSABLE OPENING
------------------------------------------------------------

The opening of the CM or DD could be reused next week by
changing only the number, date, or surface detail.

Common forms:
- Stat-led openings with running totals (e.g., "Sixty-three
  straight Win days." "Nine consecutive Growth weeks.")
- Template openings where substituting another athlete
  would still work ("Week 9 in your program...")
- Generic scene-setters that do not reference anything
  specific from THIS week
- "[N] weeks into this program" or similar temporal
  anchors as opening hooks

REUSABLE OPENING TEST: If the opening could be reused next
week by changing only the number or date, it is too
templated. The exception is when the stat itself genuinely
defines the week (a milestone, a record, a first, a streak
that just broke). A stat that continues a trendline is
templated. A stat that changes the meaning of the week is
not.

AUTO-CORRECTION: NOT AUTO-CORRECTABLE. Requires full
opening rewrite with meaning-first framing. REJECT AND
REGENERATE with reference to Voice Reference Category A
(Opening a Message) exemplars.

------------------------------------------------------------
FAIL CONDITION 14.2: TELEGRAPHING
------------------------------------------------------------

A sentence announces the coaching move instead of making
it. The announcement is the tell. If the sentence can be
deleted and the paragraph still works, it was scaffolding,
not content.

Common forms:
- "Now I want to push you on something this week."
- "Here's what I want you to see."
- "There's something I want to talk about."
- "I'm going to be direct about something."
- "Now here's where it gets interesting."
- "Let me tell you what stood out."
- "I need to name this directly."

Common variants that should also trigger:
- "That kind of [X] doesn't happen by accident." (abstracts
  a specific observation into a general principle nobody
  needed)
- "Here is what this week was really about." (summary-led
  framing instead of direct observation)

AUTO-CORRECTION: Delete the announcing sentence. Verify
the following sentence still reads naturally. If yes,
correction complete. If no, REJECT AND REGENERATE.

------------------------------------------------------------
FAIL CONDITION 14.3: UNTRACEABLE PARAGRAPH
------------------------------------------------------------

A paragraph is not clearly traceable to a specific athlete
behavior, statement, or moment from the week. Every
paragraph in a CM or DD should be responding to something
the athlete did or said. Paragraphs that apply generic
coaching principles without specific athlete grounding
fail this check.

Check method: For each paragraph, identify the specific
JSON field or athlete input the paragraph is responding
to. If you cannot point to specific data, the paragraph
is filler.

AUTO-CORRECTION: NOT AUTO-CORRECTABLE. Requires new
content grounded in athlete data. REJECT AND REGENERATE
with reference to Voice Reference Principle 5 (React, Don't
Compose).

------------------------------------------------------------
FAIL CONDITION 14.4: CATEGORY COVERAGE OVER RESPONSE
------------------------------------------------------------

The message reads like it is covering required categories
(recognition, strength, drift, identity, action) rather
than responding to THIS week.

Signs:
- Evenly balanced paragraphs when the week was not evenly
  balanced
- Transition sentences that announce section shifts
- Coverage of every field in the JSON regardless of
  relevance
- The message reads like a report moving through a
  template
- One-paragraph-per-data-point structure regardless of
  what mattered most in the week

Coaching Output Instructions v2.3 Section 36 (Uneven
Coverage Principle) governs generation-time application.
This criterion catches its absence at audit.

AUTO-CORRECTION: NOT AUTO-CORRECTABLE. Structural problem
requiring full regeneration with proportional weighting to
the week's dominant story. REJECT AND REGENERATE.

------------------------------------------------------------
FAIL CONDITION 14.5: BRAND COPY CLOSE
------------------------------------------------------------

The closing reads like marketing copy, a motivational
slogan, or a generic brand phrase rather than athlete-
specific language.

The close should connect to something from THIS athlete's
arc: identity claim, hinge habit, a specific moment, or
their own language. Standard rotation phrases ("Stay
centered," "Lead yourself first," "Control what's in your
center ring") are acceptable as fallback only when the
week does not offer a natural athlete-specific closing.

Common failures:
- "Simple. Repeatable. Reliable." (brand phrase stack)
- "Keep showing up for yourself." (generic)
- "This is who you're becoming." (generic without
  specific connection to the athlete's actual arc)
- "You've got this." (hype)
- Any closing that could appear on a social media post

AUTO-CORRECTION: Conditionally auto-correctable. If the
week's JSON contains clear hooks for an athlete-specific
close (identity_claim, hinge_habit, or athlete_voice.
key_phrase), rewrite the closing to reference them. If
the week does not offer natural hooks, replace with a
standard rotation phrase. If the structural framing
around the close requires broader rewriting, REJECT AND
REGENERATE.

------------------------------------------------------------
FAIL CONDITION 14.6: SAME CONSTRUCTION IN 3-WEEK WINDOW
------------------------------------------------------------

The same rhetorical construction appears in the current
week's CM or DD and in either of the prior 2 weeks for
the same athlete. This requires cross-week reading: the
auditor must have access to the two most recent prior
outputs for the same athlete.

Common forms to check:
- Same opening approach (stat-led, moment-led, etc.)
- Same transition phrase or bridging construction
- Same italicized observation structure in DD
- Same closing phrase
- Same reframe move ("Not X. Y.")

If access to prior weeks is not available, skip this
check and note "insufficient data for cross-week check"
in the Audit Summary.

AUTO-CORRECTION: Conditionally auto-correctable. If the
repeated construction is isolated (a single phrase or
sentence), rewrite that sentence using an alternative
approach. If the repetition is structural (same opening
pattern three weeks in a row), REJECT AND REGENERATE with
reference to Voice Reference opening category exemplars.

------------------------------------------------------------
FAIL CONDITION 14.7: DD AS EXPANDED CM
------------------------------------------------------------

The Deep Dive reads like an expanded or repeated version
of the Coaching Message instead of a different angle on
the same week. The CM is the highlight reel. The DD is
the full picture. They should feel like two complementary
perspectives from the same coach, not the same draft at
two lengths.

Check method: Compare the CM and DD for the same week.
Look for:
- Substantial phrase duplication (more than a few words
  of natural coaching vocabulary)
- Identical framing of the same observation
- DD sections that simply expand the corresponding CM
  section without adding new depth, new angle, or new
  evidence

Natural coaching vocabulary ("your controllables," "the
hinge habit," "sit with it") is not duplication. Identical
sentence constructions and identical framings are.

AUTO-CORRECTION: NOT AUTO-CORRECTABLE. Structural problem
requiring DD to be rewritten as a different angle. REJECT
AND REGENERATE.

------------------------------------------------------------
FAIL CONDITION 14.8: "THAT IS [NOUN]." STANDALONE OVERUSE
------------------------------------------------------------

"That is [noun]." standalone declaratives (e.g., "That is
ownership." "That is discipline." "That is composure.")
appearing more than once in a CM, more than once in a DD,
or in both CM and DD for the same week.

Occasional use as genuine emphasis is acceptable. Repeated
use as a structural habit fails this check.

AUTO-CORRECTION: AUTO-CORRECTABLE. Convert extra instances
to alternative constructions from Voice Reference Category
B exemplars: embed the judgment ("You owned that moment"),
use a fragment ("Ownership. Plain and simple."), let
evidence speak (delete the declarative), or call back to
athlete language.

------------------------------------------------------------
FAIL CONDITION 14.9: "VERSION OF [X]" CONSTRUCTION
------------------------------------------------------------

Any use of "the version of you," "the version of
confidence," "the version of [athlete's quality]"
constructions.

This construction is retired entirely. No exceptions.

AUTO-CORRECTION: AUTO-CORRECTABLE. Replace "the version
of [X]" with "[X]" or direct language.

  WRONG: "The version of you that showed up Thursday..."
  CORRECT: "You, on Thursday..."

  WRONG: "The version of confidence you are building..."
  CORRECT: "The confidence you are building..."

------------------------------------------------------------
FAIL CONDITION 14.10: TRIPLE-NEGATIVE SETUP
------------------------------------------------------------

"Not X, not Y, not Z. Just W." constructions as openers
or transitions. This is a recognizable AI pattern and
should be replaced with direct positive framing.

  WRONG: "Not a failure. Not a breakthrough. Not a moment
  to reset from. Just a week that asked you to stand."

  CORRECT: "This week asked you to stand. You did."

AUTO-CORRECTION: AUTO-CORRECTABLE. Replace triple-negative
setups with direct positive framing. If the construction
appears in an opening, verify the correction does not
produce a different Criterion 14.1 reusable-opening failure.

------------------------------------------------------------
FAIL CONDITION 14.11: TRENDLINE ARC RECITATION
------------------------------------------------------------

DD Weekly Trendline section enumerates more than 3 prior
weeks or uses hyphenated compound-adjective arc labels
("outcome-reactive," "preparation-driven," "structure-
sustained," "competition-validated," etc.).

The trendline should reference 2-3 inflection points
relevant to THIS week's story, in plain coaching language,
not recite the program history from Week 1 every time.

AUTO-CORRECTION: AUTO-CORRECTABLE. Trim to 2-3 inflection
points with plain-language descriptions. Remove hyphenated
compound-adjective arc labels. If the entire Weekly
Trendline section is recitation without selection, REJECT
AND REGENERATE.

------------------------------------------------------------
FAIL CONDITION 14.12: PIPELINE-ADJACENT VOCABULARY
------------------------------------------------------------

Terminology that is not a direct Category E/F/G violation
(which would fail Criterion 13) but still reads as
pipeline-adjacent:

- "Growth weeks" / "Win days" as named categories
- "approach-framed" (in any form)
- "the system" as a reference to the VirtusFocus program
  (when it reads clinically)
- "longitudinal" or "longitudinal pattern" in coaching
  prose
- Other jargon-adjacent terms that a real coach would not
  use

AUTO-CORRECTION: AUTO-CORRECTABLE. Replace with behavioral
description or natural coaching language.

  WRONG: "Three consecutive Growth weeks."
  CORRECT: "Three weeks of steady Wins."

  WRONG: "Your approach-framed focus held."
  CORRECT: "You stayed focused on competing to score,
  not avoiding losing."

------------------------------------------------------------
AUTO-CORRECTION DECISION RULES FOR CRITERION 14
------------------------------------------------------------

Auto-correct only when the fix is clear and mechanical.
The following are auto-correctable:

  14.2 (telegraphing) - delete announcing sentence
  14.5 (brand copy close) - conditional, if hooks exist
  14.6 (3-week repetition) - conditional, if isolated
  14.8 (That is [Noun] overuse) - mechanical alternatives
  14.9 (version of X) - replace with direct language
  14.10 (triple-negative setup) - replace with positive
  14.11 (trendline recitation) - conditional, trim
  14.12 (pipeline-adjacent vocabulary) - replace

REJECT AND REGENERATE when:

  14.1 (reusable opening) - requires full opening rewrite
  14.3 (untraceable paragraph) - requires new grounded
       content
  14.4 (category coverage over response) - structural
       problem
  14.7 (DD as expanded CM) - requires DD rewrite as
       different angle
  3+ auto-correctable fails combined - reject rather than
       patch (indicates systemic voice quality issue)

------------------------------------------------------------
REGENERATION INSTRUCTION TEMPLATE FOR CRITERION 14
------------------------------------------------------------

When rejecting for Criterion 14, include in the
regeneration instruction:
  - Which fail conditions triggered (list all)
  - The specific passage(s) that failed (quote verbatim)
  - Reference to the relevant Voice Reference exemplar
    category for calibration (A Opening, B Recognition,
    C Challenge, D Failure, E Trendline, F Identity/Closing)
  - Explicit instruction to re-read the Voice Reference
    and the Voice Foundations document before regenerating
  - Instruction to apply the Uneven Coverage Principle
    (Coaching Output Instructions v2.3 Section 36) and
    Exemplar Category Matching (Section 37) in the
    regeneration

------------------------------------------------------------
RELATIONSHIP TO OTHER CRITERIA
------------------------------------------------------------

CRITERION 11 (BRAND VOICE TONAL SCAN): Handles tonal
register violations (hype, cliche, guilt, academic,
preachy). Criterion 14 handles voice quality regression
at the rhetorical-move level. Both apply. A passage can
pass 11 and fail 14 (tonally on-brand but rhetorically
templated), or pass 14 and fail 11 (rhetorically grounded
but tonally preachy).

CRITERION 13 (PIPELINE DATA LEAKAGE): Handles raw pipeline
data surfacing (Category E execution signals, Category F
intake profile + longitudinal PPD, Category G forward
anchor). Criterion 14 handles pipeline-adjacent vocabulary
that does not directly surface data but reads as jargon
(14.12). Both apply. A passage can pass 13 (no named
scores, no field names) and fail 14.12 (still uses
"Growth weeks" as a named category).

CRITERIA 1-4 (Tier 1): Take precedence over Criterion 14.
If a Tier 1 failure exists, reject for that first and do
not waste cycles on Criterion 14 auto-correction until
Tier 1 is resolved in regeneration.


============================================================
SECTION 15 - STRUCTURAL INTEGRITY CHECKS
============================================================

These are not scored criteria. They are prerequisite
verifications run before the 14 criteria. If either
structural check fails, the output is REJECTED without
running the full audit.

COACHING MESSAGE STRUCTURE CHECK:
The Coaching Message must contain all 5 mandatory sections
from the Coaching Message Framework:
  1. Recognition
  2. Strength Signals (1-2)
  3. Drift Identification (if present in JSON)
  4. Identity Anchor
  5. ONE Action (micro_commitment)

If any section is missing, REJECT AND REGENERATE with a
note identifying the missing section(s).

Exception: Drift Identification may be absent if no drift
is present in the JSON (core_pattern shows no outer-ring
fixation, no reactive pattern, and bullseye primary_focus_ring
= Control).

DEEP DIVE STRUCTURE CHECK:
The Deep Dive must contain all 5 mandatory sections from the
Deep Dive Framework:
  1. Weekly Overview
  2. Strengths & Wins (4-6 sections)
  3. Opportunities & Growth Areas (3-5 sections)
  4. Mindset Summary
  5. Coaching Focus Areas (Next Week)

If any section is missing, REJECT AND REGENERATE with a
note identifying the missing section(s).

SECTION 16 - CLINICAL LANGUAGE AND PIPELINE DATA
             EXCLUSION LIST
============================================================

The following terms are NEVER acceptable in athlete-facing
coaching content. This list applies to both the Coaching
Message and the Deep Dive.

CATEGORY A - DIAGNOSTIC / CLINICAL TERMS:
  anxiety (use: "pressure", "nerves", "tension")
  depression (use: "low energy", "flat", "disengaged")
  trauma (use: "difficult experience", "tough stretch")
  disorder (never appropriate, remove entirely)
  diagnosis (never appropriate, remove entirely)
  pathology / pathological (never appropriate)
  symptom / symptoms (use: "pattern", "signal")
  syndrome (never appropriate)
  clinical (never appropriate)
  psychiatric (never appropriate)
  psychological (use: "mindset", "mental", "performance")
  psychologist (never appropriate)
  therapist / therapy / therapeutic (never appropriate)
  counseling / counselor (never appropriate)
  mental illness (never appropriate)
  mental health condition (never appropriate)
  mentally ill (never appropriate)

CATEGORY B - TREATMENT / INTERVENTION TERMS:
  intervention (use: "adjustment", "coaching focus")
  treatment (use: "approach", "coaching direction")
  treatment plan (use: "coaching plan", "game plan")
  protocol (use: "system", "routine", "process")
  cognitive behavioral (never appropriate)
  CBT (never appropriate)
  exposure therapy (never appropriate)
  desensitization (never appropriate)
  psychoeducation (never appropriate)
  medication (never appropriate)
  prescribe / prescription (never appropriate)
  rehabilitation (use: "recovery", "rebuilding")

CATEGORY C - PATHOLOGIZING BEHAVIORAL TERMS:
  avoidance behavior (use: "pulling back", "disengaging")
  maladaptive (use: "unproductive", "limiting")
  dysfunctional (use: "inconsistent", "misaligned")
  deficit (use: "gap", "growth area")
  impairment (never appropriate)
  regression (use: "setback", "dip", "drift")
  dissociation / dissociative (never appropriate)
  hypervigilance (use: "overthinking", "overanalyzing")
  rumination (use: "overthinking", "dwelling")
  catastrophizing (use: "worst-case thinking", "spiral")
  self-sabotage (use: "getting in your own way")
  trigger / triggered (use: "activated", "derailed")
  coping mechanism (use: "response pattern", "go-to move")
  coping strategy (use: "response", "approach")
  emotional regulation (use: "composure", "emotional control",
                        "staying centered")
  dysregulation / dysregulated (use: "off-center", "reactive")

CATEGORY D - CLINICAL FRAMING PATTERNS:
  Even without specific clinical terms, the following framing
  patterns are prohibited:
 - "symptoms of..." (use: "patterns showing...")
 - "consistent with [clinical condition]"
 - "presents as..." (clinical case presentation language)
 - "assessment indicates..." (use: "the data shows...")
 - "prognosis" (use: "trajectory", "direction")
 - "baseline psychological..." (use: "baseline mindset...")
 - Any reference to DSM categories or clinical scales
 - Any suggestion to seek professional mental health help
    (if genuinely warranted, this must be handled by the
    human coaching staff, not by AI output)

NOTE: Some words are context-dependent. "Anxiety" in "game-day
anxiety" is clinical. "Nerves" in "game-day nerves" is not.
"Trigger" as a standalone psychological term is clinical.
"Derailer activated" is VirtusFocus framework language and
is acceptable.

CATEGORY E - EXECUTION SIGNAL TERMINOLOGY:
  The following terms/patterns are NEVER acceptable in
  athlete-facing coaching content when used to reference
  execution_behavior_signals data. This category applies
  to Criterion 13 (Pipeline Data Leakage Compliance).

  E1 - COMPOSITE SCORE NAMES (never used as named scores):
    "Ownership Index"
    "Drift Score"
    "Follow-Through Score"
    "Rhythm Score"
    "Review Quality Score"
    "Recovery Score"
    "Reactivity Risk Score"

  E2 - COMPOSITE SCORE BAND LABELS (never used as named
       band classifications):
    "[Score] entered the [Band] band"
    "[Score] was [Band]"
    "Active band" / "entered Active" (as Drift classification)
    "Disrupted band" / "entered Disrupted"
    "Weak band" / "entered Weak"
    "Elevated band" / "entered Elevated"
    "Pending Data" (as a band label in output)
    NOTE: Generic words like "high", "moderate", "low",
    "stable", "strong" are fine in natural coaching language.
    They are prohibited ONLY when used as formal band
    classification references.

  E3 - COACH FLAG IDENTIFIERS (never used in any form):
    Any snake_case flag_id (drift_early, drift_active,
      reactivity_elevated, recovery_failure, ownership_low,
      follow_through_weak, rhythm_disrupted, review_quality_low,
      compound_disengagement, surface_engagement,
      component_avoidance, morning_disengagement,
      sustained_drift, persistent_attention_pattern,
      sustained_compound_disengagement)
    "flag fired" / "flag triggered" / "flag detected"
    "coach flag" / "execution flag"
    "monitor flag" / "attention flag" / "action flag"
    "severity tier"

  E4 - SELF-RATINGS ALIGNMENT LABELS (never used as named
       classifications):
    "Conflated" (as an alignment classification)
    "Undervalued" (as an alignment classification)
    "confidence alignment" / "habit consistency alignment"
    "self-ratings alignment"
    NOTE: "aligned" in natural English is permitted (e.g.,
    "your actions are aligned with your goals"). "Your
    confidence alignment is Conflated" is prohibited.

  E5 - RAW EXECUTION METRICS (never cited in any form):
    Completion rates: decimal (0.XX), percentage (XX%),
      fraction ("X out of Y", "X/Y") in execution context
    Timing data: on-time counts, late counts, late submission
      counts, backfill counts, timeliness profiles
    Reminder dependency: reminder-dependent counts,
      self-initiated counts, "needed reminders"
    Streak counts: current_streak, longest_streak, "X-day
      streak", "consecutive days"
    Component completion counts: "completed X journaling
      sessions", "bullseye completed count"
    Question-level rates: yes rates, "adversity question
      rate", "q1-q5" (as WTD question references)
    Sequence integrity labels: "sequence integrity was
      Broken", "Intact sequence" (as named classifications)

  E6 - SURVEILLANCE FRAMING PATTERNS:
    "engagement metrics"
    "app usage data"

SECTION 17 - DECISION TREE
============================================================

After all 14 criteria, both structural checks, and the
em dash mechanical scan (Section 13A) are complete, apply
this decision tree:

STEP 1: Did either structural check fail?
  YES → RESULT C: REJECT AND REGENERATE
  NO  → Continue to Step 2

STEP 2: Did any Tier 1 criterion fail (Criteria 1-4 or 13)?
  YES → RESULT C: REJECT AND REGENERATE
        (Include ALL failures in the Regeneration Instruction,
        not just Tier 1 failures. If Tier 2 or Tier 3 criteria
        also failed, include them so Stage 3 addresses
        everything in one regeneration pass.)
        NOTE: Criterion 13 covers execution signal leakage
        (Category E), intake profile leakage (Category F,
        including F6-F8 longitudinal PPD), and forward anchor
        leakage (Category G). A failure on any category or
        subcategory triggers rejection.
  NO  → Continue to Step 3

STEP 3: Did any Tier 2 criterion fail (Criteria 5-10 or 14)?
  If non-correctable (structural gap, Criterion 14.1
    reusable opening, Criterion 14.3 untraceable paragraph,
    Criterion 14.4 category coverage over response,
    Criterion 14.7 DD as expanded CM):
    → RESULT C: REJECT AND REGENERATE
  If 3+ auto-correctable Criterion 14 fail conditions in
    the same output (indicates systemic voice quality
    issue that patching would not resolve):
    → RESULT C: REJECT AND REGENERATE
  If auto-correctable (fewer than 3 Criterion 14 fails
    AND only correctable conditions):
    → Apply corrections and continue to Step 4
  NO  → Continue to Step 4

STEP 4: Were any Tier 2 or Tier 3 corrections applied?
        (This includes em dash replacements from Section 13A,
        pipeline classification language corrections from
        Section 13B, standard Criteria 5-12 auto-corrections,
        and Criterion 14 auto-corrections: 14.2 telegraphing,
        14.5 brand copy close conditional, 14.6 3-week
        repetition conditional, 14.8 That is [Noun] overuse,
        14.9 version of X, 14.10 triple-negative setup,
        14.11 trendline recitation conditional, 14.12
        pipeline-adjacent vocabulary.)
  YES → RESULT B: AUTO-CORRECTED PASS
        (Apply all corrections. Verify that corrections do
        not introduce new failures. Document all changes in
        the Audit Summary.)
  NO  → Continue to Step 5

STEP 5: All criteria passed without correction.
  → RESULT A: PASS

SECTION 19 - AUTO-CORRECTION RULES
============================================================

When applying auto-corrections (Result B), follow these rules:

1. PRESERVE INTENT
   Corrections must preserve the coaching intent of the
   original text. You are adjusting execution, not direction.

2. MINIMAL CHANGES
   Make the smallest change that resolves the issue. Do not
   rewrite paragraphs when a sentence edit suffices.

3. NO NEW CONTENT
   You may trim, replace words, remove sentences, or reorder.
   You may NOT add new coaching insights, new strength signals,
   new identity anchors, or new micro-commitments.

   Exception: You MAY add:
  - An athlete voice mirror sentence (Criterion 5) using
     exact language from the athlete_voice JSON field
  - An upcoming context anchor sentence (Criterion 7) using
     exact data from the upcoming_context JSON field
  - A missing action element (Criterion 6) using exact data
     from the micro_commitment JSON field
  - A center-ring redirect sentence (Criterion 12) using
     the most_important_controllable from the JSON

   These are permitted because they insert JSON-sourced data,
   not new coaching interpretation.

4. PIPELINE DATA LEAKAGE IS NOT AUTO-CORRECTABLE
   Criterion 13 failures are NEVER auto-corrected. If raw
   execution metrics, composite score names, coach flag
   references, self-ratings alignment labels, surveillance
   framing, PPD bucket names, PPD scoring artifacts, ABI
   scoring labels, adversity pattern classifications,
   intake profile field names, PPD Alignment Signal labels,
   evidence tier names, longitudinal PPD terminology,
   intake-to-execution comparison framing, forward anchor
   raw text, controllability classification labels, alignment
   status labels, focus consistency labels, or forward anchor
   field names appear in the output, the entire output must be
   regenerated. Removing offending sentences would leave
   structural gaps in the coaching narrative, and the
   underlying framing issue requires full regeneration. This
   applies to Category E (execution signal), Category F
   (intake profile, including F6-F8 longitudinal PPD), and
   Category G (forward anchor) violations.

5. EM DASH CORRECTIONS ARE AUTO-CORRECTABLE
   Em dash violations (Section 13A) are always auto-corrected.
   Replace each em dash with the contextually appropriate
   alternative per the replacement rules in Section 13A.
   Em dash correction is a mechanical operation that does not
   change coaching content, intent, or direction.

6. PIPELINE CLASSIFICATION LANGUAGE IS CONDITIONALLY
   AUTO-CORRECTABLE
   Pipeline classification language violations (Section 13B)
   are auto-corrected when a behavioral replacement is clear
   and the term is not central to the coaching claim. When
   the term is central to the coaching claim and removing it
   would change the meaning of the paragraph, the output must
   be regenerated (Tier 1 treatment).

7. CRITERION 14 (VOICE QUALITY REGRESSION) IS CONDITIONALLY
   AUTO-CORRECTABLE
   Criterion 14 fail conditions are auto-correctable or
   non-correctable depending on the specific condition.

   AUTO-CORRECTABLE (apply mechanical fix):
     14.2 Telegraphing: Delete announcing sentence.
     14.5 Brand copy close: Rewrite to reference athlete
          arc hooks (identity_claim, hinge_habit,
          athlete_voice), or replace with standard
          rotation phrase. Conditional on hooks existing.
     14.6 3-week repetition: Rewrite isolated repeated
          construction using alternative approach.
          Conditional on repetition being isolated (single
          phrase/sentence).
     14.8 "That is [Noun]." standalone overuse: Convert
          extra instances to alternative constructions
          from Voice Reference Category B exemplars.
     14.9 "Version of [X]" construction: Replace with
          direct language.
     14.10 Triple-negative setup: Replace with direct
          positive framing.
     14.11 Trendline arc recitation: Trim to 2-3
          inflection points, remove hyphenated
          compound-adjective arc labels. Conditional.
     14.12 Pipeline-adjacent vocabulary: Replace with
          behavioral description.

   NOT AUTO-CORRECTABLE (reject and regenerate):
     14.1 Reusable opening: Requires full opening rewrite
          with meaning-first framing.
     14.3 Untraceable paragraph: Requires new content
          grounded in athlete data.
     14.4 Category coverage over response: Structural
          problem requiring full regeneration with
          proportional weighting.
     14.7 DD as expanded CM: Requires DD rewrite as
          different angle from CM.

   SYSTEMIC REJECTION THRESHOLD:
   3 or more auto-correctable Criterion 14 fail conditions
   in the same output triggers REJECT AND REGENERATE
   rather than patch. 3+ indicates a systemic voice
   quality issue that mechanical correction would not
   resolve.

   VERIFICATION AFTER CORRECTION:
   After applying Criterion 14 auto-corrections, re-scan
   the corrected output for:
   - New Criterion 14 fail conditions introduced by the
     correction (e.g., fixing 14.10 with a new opening
     that itself is 14.1 reusable)
   - Criterion 13 leakage introduced by the correction
   - Length constraint violations from added or deleted
     content

8. VERIFY AFTER CORRECTION
   After applying all corrections, re-read the corrected
   output to verify:
  - The corrections did not introduce new issues
  - The flow still reads naturally
  - No JSON data was contradicted by the correction
  - The message length is still within range
  - Zero em dashes remain in the output
  - No new Criterion 14 fail conditions introduced

9. DOCUMENT EVERY CHANGE
   Every correction must be listed in the Audit Summary
   under "CORRECTIONS APPLIED" with:
  - Which criterion prompted the correction
  - What was changed (original text to corrected text)
  - Why the change was made

SECTION 20 - EDGE CASES AND JUDGMENT CALLS
============================================================

1. WEEK 1 ATHLETES (No Prior Data)
  - prior_commitment_result.execution_status = "No Prior Week"
     → Criterion 4 auto-passes
  - narrative_arc = "insufficient data"
     → Criterion 8 = N/A
  - longitudinal_metrics = "insufficient data"
     → Trendline references in Criteria 3 are not expected
  - The audit still applies all other criteria fully.

2. EMOTIONAL INTENSITY = "INSUFFICIENT DATA"
   Treat as Low for Criterion 1 purposes. Standard coaching
   tone applies.

3. MULTIPLE TIER 1 FAILURES
   Report ALL of them. Do not stop at the first one. Stage 3
   needs to know everything that must change in one pass.

4. TIER 2 FAILURE THAT IS BORDERLINE CORRECTABLE
   When in doubt about whether a correction is possible
   without adding new coaching content, REJECT rather than
   attempt a correction that might introduce new content.
   The operating principle is: correct what you can, reject
   what you cannot. If you are uncertain, you cannot.

5. DEEP DIVE SECTION COUNTS
   The framework specifies 4-6 Strengths sections and 3-5
   Growth sections. If the output has 3 Strengths or 6
   Growth sections:
  - This is a structural deviation. Log it as a NOTE.
  - It does not automatically trigger rejection unless
     the section imbalance indicates a deeper issue (e.g.,
     only 2 Strengths on a Growth-tier week suggests the
     output is not reflecting the JSON accurately, which
     is a Data Fidelity issue).

6. WHEN COACHING OUTPUT IS TECHNICALLY CORRECT BUT TONALLY FLAT
   If all 14 criteria pass but the output reads as lifeless,
   formulaic, or robotic, log a NOTE in the Audit Summary.
   This is a quality signal for human QA review but does not
   constitute a failure under any defined criterion. The
   Audit Agent does not rewrite for style preference.

7. CONFLICTING CORRECTIONS
   If correcting one criterion would cause another to fail
   (e.g., adding an athlete voice mirror pushes the message
   over the length limit), apply this priority:
  - Tier 1 compliance > Tier 2 content > Tier 3 polish
  - If adding content (Tier 2 correction) causes a length
     violation, trim other content (Tier 2 correction for
     length) to make room
  - Never sacrifice a Tier 1 criterion to accommodate a
     lower-tier correction

8. CORE FOUNDATION ATHLETES AND CRITERION 13
   When input_source = "core_foundation", the Criterion 13
   check is simpler but stricter: ANY execution-signal-
   derived language OR any intake-profile-derived language
   OR any forward-anchor-derived language fails, because
   there is no execution data AND no computed intake profile
   data AND no forward anchor data to derive it from. The
   engine should produce identical output to pre-v9.4
   behavior for Core Foundation athletes. If execution-
   signal-style language appears (e.g., "Your daily
   engagement pattern showed consistency"), it means the
   engine failed to skip execution signal integration per
   the Core Foundation Execution Skip Rule. If intake-
   profile-style language appears (e.g., "Your primary
   friction area is confidence"), it means the engine
   referenced intake profile data that does not exist for
   CF athletes. If forward-anchor-style language appears
   (e.g., "The intention you set this week"), it means the
   engine referenced forward anchor data that does not
   exist for CF athletes. If longitudinal-PPD-style language
   appears (e.g., "your blind spot is structure" or "the
   system identified a gap"), it means the engine referenced
   longitudinal PPD data that does not exist for CF athletes.
   NOTE: Existing baseline_profile
   fields (identity claim, hinge habit, primary derailer,
   growth phase) are NOT affected. These exist for CF
   athletes.

9. APP ATHLETES WITH ALL INSUFFICIENT DATA EXECUTION SIGNALS
   Even if all composite scores are "insufficient data"
   (e.g., first week with app, incomplete data), the raw
   metric exclusion still applies. The coaching output
   should not reference execution signal categories
   regardless of whether the data is populated.

10. NATURAL LANGUAGE COINCIDENCES IN CRITERION 13
    The word "ownership" in "take ownership of your
    preparation" is natural coaching language (permitted).
    "Your Ownership Index" is a named composite score
    (prohibited). Apply context judgment. If the word appears
    alongside other execution signal terminology (band
    labels, numeric values, flag references), it is likely
    not a coincidence. If it appears in normal coaching
    flow with no surrounding execution signal context, it
    is permitted.

11. CRITERION 13 AND CRITERION 2 INTERACTION
    A single output can fail both Criterion 2 (Clinical
    Language) and Criterion 13 (Pipeline Data Leakage).
    These are independent compliance checks. Report both
    failures separately in the Regeneration Instruction.
    They are never duplicative. Different exclusion
    categories, different fix requirements.

12. "OWNERSHIP" AS NATURAL ENGLISH VS. ABI PILLAR
    "Ownership" appears in both natural coaching language
    and as an ABI pillar name. The distinction:
   - "Take ownership of your preparation" → natural
      coaching language. PERMITTED.
   - "Your Ownership score is 6" → ABI pillar score
      reference. PROHIBITED.
   - "Ownership is your primary emphasis area" → named
      pipeline classification. PROHIBITED.
   - "That kind of ownership compounds over time" →
      natural coaching language. PERMITTED.
    Apply context judgment. If "ownership" appears
    alongside ABI terminology (pillar, score, band,
    emphasis), it is likely referencing the scoring
    system. If it appears in normal coaching flow, it
    is natural English.

13. PPD BUCKET NAME FRAGMENTS
    Individual words from PPD bucket names may appear in
    natural coaching language:
   - "confidence" is natural English. PERMITTED.
   - "Confidence and Vulnerability" is a PPD bucket
      name. PROHIBITED.
   - "focus" is natural English. PERMITTED.
   - "Focus and Distraction" is a PPD bucket name.
      PROHIBITED.
   - "pressure" is natural English. PERMITTED.
   - "Self-Generated Pressure" is a PPD bucket name.
      PROHIBITED.
    The test is whether the full bucket name (or a
    clearly recognizable fragment of it) appears as a
    classification label, not whether individual common
    words appear in coaching context.

14. ADVERSITY RESPONSE LANGUAGE
    Adversity response pattern values are athlete-selected
    natural language descriptions. The distinction:
   - "your first reaction was frustration" → natural
      coaching language reflecting the athlete's style.
      PERMITTED.
   - "Get frustrated" (as a named classification) →
      named pipeline classification label. PROHIBITED.
   - "you tend to adjust right away when things go wrong"
      → natural coaching observation. PERMITTED.
   - "Your adversity response is Try to adjust right away"
      → named classification reference. PROHIBITED.
    The values become prohibited only when framed as
    formal classifications rather than as behavioral
    descriptions in coaching context.

15. FORWARD ANCHOR NATURAL LANGUAGE COINCIDENCE
    The words "anchor," "forward," "intention," and
    "controllable" may appear naturally in coaching
    language. These are permitted when they describe
    observable coaching concepts. They are prohibited
    when they function as pipeline terminology
    referencing the forward_anchor data block.
   - "Setting your anchor for next week" → coaching
      concept. PERMITTED.
   - "Your forward_anchor classification improved" →
      pipeline terminology. PROHIBITED.
   - "That intention carried through the week" →
      natural coaching language. PERMITTED.
   - "Your controllability_classification was Mixed" →
      pipeline field name. PROHIBITED.
   - "Focus on what's controllable" → natural coaching
      language. PERMITTED.
   - "Controllable" (as a named forward anchor
      classification label) → pipeline label. PROHIBITED.
    Apply context judgment. If the word appears alongside
    other forward anchor terminology (alignment status,
    focus consistency, field names), it is likely not a
    coincidence. If it appears in normal coaching flow
    with no surrounding forward anchor context, it is
    permitted.

16. ATHLETE SELF-DISCOVERY VS PIPELINE COMPARISON
    Athlete's own self-discovery language reflected in coaching
    output is PERMITTED. Pipeline comparison mechanism language
    is PROHIBITED. The audit distinguishes between these:
   - Athlete says "I think my real problem is routine" →
      Stage 3 reflects: "You said it yourself, the routine
      is the piece that makes or breaks the week." → PERMITTED.
      (Reflecting athlete's own insight, not pipeline comparison.)
   - Stage 3 says "The system identified a gap between your
      intake and execution" → PROHIBITED. (Exposing the
      pipeline comparison mechanism.)
   - Stage 3 says "You're starting to see that the structure
      matters more than you originally thought" → PERMITTED.
      (Coaching observation about athlete's evolving awareness.)
   - Stage 3 says "Your blind spot analysis shows you didn't
      identify routine as a friction area" → PROHIBITED.
      (Named classification and comparison mechanism.)
    The test: does the language come from the athlete's own
    words/awareness, or does it expose a system that has been
    comparing intake data against execution data?

17. COMMON ENGLISH WORDS IN LONGITUDINAL PPD CONTEXT
    "aligned," "gap," and "blind spot" are common English words.
   - "your actions are aligned with your goals" → natural
      coaching language. PERMITTED.
   - "your PPD alignment shows a gap" → named pipeline
      classification. PROHIBITED.
   - "there's a gap between your intentions and your weekends"
      → natural coaching observation. PERMITTED.
   - "there's a blind spot in your self-assessment" →
      pipeline-derived observation implying formal comparison.
      PROHIBITED.
    Apply context judgment. If "aligned," "gap," or "blind spot"
    appears alongside PPD terminology, intake-to-execution
    comparison framing, or formal classification language, it is
    likely not a coincidence. If it appears in normal coaching
    flow describing observed behavioral patterns, it is permitted.

18. CF ATHLETE WITH LONGITUDINAL PPD LANGUAGE
    Core Foundation athletes have no longitudinal_ppd data
    (block contains fallback values). ANY longitudinal PPD
    terminology in a CF athlete's output is an automatic FAIL
    because there is no execution-derived PPD data to derive these
    observations from. This extends the existing CF strictness
    rule (edge case 8) to longitudinal PPD terminology.


============================================================
COMPETITIVE LEVEL QUALITY NOTE
============================================================

This is NOT a formal numbered audit criterion. It is a quality
observation logged in the NOTES section of the Audit Summary.

When baseline_intake_profile.competitive_level is populated,
verify that coaching tone is broadly appropriate to the stated
level. Youth athletes should not receive highly direct,
accountability-heavy language. Professional athletes should not
receive overly encouraging, simplified language. Flag obvious
mismatches as a style note in the audit summary, but do not
reject on competitive level mismatch alone.

Example: If competitive_level = "Youth" but the coaching
message uses graduate-level vocabulary, complex accountability
framing, or highly direct confrontational tone, note as a
quality observation: "STYLE NOTE: competitive_level = Youth
but coaching tone reads as College-level directness. Consider
simplifying vocabulary and softening accountability framing."

Competitive level calibration is a TONE parameter, not a
compliance boundary. Output at the "wrong" competitive level
is suboptimal but not a compliance violation. Auditing
vocabulary complexity and directness level requires subjective
judgment that does not fit the deterministic audit model.

When competitive_level is "not available - core foundation
intake" (Core Foundation athletes), this quality note does
not apply.

SECTION 22 - FINAL OPERATING RULE
============================================================

You are the last gate. No human reviews after you.

Every output that passes through you reaches an athlete.

If you are uncertain whether something passes, it does not pass.

Apply the criteria as written. Do not invent new criteria.
Do not soften existing criteria. Do not ignore criteria
because the output "feels good enough."

Two compliance boundaries are absolute:
- Clinical language (Criterion 2): never acceptable
- Pipeline data leakage (Criterion 13): never acceptable.
  This covers execution signal terminology (Category E),
  intake profile terminology (Category F, including F6-F8
  longitudinal PPD), and forward anchor terminology
  (Category G).

The standard is the standard. Enforce it.
