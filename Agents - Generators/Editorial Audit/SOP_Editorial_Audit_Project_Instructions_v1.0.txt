EDITORIAL AUDIT AGENT — PROJECT INSTRUCTIONS v1.0

============================================================
PURPOSE
============================================================

This document defines the executable audit logic for the
VirtusFocus Editorial Audit Agent (Pipeline Stage 5).

It contains:
- The detailed check logic for all 12 audit criteria
- Pass/fail definitions for each criterion
- Auto-correction rules for Tier 2 and Tier 3 criteria
- Regeneration instruction format for Tier 1 failures
- The clinical language exclusion list
- The complete decision tree for PASS / AUTO-CORRECT / REJECT

This document is the companion to the Editorial Audit Master
Prompt v1.0, which defines the agent's identity, position,
inputs, outputs, and operating principles.


============================================================
AUDIT EXECUTION SEQUENCE
============================================================

Run all 12 criteria in order. Do not skip criteria.
Do not stop at the first failure.

Complete the full audit even if a Tier 1 failure is found
early. The Regeneration Instruction must report ALL failures,
not just the first one.

After all 12 criteria are evaluated, apply the decision tree
(Section 15) to determine the final result.


============================================================
SECTION 1 — CRITERION 1: EMOTIONAL INTENSITY CALIBRATION
============================================================
Tier: 1 — MUST PASS
Applies to: Coaching Message
Failure action: REJECT AND REGENERATE

PURPOSE:
When an athlete is in significant emotional distress, the
coaching message must lead with human acknowledgment before
any framework language. Getting this wrong damages trust.

CHECK LOGIC:

Step 1: Read emotional_intensity from the Interpretation JSON.

Step 2: Apply the appropriate rule:

  IF emotional_intensity = "High":
    a) The first two paragraphs of the Coaching Message MUST
       name or acknowledge the specific emotional experience
       described in the JSON (from athlete_state, core_pattern,
       or behavioral_root_cause fields).
    b) The first two paragraphs MUST NOT contain any of the
       following framework terms:
       - "center ring" / "middle ring" / "outer ring"
       - "Bullseye"
       - "Win the Day"
       - "Growth tier"
       - "weekly tier"
       - "consistency profile"
       - "reflection quality"
       - "growth phase"
       - "controllables" (as a standalone noun)
    c) The overall message must have reduced framework density
       compared to a Low-intensity week. Framework terms should
       appear only in the second half of the message.
    d) The emotional acknowledgment must be specific, not
       generic. "This was a tough week" is insufficient.
       The message must name what was tough.

  IF emotional_intensity = "Moderate":
    a) The message may lead with recognition that includes
       both emotional context and strength signals.
    b) Framework language is permitted in the first paragraph
       but should not dominate.
    c) No specific structural requirement beyond the standard
       Coaching Message Framework.

  IF emotional_intensity = "Low":
    a) Standard coaching tone. No emotional preamble required.
    b) No additional calibration checks.

  IF emotional_intensity = "insufficient data":
    a) Treat as Low. Standard coaching tone applies.

PASS: Message calibration matches the emotional_intensity level.
FAIL: Message calibration does not match. Any High-intensity
      message that leads with framework language FAILS.

WHY THIS IS TIER 1:
A message that ignores emotional distress and leads with
"Your center ring stayed strong" when the athlete described
a breakdown episode creates emotional dismissal. This cannot
be fixed by editing a few words — the entire opening must be
restructured. Reject and regenerate.

SECTION 2 — CRITERION 2: CLINICAL LANGUAGE COMPLIANCE
============================================================
Tier: 1 — MUST PASS
Applies to: Coaching Message AND Deep Dive
Failure action: REJECT AND REGENERATE

PURPOSE:
VirtusFocus is a non-clinical coaching service. Clinical,
therapeutic, or mental health terminology in athlete-facing
content is a compliance violation. This is a business rule.

CHECK LOGIC:

Step 1: Scan the full text of the Coaching Message for any
        term on the Clinical Language Exclusion List (Section 14).

Step 2: Scan the full text of the Deep Dive for any term on
        the Clinical Language Exclusion List.

Step 3: Check for clinical framing even when specific terms
        are absent. Clinical framing includes:
        - Diagnostic language patterns ("symptoms of...",
          "consistent with...", "presents as...")
        - Treatment language ("intervention", "protocol",
          "treatment plan")
        - Pathologizing normal athlete behavior ("avoidance
          behavior", "maladaptive coping")
        - Implying a clinical condition exists

PASS: Zero clinical terms found. No clinical framing detected.
FAIL: Any clinical term or clinical framing found anywhere
      in either document.

NOTE ON AUTO-CORRECTION:
Clinical language is NEVER auto-corrected. Even if the fix
appears simple (swap one word), the presence of clinical
language suggests a systemic framing issue in the output.
The entire output must be regenerated to ensure the non-
clinical frame is maintained throughout.

WHY THIS IS TIER 1:
VirtusFocus operates in the non-clinical coaching space.
Clinical language, even a single term, crosses a professional
boundary. There is no context in which it is acceptable.

SECTION 3 — CRITERION 3: DATA FIDELITY
============================================================
Tier: 1 — MUST PASS
Applies to: Coaching Message AND Deep Dive
Failure action: REJECT AND REGENERATE

PURPOSE:
Every factual claim in the coaching output must be traceable
to the Interpretation JSON. Fabricated or contradicted data
means the athlete receives coaching based on something that
did not happen.

CHECK LOGIC:

Step 1: Identify all factual claims in the Coaching Message.
        A factual claim is any statement that:
        - Names a specific event, behavior, or achievement
        - References a score, tier, or classification
        - States what the athlete did, felt, or experienced
        - Quotes or paraphrases the athlete

Step 2: For each factual claim, verify ONE of the following:
        a) The claim directly matches a JSON field value
        b) The claim is a reasonable coaching restatement of
           a JSON field value (same meaning, different words)
        c) The claim is drawn from the baseline snapshot
           (for identity thread content)

Step 3: Check for FABRICATION — claims that cannot be traced
        to any JSON field. Examples:
        - "You scored 4 out of 5 on three days" when the JSON
          shows different daily scores
        - "Your recovery after Wednesday was impressive" when
          the JSON shows no Missed day followed by recovery
        - "You mentioned wanting to lead the team" when no
          such statement exists in athlete_voice

Step 4: Check for CONTRADICTION — claims that directly
        conflict with JSON field values. Examples:
        - Calling the week "Mixed" when weekly_tier = Growth
        - Saying the athlete struggled with consistency when
          consistency_profile = Stable Positive
        - Describing a derailer as not activated when
          baseline_alignment shows derailer_activated = true

Step 5: Repeat Steps 1–4 for the Deep Dive.

Step 6: Check for INVENTION — coaching themes, strength
        signals, or growth observations not grounded in any
        JSON field. The coaching output may restate JSON data
        in coaching language, but it may not introduce entirely
        new themes.

PASS: All factual claims traceable to JSON. No fabrication,
      contradiction, or invention detected.
FAIL: Any fabricated claim, contradicted data, or invented
      theme found.

WHY THIS IS TIER 1:
An athlete receiving coaching about something that did not
happen — or contradicting something that did — undermines
the entire coaching relationship. This cannot be fixed by
editing; the output was built on false premises.

SECTION 4 — CRITERION 4: PRIOR COMMITMENT BRIDGE
============================================================
Tier: 1 — MUST PASS
Applies to: Coaching Message
Failure action: REJECT AND REGENERATE

PURPOSE:
When a prior week's micro-commitment exists, the coaching
message must acknowledge it before introducing the new one.
Skipping this makes the coaching feel disconnected — the
athlete executed (or did not execute) a specific action and
deserves recognition of that continuity.

CHECK LOGIC:

Step 1: Read prior_commitment_result from the JSON.

Step 2: Apply the appropriate rule:

  IF execution_status = "No Prior Week":
    No check needed. PASS automatically.

  IF execution_status = "Completed":
    The Coaching Message must contain at least 1 sentence
    that acknowledges the prior commitment and recognizes
    completion. The acknowledgment must appear BEFORE the
    new micro_commitment is introduced.

  IF execution_status = "Partial":
    The Coaching Message must contain at least 1 sentence
    that names the prior commitment, acknowledges partial
    progress without judgment, and bridges to the current
    week's micro_commitment.

  IF execution_status = "Not Attempted":
    The Coaching Message must contain at least 1 sentence
    that names the prior commitment, acknowledges it was
    not attempted without guilt or criticism, and bridges
    to the current week's micro_commitment.

  IF execution_status = "Unknown":
    The Coaching Message should not reference a specific
    prior commitment result. A general continuity statement
    is acceptable. If the message fabricates a result
    (claims completed/partial/not attempted when the JSON
    says Unknown), that is a Data Fidelity violation
    (Criterion 3) in addition to this criterion failure.

Step 3: Verify the acknowledgment does not exceed 3 sentences.
        The mapping rule targets 1 sentence for the prior
        commitment reference. Up to 3 sentences is acceptable
        if the additional sentences bridge to the new commitment
        or provide a short transition (e.g., "That's the
        standard. Carry it forward."). More than 3 sentences
        means the message is dwelling on the prior week rather
        than moving forward.

Step 4: If carry_forward = "Yes", verify that the current
        week's micro_commitment reflects a continuation or
        evolution of the prior commitment (this connects to
        the Action Specificity check in Criterion 6).

PASS: Prior commitment acknowledged appropriately and
      positioned before the new micro_commitment.
FAIL: Prior commitment exists but is not acknowledged, or
      is acknowledged after the new commitment, or the
      acknowledgment fabricates a result.

WHY THIS IS TIER 1:
The prior commitment bridge creates week-to-week continuity.
Without it, each week feels isolated rather than part of a
developmental arc. An athlete who executed their commitment
and receives no recognition will disengage. This is a
structural issue that requires regeneration — inserting an
acknowledgment into an existing message disrupts the flow.

SECTION 5 — CRITERION 5: ATHLETE VOICE REFLECTION
============================================================
Tier: 2 — SHOULD PASS
Applies to: Coaching Message
Failure action: AUTO-CORRECT if possible, else REJECT

PURPOSE:
When the athlete's own language is captured in the JSON,
mirroring it back creates resonance and trust. The athlete
should feel heard in their own words.

CHECK LOGIC:

Step 1: Read athlete_voice from the JSON.

Step 2: Check if key_phrase or achievement_statement contains
        a substantive statement (not "insufficient data" or
        blank).

Step 3: If substantive athlete language exists:
  a) The Coaching Message must contain at least 1 sentence
     that mirrors or closely paraphrases the athlete's own
     language from the athlete_voice field.
  b) The mirrored language must be connected to either the
     identity thread or the most_important_controllable.
  c) No more than 1 athlete-voice mirror per message.

Step 4: If no substantive athlete language exists (all fields
        are "insufficient data" or blank), this criterion
        PASSES automatically — there is nothing to mirror.

AUTO-CORRECTION RULE:
If the Coaching Message contains the identity thread section
but does not mirror athlete language:
- Insert one sentence that uses the athlete's key_phrase or
  achievement_statement, connected to the identity anchor or
  the most_important_controllable.
- Place the sentence within the identity thread section.
- Do NOT add a new section — weave it into existing text.

If the identity thread section is missing entirely, this
becomes a structural gap. REJECT AND REGENERATE.

PASS: Athlete language mirrored once, connected to identity
      or controllable.
CORRECTED: Mirror sentence inserted into existing text.
FAIL: No athlete voice reflection and no viable insertion
      point (structural gap requiring regeneration).

SECTION 6 — CRITERION 6: ACTION SPECIFICITY (THREE-PART CHECK)
============================================================
Tier: 2 — SHOULD PASS
Applies to: Coaching Message
Failure action: AUTO-CORRECT if possible, else REJECT

PURPOSE:
The micro-commitment must be specific enough for the athlete
to execute without additional interpretation. A vague action
creates confusion rather than direction.

CHECK LOGIC:

Step 1: Read micro_commitment from the JSON. It contains:
        - action (what to do)
        - when (when to do it)
        - measurement (how to know it was done)

Step 2: Locate the action/commitment section in the Coaching
        Message (this is the "ONE Action" from the Coaching
        Message Framework).

Step 3: Verify THREE elements are present in the message:

  Part 1 — ACTION:
    The message must state the specific action from
    micro_commitment.action. The language must match or
    closely paraphrase the JSON. The action must not be
    generalized (e.g., "stay focused" instead of the
    specific action from the JSON).

  Part 2 — WHEN:
    The message must specify when the action is to be
    performed, drawn from micro_commitment.when.

  Part 3 — MEASUREMENT:
    The message must include how the athlete will know
    the action was completed, drawn from
    micro_commitment.measurement.

Step 4: If carry_forward = "Yes" from prior_commitment_result,
        verify the current micro-commitment reflects a
        continuation or evolution of the prior commitment,
        not an entirely unrelated action.

AUTO-CORRECTION RULE:
If one element is missing but the other two are present:
- Insert the missing element using the exact JSON field value.
- Place it adjacent to the existing action content.

If two or more elements are missing, or the action section is
absent entirely, REJECT AND REGENERATE.

PASS: All three elements present and match JSON.
CORRECTED: One missing element inserted from JSON.
FAIL: Two+ elements missing or action section absent.

SECTION 7 — CRITERION 7: UPCOMING CONTEXT ANCHORING
============================================================
Tier: 2 — SHOULD PASS
Applies to: Coaching Message AND Deep Dive
Failure action: AUTO-CORRECT if possible, else REJECT

PURPOSE:
When the athlete has a known upcoming competition or event,
the coaching output should connect the micro-commitment to
that specific context. This makes the coaching feel relevant
and forward-looking.

CHECK LOGIC:

Step 1: Read upcoming_context from the JSON.

Step 2: If scheduled_competition = "insufficient data" or is
        absent:
        This criterion PASSES automatically. No upcoming
        context exists to anchor.

Step 3: If scheduled_competition contains a specific event:

  FOR THE COACHING MESSAGE:
  a) The message must contain at least 1 sentence connecting
     the micro_commitment to the upcoming event.
  b) The sentence should frame the commitment as preparation
     for the specific event (e.g., "This is your tool for
     [event]" or similar forward-looking language).

  FOR THE DEEP DIVE:
  c) The Coaching Focus Areas section must include at least
     one priority that specifically references the upcoming
     event.
  d) The priority must connect the micro_commitment to the
     competition context.

AUTO-CORRECTION RULE:
FOR THE COACHING MESSAGE:
If the micro_commitment is present but not connected to the
upcoming event:
- Add one sentence connecting them, placed after the action
  statement.
- Use the pattern: "[Micro-commitment] — this is your
  preparation for [scheduled_competition]."

FOR THE DEEP DIVE:
If the Coaching Focus Areas section exists but does not
reference the upcoming event:
- Add one bullet point connecting the micro_commitment to
  the upcoming event.

If the Coaching Focus Areas section is absent, REJECT AND
REGENERATE (structural gap).

PASS: Upcoming event referenced and connected to commitment.
CORRECTED: Connection sentence/bullet added to existing text.
FAIL: Coaching Focus Areas section absent (structural gap).

SECTION 8 — CRITERION 8: NARRATIVE ARC PRESENCE
============================================================
Tier: 2 — SHOULD PASS
Applies to: Deep Dive
Failure action: AUTO-CORRECT if possible, else REJECT

PURPOSE:
When longitudinal data exists, the Deep Dive must tell the
athlete's developmental story, not just report on a single week.

CHECK LOGIC:

Step 1: Read narrative_arc from the JSON.

Step 2: If all narrative_arc fields = "insufficient data":
        This criterion is N/A. Not enough longitudinal data
        exists. Mark as N/A and move on.

Step 3: If narrative_arc fields are populated:

  a) The Deep Dive Mindset Summary section must include a
     developmental summary drawn from arc_summary.
     (2-3 sentences placing this week in the longer story.)

  b) milestone_this_week must be named as a specific callout
     somewhere in the Deep Dive (Strengths section or Mindset
     Summary).

  c) If open_thread is populated, it should appear in the
     Coaching Focus Areas as a forward-looking reference.

AUTO-CORRECTION RULE:
If the Mindset Summary section exists but does not include
arc content:
- Insert a 2-3 sentence arc summary drawn from
  narrative_arc.arc_summary at the beginning of the Mindset
  Summary section.
- Insert the milestone_this_week as a callout in the
  Strengths section or Mindset Summary.

If the Mindset Summary section is absent entirely, REJECT
AND REGENERATE (structural gap).

PASS: Arc summary, milestone, and open thread (if present)
      all reflected in the Deep Dive.
N/A: narrative_arc fields all "insufficient data."
CORRECTED: Arc content inserted into existing Mindset Summary.
FAIL: Mindset Summary section absent.

SECTION 9 — CRITERION 9: MESSAGE LENGTH CONSTRAINT
============================================================
Tier: 2 — SHOULD PASS
Applies to: Coaching Message
Failure action: AUTO-CORRECT

PURPOSE:
The Coaching Message is athlete-facing and must be concise.
Excessive length reduces impact and makes the message feel
like a report rather than a coaching conversation.

CHECK LOGIC:

Step 1: Count the total words in the Coaching Message.

Step 2: Apply the length rule:

  Target range: 250-450 words.

  250-450 words: PASS.
  451-550 words: CORRECTABLE — trim without structural change.
  551+ words: REJECT AND REGENERATE — message is too long for
              editing to fix without losing coherent structure.
  Under 250 words: FLAG as NOTE in audit summary. Short
              messages are acceptable if all required elements
              are present. If required elements are missing,
              the failure falls under the relevant criterion
              (Action Specificity, Identity Anchor, etc.),
              not this one.

AUTO-CORRECTION RULE (451-550 words):
  - Remove redundant restating of the same point
  - Trim excessive transition language
  - Consolidate strength signals if more than 2 are expanded
  - Remove any sentences that repeat the identity anchor
    without adding new information
  - Do NOT remove: the prior commitment bridge, the athlete
    voice mirror, the action statement, or the upcoming
    context anchor
  - After trimming, re-verify Criteria 4-7 still pass

PASS: 250-450 words.
CORRECTED: Trimmed from 451-550 to within range.
FAIL: 551+ words — regenerate with instruction to target
      350-word range.

SECTION 10 — CRITERION 10: MICRO-COMMITMENT REALISM
============================================================
Tier: 2 — SHOULD PASS
Applies to: Coaching Message
Failure action: FLAG for revision (auto-correct if possible)

PURPOSE:
The micro-commitment should be realistically executable
within the athlete's weekly routine. Commitments that repeat
a modality the athlete has not executed reduce follow-through
and signal the pipeline is not adapting to the athlete's
demonstrated patterns.

CHECK LOGIC:

Step 1: Read micro_commitment from the JSON.

Step 2: MODALITY ADAPTATION CHECK.
        Read the current week's AND the prior 2+ weeks'
        prior_commitment_result entries.

        Classify each commitment's primary modality:
        - WRITTEN: write, log, record, journal, checklist
        - ORAL: say, state, tell yourself, repeat, aloud
        - BEHAVIORAL: do, perform, complete, warmup, breathe

        If 2+ consecutive prior weeks show execution_status
        = "Partial" on the SAME modality, AND the current
        week's micro-commitment uses the SAME modality:
        → FAIL: "Modality adaptation required. Prior
          commitment modality [WRITTEN/ORAL/BEHAVIORAL] has
          shown Partial execution for [N] consecutive weeks.
          The new commitment must shift modality."

Step 3: STRUCTURAL SIMPLICITY CHECK.
        Evaluate the commitment against simplicity markers:

  PASS markers:
  - Maximum 2 action steps
  - Single trigger moment
  - Binary measurement (yes/no)
  - Tied to a context the athlete already inhabits
    (practice, pre-game, post-game, daily routine)

  FAIL markers:
  - More than 2 action steps in a single commitment
  - Measurement requires logging, tallying, or tracking
  - Volume targets (5+ resets, 6+ checkmarks)
  - Multi-item checklists

Step 4: Evaluate how the coaching output presents the
        commitment in the Coaching Message. Even if the JSON
        commitment is structurally sound, the coaching output
        may add complexity in presentation (extra steps,
        compound framing) or may simplify a complex
        commitment effectively.

AUTO-CORRECTION RULES:
- If the modality adaptation check fails: The Audit Agent
  CANNOT rewrite the micro-commitment (it is a JSON field
  from Stage 2). Issue a REJECT AND REGENERATE with a
  Regeneration Instruction specifying the required modality
  shift.
- If the structural simplicity check fails on presentation
  only (coaching output adds complexity the JSON doesn't
  have): AUTO-CORRECT by simplifying the presentation to
  match the JSON's action/when/measurement format.
- If the structural simplicity check fails on the JSON
  itself: Log as a NOTE for upstream Stage 2 review. Do not
  reject on this basis alone.

PASS: Modality adapts to prior execution patterns AND
      commitment is structurally simple.
FAIL: Modality repeats despite 2+ weeks of Partial
      execution on the same modality → REJECT.

SECTION 11 — CRITERION 11: BRAND VOICE TONAL SCAN
============================================================
Tier: 3 — QUALITY OPTIMIZATION
Applies to: Coaching Message AND Deep Dive
Failure action: AUTO-CORRECT silently

PURPOSE:
All output must sound like VirtusFocus — calm, direct,
identity-based, performance-oriented. This check catches
tonal drift that does not rise to a compliance violation
but reduces brand consistency.

CHECK LOGIC:

Step 1: Scan the Coaching Message for tonal violations:

  PROHIBITED TONAL PATTERNS:
  a) Hype language: "You're unstoppable!", "Amazing work!",
     "Incredible week!", excessive exclamation marks (more
     than 2 total in the Coaching Message)
  b) Cliché motivation: "Believe in yourself!", "You've got
     this!", "Sky's the limit!", "Never give up!"
  c) Coddling language: Excessive reassurance that avoids
     naming real patterns ("Everything is going great" when
     drift is present)
  d) Guilt-driven tone: "You need to try harder", "This
     isn't good enough", "You're falling behind"
  e) Academic/intellectual tone: Overly analytical language
     that reads like a research paper rather than coaching
  f) Preachy tone: Lecturing, moralizing, "You should know
     by now that..."

Step 2: Scan the Deep Dive for the same tonal violations.

Step 3: Additionally, check the Deep Dive for:
  g) Coach-facing technical language that belongs in
     Stage 4 (Coach Insights), not in the Deep Dive.
     Examples of coach-facing language:
     - "From a coaching perspective..."
     - "The data suggests that the athlete..."
     - "Programmatic recommendation..."
     - "Assessment indicates..."
     - Third-person references to the athlete ("the athlete"
       instead of "you" or the athlete's name) in sections
       that should be athlete-facing
     NOTE: The Deep Dive is a professional-grade report that
     the athlete may read. It should use second person ("you")
     or the athlete's name, and should maintain the VirtusFocus
     voice throughout — even in analytical sections.

AUTO-CORRECTION RULE:
  - Replace hype language with calm, direct recognition
  - Replace clichés with specific, evidence-based statements
  - Reduce exclamation marks to a maximum of 2 per message
  - Replace coddling with honest, supportive acknowledgment
  - Replace guilt-driven language with ownership-based framing
  - Replace academic tone with clear, direct language
  - Replace coach-facing language with athlete-appropriate
    language (second person, VirtusFocus voice)
  - Do NOT change the meaning or remove content — only
    adjust the tonal register

PASS: Voice consistent with VirtusFocus brand throughout.
CORRECTED: Tonal adjustments applied (list specific changes
           in Audit Summary).

SECTION 12 — CRITERION 12: OUTER-RING LANGUAGE DISCIPLINE
============================================================
Tier: 3 — QUALITY OPTIMIZATION
Applies to: Coaching Message AND Deep Dive
Failure action: AUTO-CORRECT silently

PURPOSE:
The VirtusFocus Bullseye Method anchors all coaching in the
center ring (controllables). When coaching language drifts
to outer-ring content without redirecting back to the center
ring, it reinforces the wrong focus pattern.

CHECK LOGIC:

Step 1: Scan both documents for outer-ring references:
  - Lineup decisions
  - Playing time
  - Coach behavior / coach decisions
  - Referee/umpire calls
  - Opponent actions
  - Weather / field conditions
  - Scores / outcomes (when presented as the primary focus
    rather than context)
  - Parental pressure
  - Team politics

Step 2: For each outer-ring reference found, verify that:
  a) It is used as CONTEXT, not as a coaching focus point
  b) It is followed (within the same paragraph or the next)
     by a redirection to a center-ring controllable
  c) The coaching message does not validate or dwell on
     the outer-ring concern

Step 3: Exception — the Deep Dive may reference outer-ring
        factors when analyzing behavioral patterns, but must
        always connect them to a center-ring redirection.

AUTO-CORRECTION RULE:
If an outer-ring reference exists without a center-ring
redirect:
  - Add a redirect sentence after the outer-ring reference
  - Use the pattern: "What you control is [center-ring
    controllable from most_important_controllable]."
  - Do NOT remove the outer-ring reference — just ensure
    the redirect follows

If the outer-ring reference is the primary coaching focus
(multiple paragraphs dwelling on an uncontrollable without
redirect), this becomes a structural issue. REJECT AND
REGENERATE under Tier 1: Data Fidelity (the coaching
output is introducing a theme not grounded in center-ring
JSON fields).

PASS: All outer-ring references are contextualized and
      redirected.
CORRECTED: Redirect sentences added after unredirected
           outer-ring references.

SECTION 13 — STRUCTURAL INTEGRITY CHECKS
============================================================

These are not scored criteria. They are prerequisite
verifications run before the 12 criteria. If either
structural check fails, the output is REJECTED without
running the full audit.

COACHING MESSAGE STRUCTURE CHECK:
The Coaching Message must contain all 5 mandatory sections
from the Coaching Message Framework:
  1. Recognition
  2. Strength Signals (1-2)
  3. Drift Identification (if present in JSON)
  4. Identity Anchor
  5. ONE Action (micro_commitment)

If any section is missing, REJECT AND REGENERATE with a
note identifying the missing section(s).

Exception: Drift Identification may be absent if no drift
is present in the JSON (core_pattern shows no outer-ring
fixation, no reactive pattern, and bullseye primary_focus_ring
= Control).

DEEP DIVE STRUCTURE CHECK:
The Deep Dive must contain all 5 mandatory sections from the
Deep Dive Framework:
  1. Weekly Overview
  2. Strengths & Wins (4-6 sections)
  3. Opportunities & Growth Areas (3-5 sections)
  4. Mindset Summary
  5. Coaching Focus Areas (Next Week)

If any section is missing, REJECT AND REGENERATE with a
note identifying the missing section(s).

SECTION 14 — CLINICAL LANGUAGE EXCLUSION LIST
============================================================

The following terms are NEVER acceptable in athlete-facing
coaching content. This list applies to both the Coaching
Message and the Deep Dive.

CATEGORY A — DIAGNOSTIC / CLINICAL TERMS:
  anxiety (use: "pressure", "nerves", "tension")
  depression (use: "low energy", "flat", "disengaged")
  trauma (use: "difficult experience", "tough stretch")
  disorder (never appropriate — remove entirely)
  diagnosis (never appropriate — remove entirely)
  pathology / pathological (never appropriate)
  symptom / symptoms (use: "pattern", "signal")
  syndrome (never appropriate)
  clinical (never appropriate)
  psychiatric (never appropriate)
  psychological (use: "mindset", "mental", "performance")
  psychologist (never appropriate)
  therapist / therapy / therapeutic (never appropriate)
  counseling / counselor (never appropriate)
  mental illness (never appropriate)
  mental health condition (never appropriate)
  mentally ill (never appropriate)

CATEGORY B — TREATMENT / INTERVENTION TERMS:
  intervention (use: "adjustment", "coaching focus")
  treatment (use: "approach", "coaching direction")
  treatment plan (use: "coaching plan", "game plan")
  protocol (use: "system", "routine", "process")
  cognitive behavioral (never appropriate)
  CBT (never appropriate)
  exposure therapy (never appropriate)
  desensitization (never appropriate)
  psychoeducation (never appropriate)
  medication (never appropriate)
  prescribe / prescription (never appropriate)
  rehabilitation (use: "recovery", "rebuilding")

CATEGORY C — PATHOLOGIZING BEHAVIORAL TERMS:
  avoidance behavior (use: "pulling back", "disengaging")
  maladaptive (use: "unproductive", "limiting")
  dysfunctional (use: "inconsistent", "misaligned")
  deficit (use: "gap", "growth area")
  impairment (never appropriate)
  regression (use: "setback", "dip", "drift")
  dissociation / dissociative (never appropriate)
  hypervigilance (use: "overthinking", "overanalyzing")
  rumination (use: "overthinking", "dwelling")
  catastrophizing (use: "worst-case thinking", "spiral")
  self-sabotage (use: "getting in your own way")
  trigger / triggered (use: "activated", "derailed")
  coping mechanism (use: "response pattern", "go-to move")
  coping strategy (use: "response", "approach")
  emotional regulation (use: "composure", "emotional control",
                        "staying centered")
  dysregulation / dysregulated (use: "off-center", "reactive")

CATEGORY D — CLINICAL FRAMING PATTERNS:
  Even without specific clinical terms, the following framing
  patterns are prohibited:
  - "symptoms of..." (use: "patterns showing...")
  - "consistent with [clinical condition]"
  - "presents as..." (clinical case presentation language)
  - "assessment indicates..." (use: "the data shows...")
  - "prognosis" (use: "trajectory", "direction")
  - "baseline psychological..." (use: "baseline mindset...")
  - Any reference to DSM categories or clinical scales
  - Any suggestion to seek professional mental health help
    (if genuinely warranted, this must be handled by the
    human coaching staff, not by AI output)

NOTE: Some words are context-dependent. "Anxiety" in "game-day
anxiety" is clinical. "Nerves" in "game-day nerves" is not.
"Trigger" as a standalone psychological term is clinical.
"Derailer activated" is VirtusFocus framework language and
is acceptable.

SECTION 15 — DECISION TREE
============================================================

After all 12 criteria and both structural checks are complete,
apply this decision tree:

STEP 1: Did either structural check fail?
  YES → RESULT C: REJECT AND REGENERATE
  NO  → Continue to Step 2

STEP 2: Did any Tier 1 criterion fail?
  YES → RESULT C: REJECT AND REGENERATE
        (Include ALL failures in the Regeneration Instruction,
        not just Tier 1 failures. If Tier 2 or Tier 3 criteria
        also failed, include them so Stage 3 addresses
        everything in one regeneration pass.)
  NO  → Continue to Step 3

STEP 3: Did any Tier 2 criterion fail (not correctable)?
  YES → RESULT C: REJECT AND REGENERATE
        (A Tier 2 failure that cannot be auto-corrected —
        typically a structural gap — requires regeneration.)
  NO  → Continue to Step 4

STEP 4: Were any Tier 2 or Tier 3 corrections applied?
  YES → RESULT B: AUTO-CORRECTED PASS
        (Apply all corrections. Verify that corrections do
        not introduce new failures. Document all changes in
        the Audit Summary.)
  NO  → Continue to Step 5

STEP 5: All criteria passed without correction.
  → RESULT A: PASS

SECTION 17 — AUTO-CORRECTION RULES
============================================================

When applying auto-corrections (Result B), follow these rules:

1. PRESERVE INTENT
   Corrections must preserve the coaching intent of the
   original text. You are adjusting execution, not direction.

2. MINIMAL CHANGES
   Make the smallest change that resolves the issue. Do not
   rewrite paragraphs when a sentence edit suffices.

3. NO NEW CONTENT
   You may trim, replace words, remove sentences, or reorder.
   You may NOT add new coaching insights, new strength signals,
   new identity anchors, or new micro-commitments.

   Exception: You MAY add:
   - An athlete voice mirror sentence (Criterion 5) using
     exact language from the athlete_voice JSON field
   - An upcoming context anchor sentence (Criterion 7) using
     exact data from the upcoming_context JSON field
   - A missing action element (Criterion 6) using exact data
     from the micro_commitment JSON field
   - A center-ring redirect sentence (Criterion 12) using
     the most_important_controllable from the JSON

   These are permitted because they insert JSON-sourced data,
   not new coaching interpretation.

4. VERIFY AFTER CORRECTION
   After applying all corrections, re-read the corrected
   output to verify:
   - The corrections did not introduce new issues
   - The flow still reads naturally
   - No JSON data was contradicted by the correction
   - The message length is still within range

5. DOCUMENT EVERY CHANGE
   Every correction must be listed in the Audit Summary
   under "CORRECTIONS APPLIED" with:
   - Which criterion prompted the correction
   - What was changed (original → corrected)
   - Why the change was made

SECTION 18 — EDGE CASES AND JUDGMENT CALLS
============================================================

1. WEEK 1 ATHLETES (No Prior Data)
   - prior_commitment_result.execution_status = "No Prior Week"
     → Criterion 4 auto-passes
   - narrative_arc = "insufficient data"
     → Criterion 8 = N/A
   - longitudinal_metrics = "insufficient data"
     → Trendline references in Criteria 3 are not expected
   - The audit still applies all other criteria fully.

2. EMOTIONAL INTENSITY = "INSUFFICIENT DATA"
   Treat as Low for Criterion 1 purposes. Standard coaching
   tone applies.

3. MULTIPLE TIER 1 FAILURES
   Report ALL of them. Do not stop at the first one. Stage 3
   needs to know everything that must change in one pass.

4. TIER 2 FAILURE THAT IS BORDERLINE CORRECTABLE
   When in doubt about whether a correction is possible
   without adding new coaching content, REJECT rather than
   attempt a correction that might introduce new content.
   The operating principle is: correct what you can, reject
   what you cannot. If you are uncertain, you cannot.

5. DEEP DIVE SECTION COUNTS
   The framework specifies 4-6 Strengths sections and 3-5
   Growth sections. If the output has 3 Strengths or 6
   Growth sections:
   - This is a structural deviation. Log it as a NOTE.
   - It does not automatically trigger rejection unless
     the section imbalance indicates a deeper issue (e.g.,
     only 2 Strengths on a Growth-tier week suggests the
     output is not reflecting the JSON accurately, which
     is a Data Fidelity issue).

6. WHEN COACHING OUTPUT IS TECHNICALLY CORRECT BUT TONALLY FLAT
   If all 12 criteria pass but the output reads as lifeless,
   formulaic, or robotic, log a NOTE in the Audit Summary.
   This is a quality signal for human QA review but does not
   constitute a failure under any defined criterion. The
   Audit Agent does not rewrite for style preference.

7. CONFLICTING CORRECTIONS
   If correcting one criterion would cause another to fail
   (e.g., adding an athlete voice mirror pushes the message
   over the length limit), apply this priority:
   - Tier 1 compliance > Tier 2 content > Tier 3 polish
   - If adding content (Tier 2 correction) causes a length
     violation, trim other content (Tier 2 correction for
     length) to make room
   - Never sacrifice a Tier 1 criterion to accommodate a
     lower-tier correction
